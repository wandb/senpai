### Idea 1: Panel Method as Cheap Data Oracle (TIER 1 — Highest Priority)

**What it is:** Use the existing vortex-panel solver (already in codebase via `--cp_panel`, `--vortex_panel_velocity`) to generate 3,000–5,000 additional *inviscid* training samples for novel tandem configurations not in the original training set. Train on this augmented dataset with a low weight on panel-generated labels, treating them as self-consistent low-fidelity supervision.

**Why it helps p_tan:** The tandem inter-foil regime is data-starved — the model sees relatively few tandem gap/stagger/AoA combinations at training time. The panel solver is already computing correct inviscid pressures for tandem configurations; using those as cheap auxiliary labels directly addresses the sparse coverage of tandem parameter space. This is fundamentally different from the failed viscous residual approach (#2324, #2362) — here panel labels are used as *additional training data*, not as a residual target.

**Literature support:**
- Springer Nature / Advances in Aerodynamics (2025): "Inductive transfer-learning of high-fidelity aerodynamics from inviscid panel methods" — shows up to 10² MSE reduction when pretraining on inviscid panel data before fine-tuning on viscous CFD. Direct precedent for this approach.
- NeuralFoil (arXiv:2503.16323, Mar 2025): uses panel method as physics oracle for airfoil ML surrogates.
- Multi-fidelity Bayesian neural networks (Zenodo dataset, May 2025): demonstrates multi-fidelity transfer from low-fidelity inviscid to high-fidelity viscous aerodynamics.

**Key implementation notes:**
- Infrastructure already exists: `--cp_panel` generates panel Cp; `--vortex_panel_velocity` generates induced velocities. The panel solver code is in train.py.
- Generate synthetic tandem configs by sampling: gap ∈ [0.3c, 1.5c], stagger ∈ [-0.3c, 0.3c], AoA pairs (α1, α2) ∈ [-5°, 15°]², Re ∈ [5e4, 5e5]. Use the NACA 4-series geometries already in the training set.
- Label generation: for each synthetic config, run the vortex-panel solver to get inviscid Cp at surface nodes. Set velocity (Ux, Uy) using potential flow (no separation — these labels will be biased for high-AoA but still informative for geometry).
- Training: add synthetic samples with `label_weight=0.1–0.3` (lower weight than CFD samples). A new `--panel_oracle_weight` flag and a `--panel_oracle_n 3000` flag.
- Alternatively — and likely simpler — use panel labels only for the SURFACE nodes (Cp), not volume nodes. This is self-consistent with what the panel solver computes accurately.
- Gotcha: panel solver labels have NO viscous effects (no separation, no boundary layer). At high AoA (>12°), panel Cp is wrong. Clamp synthetic AoA to ≤10° or add a confidence weight that decays with sin(AoA).

**Suggested experiment design:**
- Generate 3000 synthetic tandem samples using the panel solver offline before training.
- Add a `--panel_oracle_data` flag that loads these samples and mixes them into training batches.
- Start with `panel_oracle_weight=0.1` (same as cp_panel_scale) — lower is safer to avoid label poisoning.
- Train with identical hyperparameters as baseline, 2 seeds.
- Measure: does p_tan improve? Does p_oodc improve (more AoA diversity)? Does p_in degrade?

**Expected improvement:** 5–10% p_tan reduction. Medium-high confidence — direct literature precedent, infrastructure exists, conceptually clean.

**Key risk:** Inviscid label bias at high AoA. Mitigate by clamping synthetic AoA ≤10°.

### Idea 2: Mamba/SSM Surface Sequence Model (TIER 1)

**What it is:** Replace the Surface Refinement Head (SRF) — currently a per-node MLP — with a Mamba State Space Model operating on surface nodes ordered by arc-length. Mamba processes the sequence O(n) rather than O(n²) like attention, enabling 6–8 stacked layers within the same compute budget.

**Why it helps p_tan:** The SRF's weakness is treating surface nodes as an *unordered set*. The aft-foil boundary layer in tandem configurations has strong arc-length structure: the stagnation point shifts, the suction peak moves, and separation (if any) propagates from LE to TE. A sequential model that can learn "what happened upstream on the surface" naturally captures this propagation. Mamba's gating mechanism (selective state space) can learn to forget irrelevant upstream history and retain the physically-relevant wake entry state.

**Literature support:**
- arXiv:2410.02113 (Oct 2024, revised Mar 2026): "Mamba Neural Operator: Who Wins?" — MNO outperforms FNO and comparable Transformer operators on multiple PDE benchmarks. GitHub: Math-ML-X/Mamba-Neural-Operator.
- ICML 2025 Proceedings: "Latent Mamba Operator for Partial Differential Equations" (Tiwari et al., v267) — Mamba as latent operator for PDEs.
- arXiv:2409.03231: "State-space models are accurate and efficient neural operators for dynamical systems" — SSMs match or beat attention-based operators at fraction of compute.

**Key implementation notes:**
- Use `mamba_ssm` package (pip-installable, MIT license).
- **CRITICAL: Mamba broke torch.compile in Phase 5.** Test compatibility with `torch.compile` disabled first; add `--no_compile` fallback.
- Replace the SRF MLP stack with: sort surface nodes by arc-length → Mamba(d_model=192, d_state=16, d_conv=4, expand=2) × 4 layers → linear output projection.
- Arc-length ordering: use the `arc_length` field already computed in `prepare.py`/`prepare_multi.py`. For tandem, treat fore/aft foil surface sequences independently then pool (or use a separator token).
- Keep the backbone Transolver unchanged — Mamba only replaces the SRF output head.
- d_state=16 is the standard default; increase to 32 if VRAM allows (96GB is ample).
- Bidirectional Mamba (scan both forward and reverse along arc-length) is worth trying — it removes the causal assumption, which is physically correct for steady-state flow.

**Suggested experiment design:**
- Implement `MambaSRF` as a drop-in replacement for `SurfaceRefinementHead`.
- Use `--mamba_srf` flag to activate.
- First run: `d_state=16, d_conv=4, expand=2, n_layers=4, bidirectional=True`.
- If torch.compile fails: disable compile for SRF module only (`@torch.compile` on backbone, no compile on SRF).
- 2 seeds as standard.

**Expected improvement:** 3–8% p_tan, 2–5% p_in. Medium confidence — strong PDE literature precedent; SRF is a clear bottleneck identified by prior arc-length experiments (#2337, #2338, #2343).

**Key risk:** Mamba compile incompatibility. Have fallback: eager mode for SRF only.

### Idea 3: SE(2)-Equivariant Geometry Encoding (TIER 1)

**What it is:** Replace the current DSDF/SDF input features (which are *not* equivariant to rotation/reflection) with SE(2)-equivariant geometric features: distances (invariant) + relative angles encoded as (cos θ, sin θ) pairs. Alternatively, apply an SE(2)-equivariant GNN layer at the input stage to produce rotation-invariant node embeddings before the Transolver backbone.

**Why it helps p_tan:** The current model's generalization failure on tandem OOD cases may partly stem from implicit coordinate-frame dependence in the input features (DSDF uses absolute x, y coordinates; TE coordinate frame helps but is incomplete). SE(2)-equivariant features ensure that rotating the entire tandem configuration produces a consistently rotated output — a hard physical constraint the model currently enforces only softly through data augmentation. For tandem, the relative foil orientation (stagger angle) is critical, and equivariant encoding makes this relationship explicit.

**Literature support:**
- arXiv:2405.20287 (2024): "Flexible SE(2) graph neural networks with applications to PDE surrogates" (Bankestad, Mogren, Pirinen) — direct application of SE(2)-equivariant GNNs to PDE surrogates, shows improved OOD generalization. Exactly the setting we are in.
- ICML 2024 AI for Science Workshop: "SE(3)-Equivariant Diffusion Graph Nets" (Lino, Pfaff, Thuerey) — SE(3)-equivariant VGAE + diffusion for flow fields on graphs.
- NeurIPS 2024: "Space-Time Continuous PDE Forecasting using Equivariant Neural Fields" (Knigge et al., UvA) — equivariant neural fields for PDE forecasting.
- NSF Public Access: "Robust Emulator for Compressible Navier-Stokes using Equivariant Geometric Convolutions" (2024) — equivariant convolutions for robustness in NS surrogates.

**Key implementation notes:**
- Minimal implementation: replace absolute (x, y) input coordinates with (r, cos θ, sin θ) relative to a canonical reference point (e.g., foil centroid). This alone makes the model rotation-equivariant without changing the architecture.
- More principled: use the `e3nn` or `escnn` library for SE(2)-equivariant layers. Apply a single SE(2)-equivariant linear layer to map raw coordinates to invariant/equivariant embeddings before the Transolver.
- The tandem-specific benefit: encode stagger angle as (cos Δθ, sin Δθ) relative to fore foil — this is invariant to global rotation and captures the relative geometry exactly.
- Simplest experiment: add (cos θ, sin θ) encoding of the angle from each node to foil centroids as input features (2 channels per foil = 4 extra channels for tandem).

**Suggested experiment design:**
- Minimal version: add 4 channels encoding SE(2)-invariant relative angle features from each node to fore/aft foil centroids. Test with `--se2_relative_angle` flag.
- This is a pure input feature addition — no architecture change, zero risk of breaking anything.
- If this works, escalate to SE(2)-equivariant input embedding layer.

**Expected improvement:** 2–5% p_tan, 2–4% p_oodc (OOD gains from better geometric invariance). Medium confidence.

**Key risk:** May be redundant with TE coordinate frame (`--te_coord_frame`) which already provides some rotational encoding. Need to verify non-redundancy.

### Idea 4: Attentive Neural Process for Conditional Pressure Prediction (TIER 2)

**What it is:** Frame the pressure field prediction as a *conditional* distribution estimation problem using an Attentive Neural Process (ANP). The context set consists of surface nodes with their (coordinates, panel Cp) pairs; the target set consists of all surface nodes where we want to predict viscous pressure. The ANP's attention mechanism directly learns which context points are relevant for each target, enabling better OOD generalization through its explicit uncertainty quantification.

**Why it helps p_tan / p_oodc:** Neural Processes are designed specifically for few-shot/OOD generalization — they maintain a distribution over functions conditioned on context observations. For the tandem case, the "context" can include the fore-foil surface (which has higher fidelity panel predictions) as conditioning context for the aft-foil predictions. The inter-foil conditioning is the exact mechanism missing from the current architecture. ANPs also directly target p_oodc by treating OOD samples as having less dense context coverage.

**Literature support:**
- ICLR 2023: "Bridge the Inference Gaps of Neural Processes via Expectation Maximization" (Wang, Federici, van Hoof, UvA) — addresses the training/inference gap in ANPs.
- ICLR 2025 workshop: "Sparse Gaussian Neural Processes" (Rochussen, Fortuin, Helmholtz AI / TUM) — scalable neural processes.
- Attentive Neural Process (Kim et al., ICLR 2019): the original ANP, cross-attention between context and target.

**Key implementation notes:**
- Architecture: context encoder (MLP on surface coordinates + panel Cp), cross-attention from target nodes to context nodes, latent aggregation, decoder MLP.
- Integrate as a replacement for/addition to the SRF: use the ANP as the final surface decoder, with backbone hidden states as additional input features.
- For tandem: fore-foil surface nodes serve as cross-foil context for aft-foil target nodes. This directly encodes inter-foil information flow.
- The `--cp_panel` feature infrastructure already provides what the ANP needs as context features.
- Latent variable: for uncertainty quantification, can use a diagonal Gaussian latent. For the regression task, can use a deterministic ANP (skip the latent) for simplicity and speed.
- CRITICAL: the 30-min training timeout constrains model size. Limit attention heads to 4, context set to surface nodes only (~256–512 per foil).

**Suggested experiment design:**
- Replace SRF with a deterministic ANP (no latent variable): context = surface nodes with (coord, cp_panel, backbone_hidden), target = all surface nodes.
- For tandem samples: context = fore-foil surface (fixed) + local neighborhood of each target node. Aft-foil gets cross-foil context from the fore-foil.
- `--anp_srf --anp_heads 4 --anp_hidden 192` flags.

**Expected improvement:** 4–8% p_tan (via cross-foil context), 3–6% p_oodc. Lower confidence — novel application to CFD, no direct prior.

**Key risk:** The 30-min training timeout may not be enough to train a full ANP decoder. Start with deterministic ANP (no latent variable) to reduce complexity.

### Idea 5: Contrastive Geometry Pretraining on UIUC Airfoil Library (TIER 2)

**What it is:** Pretrain a geometry encoder (the part of the model that processes airfoil shape) using contrastive learning on the UIUC airfoil library (~1,600 airfoils) and NACA 4/5 series parametric family. Positive pairs: geometrically similar airfoils (same camber/thickness family). Negative pairs: dissimilar families. Then fine-tune on the full TandemFoilSet.

**Why it helps p_tan:** The model currently learns geometry representations only from the limited geometry diversity in TandemFoilSet. Pretraining on a 100× larger geometry library teaches the encoder geometry-aware representations — thickness distribution, camber line, LE radius — that generalize better to OOD geometries. For tandem, good geometry representations are crucial for the model to correctly assign roles to fore/aft foils.

**Literature support:**
- SimCLR (Chen et al., ICML 2020): contrastive pretraining framework for general representations.
- DINOv2 (Oquab et al., 2024): self-supervised pretraining shows strong transfer to downstream tasks.
- NeuralFoil (arXiv:2503.16323, Mar 2025): trains on large UIUC+NACA airfoil libraries.
- GeoMPNN (arXiv:2412.09399, NeurIPS 2024 ML4CFD Best Student Paper): geometry-aware message passing for CFD surrogates — shows geometry encoding quality is a primary bottleneck.

**Key implementation notes:**
- Geometry representation: the Transolver backbone processes the x-coordinate input features. Freeze or fine-tune the first N=1 transformer layers as the geometry encoder.
- Contrastive objective: InfoNCE loss on geometry embeddings. Positive pairs: same airfoil family (NACA 4412 and NACA 4415 are positive; NACA 4412 and NACA 23012 are negative).
- UIUC database: 1,600+ airfoils, freely available as coordinate files. Can be sampled onto a uniform mesh using the existing data pipeline.
- Pretraining does NOT require CFD labels — only geometry coordinates needed. Panel solver can generate cheap inviscid labels for additional supervision.
- Two-phase training: Phase 1 (pretraining, geometry only, 30 min), Phase 2 (fine-tuning on TandemFoilSet with frozen geometry encoder, 30 min).
- CRITICAL: within the 30-min timeout, may need to shorten Phase 2. Options: reduce epochs to 100, use higher LR for fine-tuning.

**Suggested experiment design:**
- Implement a `--contrastive_pretrain` flag that runs geometry encoder pretraining before the main training loop.
- Generate UIUC geometry samples offline, store as a numpy file.
- InfoNCE loss with temperature=0.07, batch size=128 geometry pairs.
- After pretraining, freeze geometry layers and train on TandemFoilSet for 100 epochs.

**Expected improvement:** 3–6% p_tan, 3–5% p_oodc. Medium-low confidence — transfer learning overhead may eat into the training budget.

**Key risk:** Two-phase training may not converge well within 30-min timeout. Consider using pretrained weights as initialization only (no freezing) to reduce the performance gap between phases.

### Idea 6: Invertible Koopman Lifting for Tandem Interference (TIER 2)

**What it is:** Learn a Koopman operator embedding that lifts the nonlinear tandem interference dynamics to a higher-dimensional linear space where the fore-foil effect on the aft-foil is approximately linear. The Koopman eigendecomposition naturally separates "modes" of inter-foil interaction.

**Why it helps p_tan:** The tandem inter-foil interference is a quadratic nonlinearity in angle of attack (fore-foil downwash changes the aft-foil effective AoA by Δα ≈ f(gap, stagger, α1)). The Koopman lifting maps this nonlinear coupling to a linear map L such that z(aft) = L · z(fore) in the lifted space. If the model learns to disentangle these Koopman modes, it can generalize the tandem interference pattern to unseen gap/stagger combinations — directly targeting p_tan.

**Literature support:**
- Papers With Code (Mar 2025): "Invertible Koopman neural operator for data-driven modeling of PDEs" — Koopman operator for PDE data-driven modeling, invertible architecture.
- arXiv:2503.22649 (Mar 2025): "Stochastic reduced-order Koopman model for turbulent flows" — Koopman for turbulent flow modeling.
- arXiv:2410.20072: "CGKN: A Deep Learning Framework for Modeling Complex Dynamical Systems and Efficient Data Assimilation" — Koopman framework for complex dynamics.

**Key implementation notes:**
- Implementation: add a Koopman lifting layer that maps the backbone hidden state (after all TransolverBlocks) to a linear space, applies a linear Koopman operator (parameterized as a low-rank matrix), then projects back.
- For tandem: the Koopman operator maps fore-foil encoded state to predicted aft-foil state delta. Non-tandem samples use identity operator (skip).
- Invertibility: use an encoder-decoder structure with reconstruction loss on the lifting (||z → x'|| ≈ ||x||) to prevent trivial solutions.
- Tandem-specific Koopman: K_tandem = U Σ V^T with rank 16–32. Apply this linear map to predict aft-foil corrections.
- Koopman decomposition can be regularized with physics constraints: the dominant mode should correspond to downwash angle shift.

**Suggested experiment design:**
- Add `--koopman_tandem` flag: activates a Koopman inter-foil lifting layer for tandem samples only.
- Koopman rank = 32, lifting dimension = 256.
- Loss: main task loss + 0.1 × Koopman reconstruction loss.
- 2 seeds as standard.

**Expected improvement:** 3–6% p_tan. Low-medium confidence — theoretically well-motivated but no direct aerodynamics surrogate precedent.

**Key risk:** Koopman operator rank may need careful tuning. Start with rank 32, add rank sweep if promising.

### Idea 7: Scale-Autoregressive Multiscale Pressure Decoder (TIER 2)

**What it is:** Decode surface pressure in a coarse-to-fine autoregressive manner: first predict a low-frequency pressure field (first K Fourier modes on the arc-length parameterization), then decode residuals at progressively finer scales. This is "scale-autoregressive" generation applied to surface pressure profiles.

**Why it helps p_tan:** The tandem pressure prediction error has two components: (1) global offset (wrong mean pressure level on aft foil — dominated by downwash AoA shift), and (2) local structure (suction peak location error, stagnation point offset). A coarse-to-fine decoder explicitly separates these: the first scale captures the global AoA-shift effect, subsequent scales refine the local structure. This is analogous to pyramid decoding in computer vision but physically motivated by the Fourier structure of airfoil Cp profiles (which are dominated by low-frequency modes).

**Literature support:**
- OpenReview (under review ICLR 2026): "One Scale at a Time: Scale-Autoregressive Image Synthesis" — scale-autoregressive generation in image synthesis.
- Multi-Scale Slice Attention (#2373, in-flight): this experiment is testing something similar at the backbone level. This idea differs by applying coarse-to-fine at the DECODER level, not the encoder.
- 1D Surface FNO Decoder (#2371, in-flight): also targets spectral surface structure but uses FNO rather than autoregressive decoding.

**CAUTION:** Ideas 7 partially overlaps with in-flight experiments #2371 and #2373. Recommend waiting for their results before assigning this. If both fail, this becomes Tier 1.

**Suggested experiment design:**
- Decompose the surface pressure prediction into K=4 Fourier mode groups (DC, modes 1–4, modes 5–16, modes 17–64).
- Predict each group sequentially, conditioned on the previously decoded groups.
- Loss: L1 on each Fourier group separately, summed.
- Implement as a stacked SRF with skip connections between scales.

**Expected improvement:** 3–7% p_tan. Medium confidence if #2371/#2373 fail; lower priority if those experiments are in-flight.

### Idea 8: Diffusion Graph Network Decoder for Tandem Pressure (TIER 3)

**What it is:** Replace the deterministic SRF output head with a score-based diffusion decoder operating on the surface node graph. During training, add noise to the pressure targets and train a graph neural network to denoise them. At test time, run 10–20 DDIM steps for fast inference.

**Why it helps p_tan:** The diffusion decoder from ICLR 2025 (Lino, Pfaff, Thuerey) demonstrated that diffusion graph networks can model complex multi-modal distributions of flow fields that deterministic models miss. For tandem, the aft-foil pressure distribution is genuinely multi-modal depending on gap/stagger (attached vs separated flow, leading-edge vs trailing-edge dominated Cp profile). A generative model can capture this distribution rather than averaging over modes.

**Literature support:**
- ICLR 2025: "Learning Distributions of Complex Fluid Simulations with Diffusion Graph Networks" (Lino, Pfaff, Thuerey, TUM/DeepMind) — diffusion on graphs for fluid simulations.
- arXiv:2511.12455 (Nov 2025): "Generative Reconstruction of Spatiotemporal Wall-Pressure in Turbulent Boundary Layers via Patchwise Latent Diffusion" — latent diffusion for wall-pressure reconstruction.
- arXiv:2407.00735 (Jun 2024): "Generative prediction of flow fields around an obstacle using the diffusion model" — diffusion for flow field prediction.
- Note: Diffusion Surface Decoder (#2349) was tried and failed. That version used the SRF directly as the diffusion network without proper graph structure. This idea uses a proper Graph Neural Network as the diffusion backbone — different architecture.

**CAUTION:** PR #2349 tried a diffusion surface decoder and failed. Read #2349's failure analysis before implementing. The key difference here is: (1) proper GNN backbone for the diffusion model (not just MLP), (2) DDIM fast sampling instead of DDPM. Difference must be clearly articulated to the student.

**Suggested experiment design:**
- Implement diffusion as a surface-node GNN with edges between k=8 nearest neighbors on arc-length.
- Use DDIM (10 steps at test time) for speed.
- Keep backbone Transolver unchanged; diffusion only in the SRF.
- Add `--diffusion_srf --ddim_steps 10 --diffusion_timesteps 100` flags.

**Expected improvement:** 3–5% p_tan. Low confidence given #2349 failure — only try after higher-tier ideas are exhausted.
