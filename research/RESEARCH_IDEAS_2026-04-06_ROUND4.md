Every one of these is a local move. The ideas below work at a categorically different level.

**Decision criterion for this round:** An idea qualifies for this list only if it addresses a structural weakness that cannot be fixed by tuning a hyperparameter in the current architecture. Each idea below either (a) changes the information available to the model at forward-pass time, (b) changes how the model is trained before or separately from the main training loop, or (c) changes the prediction target itself.

## Idea 1: Panel-Method Inviscid Cp as Physics-Informed Input Features

### What it is
Run a fast 2-element vortex panel solver (e.g., `aeropy`, `xfoil-python`, or `PyXfoil`) on every training sample to compute the inviscid pressure coefficient Cp at each surface node. Feed this as an additional input feature channel alongside the existing geometric descriptors (DSDF, coordinates). The network then learns to predict the viscous correction residual Δp = p_viscous - p_inviscid rather than p from scratch.

### Why it might help here
This directly attacks the core structural weakness: p_tan is 2.19x worse than p_in because the NACA6416 fore-foil shape is OOD and its upstream wake creates pressure patterns the model has never seen interpolate. An inviscid panel method solves the exact same tandem geometry analytically — the Cp distribution is a deterministic function of the geometry alone, not of the training data distribution. The viscous correction (boundary layer effects, separation) is a much smaller, smoother signal. By making the model predict only the correction, we reduce the dynamic range of the residual target and give the model a physically grounded prior.

**B-GNN (arXiv:2503.18638, 2025)** tested exactly this on single-foil cases: adding inviscid Cp + local Re_x as input features achieved 88% OOD error reduction and 87% reduction in required training data. The tandem case may benefit even more since the panel method naturally captures the fore-aft aerodynamic interaction (downwash, circulation effects) which is precisely what the network struggles with.

**NeuralFoil (arXiv:2503.16323, 2025)** adopted the same blueprint: panel solution as physics prior → neural network predicts viscous correction Δp.

### Key papers
- B-GNN: "Boundary Condition Informed Graph Neural Network" arXiv:2503.18638 — 88% OOD error reduction with panel-method features
- NeuralFoil: arXiv:2503.16323 — panel method baseline + neural viscous correction for single-element airfoils
- These validate the approach on adjacent problems with strong results

### Implementation plan
1. Pre-compute inviscid panel solution offline for all training and validation samples. A 2-element panel code in Python can process ~100 samples/second. For 50k training samples at AoA × gap/stagger sweep, pre-computation takes <15 minutes. Store as a `.npy` cache keyed by case ID.
2. Interpolate panel Cp values to mesh nodes using the DSDF-based surface assignment already in the codebase (surface nodes are already identified).
3. Add the panel Cp as feature index 25 (or wherever the next slot is in the 24-dim input) for surface nodes; fill with 0 for volume nodes.
4. The network sees panel_Cp as a new input channel. The prediction target remains the full p (the residual connection in `--residual_prediction` already accounts for the freestream; now we can layer the panel prediction on top).
5. Alternatively: add a flag `--panel_cp_residual` that makes the loss compute against `p - panel_Cp` instead of `p - p_freestream`, reducing the residual to pure viscous correction.

**Simplest first experiment:** Add pre-computed panel Cp as a new input feature (no change to loss), train with everything else held at baseline. If panel Cp is informative, the model should learn to use it automatically. One flag: `--panel_cp_feature`.

**Risk:** Panel solver may not handle extreme AoAs or thick-foil separation correctly. Clamp or use a simple linear vortex panel code for reliability. The viscous correction still needs to be learned but the feature provides a useful prior.

**Expected impact on p_tan:** High. The panel method natively solves for fore-aft aerodynamic interaction.

**Confidence:** Strong evidence from B-GNN and NeuralFoil in adjacent settings. Main uncertainty is tandem-foil panel code correctness at OOD configurations.

## Idea 2: Ensemble Distillation — Train a Single Model to Match Soft Ensemble Predictions

### What it is
We have 45+ trained model checkpoints from all prior experiments. Rather than using them as a post-hoc ensemble, use their averaged predictions as soft training targets for a fresh single model. This is knowledge distillation where the teacher is the ensemble.

More specifically: for each training sample, run inference through the 23-model ensemble (16-seed + 7 seeds 100-106) to produce soft prediction targets. The student model is trained to minimize KL divergence (for classification) — but for regression, train on a weighted sum of L1 loss to the ground truth plus L1 loss to the ensemble mean. The insight from DEGU (npj AI, 2025) is that students trained this way can *outperform* the ensemble by exploiting the ensemble's variance-reduced signal.

### Why it might help here
The single model already beats the 16-seed ensemble on p_tan (28.60 vs 29.1). This paradox means: the ensemble is averaging toward a biased mean, while the best single models learn something different. A distilled student trained on ensemble soft targets avoids ensemble bias while inheriting ensemble variance reduction.

For the OOD tandem case: the ensemble predictions have lower variance than any individual model, reducing noise in the training signal for these harder examples. The student can then specialize more aggressively on the high-error regions.

### Key papers
- DEGU: "Distribution-free Ensemble Distillation for Generalized Uncertainty" npj Computational Materials (2025) — student outperforms ensemble on regression tasks by learning distributional targets
- Hinton et al. (2014): original knowledge distillation; teacher-soft-target training improves generalization vs hard labels

### Implementation plan
1. Generate ensemble soft predictions offline: run all 23 models on the full training set, average their predictions. Store as a `.npy` cache. 23 models × 160 epochs × inference = ~20 GPU-minutes.
2. Add a new loss term: `L_distill = alpha * L1(pred, ensemble_mean_target)` where `ensemble_mean_target` is loaded as an additional target tensor.
3. Combined loss: `L = (1 - alpha) * L1(pred, gt) + alpha * L1(pred, ensemble_mean)`. Try `alpha = {0.3, 0.5}`.
4. Alternatively try matching ensemble variance (uncertainty) as an auxiliary loss, so the student also learns where the ensemble is uncertain.

**Simplest first experiment:** `--ensemble_distill_target <path_to_cached_predictions>` with `alpha=0.3`. Run 2 seeds.

**Risk:** The ensemble mean may itself be biased (systematic underprediction at leading edge). The student inherits this bias. To test: compare student vs ensemble on OOD validation first.

**Expected impact on p_tan:** Medium. The ensemble already reaches 29.1; the student might get to 27-28 range if it exploits ensemble variance-reduction without the bias.

**Confidence:** Moderate. DEGU is convincing for regression settings but the ensemble quality here is uncertain (ensemble p_tan=29.1 is already *worse* than best single model p_tan=28.60).

## Idea 3: Test-Time Low-Rank Adaptation (GEPS) for OOD Tandem Generalization

### What it is
At inference time on the OOD NACA6416 tandem validation cases, run a small number (5-20) of gradient steps to adapt a minimal set of low-rank context parameters, using a physics-based self-supervised signal (no ground truth needed). The adapted parameters are thrown away after each test sample.

The signal for adaptation can be: (1) continuity equation residual ∂Ux/∂x + ∂Uy/∂y ≈ 0 on a neighborhood of nodes, (2) pressure-velocity Bernoulli consistency at surface nodes, or (3) matching the inviscid panel Cp at surface nodes (which is always computable).

### Why it might help here
The NACA6416 fore-foil shape is OOD — it is unlike anything in the training distribution. Standard inference cannot generalize across this gap. If we can adapt the model's latent representation using physics equations that hold regardless of foil shape, the model can effectively "see" this test case as a free-field adaptation problem.

**GEPS (arXiv:2410.23889, NeurIPS 2024)** showed that low-rank context adaptation at test time (without ground truth) generalizes PDE surrogates to unseen initial conditions, PDE coefficients, and forcing terms. The key: the adapted parameters are a small, low-rank set (frozen large weights, only context parameters are updated). This prevents overfitting to noise.

### Key papers
- GEPS: "Generalization of PDE Surrogates via Context Adaptation" NeurIPS 2024, arXiv:2410.23889 — test-time low-rank adaptation for unseen PDE parameters; validated on Burgers, Navier-Stokes, Darcy
- LoRA (Hu et al., 2022): low-rank adaptation mechanism that GEPS builds on

### Implementation plan
1. Add a small set of low-rank context parameters (r=4 rank, 2 matrices per attention layer) initialized to zero. Total new parameters: `n_layers * 2 * r * n_hidden` = 3 * 2 * 4 * 192 = ~4600 params.
2. At inference: for each OOD test batch, run 10-20 gradient steps on only the context parameters, minimizing the continuity residual `||div(U_pred)||` computed via finite differences on the mesh.
3. Use the adapted model for prediction, then reset context parameters to zero.
4. Training: the context parameters are trained to zero by default; the adaptation is purely test-time.

**Implementation note:** This requires mesh connectivity information to compute finite differences. The mesh structure is available in `data/prepare.py`. A simple graph-Laplacian divergence approximation using nearest neighbors should suffice.

**Alternative signal:** If continuity residual is hard to implement, use the panel Cp matching signal (Idea 1) — always available at test time without any GT.

**Simplest first experiment:** Add `--tta_context_adapt --tta_steps 10` flag. Train normally. At validation on `val_ood_tandem`, run adaptation before prediction. Compare against non-adapted baseline.

**Risk:** The continuity equation may not have sufficient gradient signal to guide adaptation toward lower pressure error. The adaptation might move in an uninformative direction.

**Expected impact on p_tan:** Medium-high. If the physics signal provides a useful gradient direction, TTA can close the OOD gap significantly (GEPS shows 10-40% reduction in OOD error).

**Confidence:** Moderate. GEPS validates on Navier-Stokes-adjacent problems. The key uncertainty is whether the incompressibility constraint gives useful gradients for pressure prediction specifically.

## Idea 4: MAE Pretraining on Mesh Geometry — Train Encoder to Reconstruct Masked Nodes

### What it is
Before the main supervised training loop, pretrain the Transolver encoder using masked autoencoder (MAE) self-supervision on mesh geometry only (no flow targets needed). Mask 25% of mesh nodes and train the encoder + a lightweight decoder to reconstruct the missing node features (coordinates, DSDF channels, surface normals estimated from DSDF gradients).

The encoder weights are then frozen or fine-tuned in the supervised training phase.

### Why it might help here
The current encoder has to learn two things simultaneously: (1) spatial reasoning about mesh structure and foil geometry, and (2) flow physics prediction. Pretraining on geometry alone separates these objectives. For OOD tandem cases where the foil geometry is novel, a geometry-pretrained encoder may generalize better because it has learned a richer geometric representation on ALL available samples (including unlabeled geometries, of which we may have thousands).

**GeoMAE (arXiv:2304.06911, CVPR 2023)** applied masked geometric reconstruction (surface normals, curvatures, centroids) to point cloud pretraining and showed significant improvements on downstream tasks. The Transolver slice-attention mechanism is structurally similar to set-abstraction in point cloud networks.

**Key insight:** We have potentially thousands of CFD mesh geometries available even without flow solutions (just the mesh files). Pretraining can use all of them.

### Key papers
- GeoMAE: "Masked Geometric Prediction for Self-Supervised Point Cloud Pre-Training" CVPR 2023, arXiv:2304.06911 — MAE on point cloud geometry improves downstream tasks
- He et al. (2022): original MAE — masking 75% of patches is optimal; strong regularization for representational learning

### Implementation plan
1. Add a `--pretrain_mae` mode to `train.py` that runs before supervised training.
2. MAE pretraining loop: for each batch, randomly mask 25% of nodes (replace features with learned [MASK] token), run the Transolver encoder to get node embeddings, run a 2-layer decoder MLP to reconstruct masked node features (x, y, DSDF channels 1-10).
3. MAE loss: L1 reconstruction on masked nodes only.
4. Save pretrained encoder weights. In supervised training: load pretrained weights with `--pretrain_checkpoint <path>` and fine-tune normally.
5. Pretraining: 50 epochs on geometry-only (no targets needed). Then supervised fine-tune for 160 epochs.

**Simplest first experiment:** Run pretraining for 50 epochs, then full supervised training. Compare to baseline trained from scratch. Flag: `--pretrain_mae --pretrain_epochs 50`.

**Risk:** Geometry pretraining may not transfer to flow prediction — the encoder needs to learn very different features for flow vs. geometry. But even as a regularizer, it may help OOD generalization.

**Expected impact on p_tan:** Medium. MAE pretraining is most valuable when labeled data is scarce. We have large labeled datasets, so the benefit may be marginal. However, the NACA6416 OOD case may benefit from better geometric understanding.

**Confidence:** Speculative but well-motivated. GeoMAE validates the geometry pretraining concept; the transfer to flow prediction has not been tested. Literature gap = genuine opportunity.

## Idea 5: Frequency-Weighted Surface Pressure Loss — DCT-Domain Loss for High-Frequency Errors

### What it is
Compute the 1D DCT (discrete cosine transform) of the surface pressure distribution along each foil surface (ordering nodes by arc-length). Apply a frequency-weighting that up-weights high-frequency DCT coefficients in the loss function. This forces the model to reduce errors in the high-frequency spatial components (leading edge stagnation point pressure peak, trailing edge separation, suction surface pressure recovery) which are currently dominated by the globally-averaged MAE.

### Why it might help here
The standard L1 surface loss weights all surface nodes equally. But pressure prediction accuracy is not uniform along the surface: the leading edge stagnation point, the suction peak near 15-25% chord, and the trailing edge are the highest-error regions and are also the most aerodynamically important. These regions correspond to high spatial frequency in arc-length space.

Standard deep learning is known to exhibit **spectral bias** — networks preferentially learn low-frequency components first. By up-weighting high-frequency DCT coefficients, we counteract this bias and force the model to attend to sharp features.

**FreqINR (arXiv:2408.13716, 2024)** demonstrated adaptive DCT-domain loss for implicit neural representations: assigning higher weights to high-frequency content improved reconstruction of sharp features. The approach is directly applicable to 1D surface pressure distributions.

### Key papers
- FreqINR: "Frequency-Aware Implicit Neural Representation" arXiv:2408.13716 (2024) — adaptive DCT loss for INRs; validated on signal reconstruction tasks
- Rahaman et al. (2019) "On the Spectral Bias of Neural Networks": foundational paper showing networks learn low-frequencies first

### Implementation plan
1. For each surface foil, order nodes by arc-length (already available via DSDF surface proximity or stored as a dataset metadata field).
2. Compute 1D DCT of both the predicted and ground-truth surface pressure arrays.
3. Apply frequency weighting: `w_k = 1 + gamma * (k / N)^alpha` where k is the DCT frequency index, N is the number of surface nodes. Try `gamma=2.0, alpha=1.5`.
4. Loss: `L_freq = mean(w_k * |DCT(p_pred)_k - DCT(p_gt)_k|)` summed over both foil surfaces.
5. Add as an auxiliary loss: `L_total = L_standard + lambda_freq * L_freq`. Try `lambda_freq = {0.1, 0.5}`.

**Simplest first experiment:** `--surface_freq_loss --surface_freq_gamma 2.0 --surface_freq_lambda 0.3`. This requires knowing arc-length node ordering, which might need a data preprocessing step.

**Implementation note:** Arc-length ordering of surface nodes is available from the mesh structure. If not directly available, it can be reconstructed by sorting surface nodes by angle around the foil centroid (which the DSDF already provides orientation for). The `prepare.py` or `prepare_multi.py` scripts may already have surface node ordering — check before implementing.

**Risk:** Surface node ordering may not be stored in the dataset; reconstructing it may introduce bugs. DCT assumes uniform arc-length spacing which may not hold for the mesh. Use a nearest-neighbor sorting approximation.

**Expected impact on p_tan:** Medium. Frequency-weighted loss addresses spectral bias which is a real phenomenon. The benefit depends on whether the leading/trailing edge errors are the binding constraint.

**Confidence:** Moderate. FreqINR validates DCT-domain losses for similar problems. The transfer to CFD surface pressure is well-motivated. Node ordering requirement adds implementation risk.

## Idea 6: Vorticity-Streamfunction Auxiliary Targets — Predict Physically Coherent Intermediate Representations

### What it is
Add auxiliary prediction heads for the vorticity field ω = ∂Uy/∂x - ∂Ux/∂y and the streamfunction ψ (where Ux = ∂ψ/∂y, Uy = -∂ψ/∂x). These can be computed exactly from the existing velocity targets using finite differences on the mesh. Train the model to predict ω and ψ as auxiliary outputs alongside p, Ux, Uy.

The key physical property: ω=0 everywhere in inviscid flow; ω≠0 only in the boundary layer. So predicting ω forces the model to localize boundary layer structure. The streamfunction is a scalar potential for the velocity field, providing a compact representation with built-in divergence-free structure.

### Why it might help here
The aft-foil pressure distribution in tandem flow is largely determined by the fore-foil's wake vorticity reaching the aft-foil. If the model explicitly predicts the vorticity field, it must learn where the wake is — which directly conditions the pressure distribution at the aft foil. This is not captured by the current pressure-first architecture which directly maps from geometry to pressure without representing wake structure.

**FINE (arXiv:2505.15329)** and **TrajectoryFlowNet (NeurIPS workshop 2024)** both showed that decomposing velocity into vorticity and streamfunction components provides physically interpretable intermediate representations that improve OOD generalization.

### Key papers
- FINE: "Flow Implicit Neural Encoding" arXiv:2505.15329 — vorticity-streamfunction decomposition with invertible spectral filters
- Physical fluid mechanics: Stokes stream function, Helmholtz decomposition

### Implementation plan
1. Pre-compute vorticity ω and streamfunction ψ from existing velocity fields offline. Use finite differences on the mesh: `omega_i = sum_j(Uy_j - Uy_i)/(x_j - x_i) - (Ux_j - Ux_i)/(y_j - y_i)` for nearest neighbors j of node i. Store in dataset.
2. Add auxiliary heads: two small 2-layer MLPs mapping the Transolver hidden representation to ω and ψ predictions.
3. Auxiliary losses: `L_vort = lambda_v * L1(omega_pred, omega_gt)` and `L_stream = lambda_s * L1(psi_pred, psi_gt)`.
4. The main prediction heads (p, Ux, Uy) are unchanged. The auxiliary losses are gradients only into the shared backbone.
5. Try `lambda_v = lambda_s = 0.1`.

**Simplest first experiment:** `--aux_vorticity --aux_vorticity_weight 0.1`. The vorticity computation requires mesh connectivity, which is available. Streamfunction requires integration over the mesh (harder) — implement vorticity first.

**Risk:** Vorticity computation from finite differences is noisy on unstructured meshes. Large errors in computed ω at mesh boundaries could hurt rather than help. Regularize with robust statistics (clip extreme vorticity values).

**Expected impact on p_tan:** Medium. The hypothesis that ω explicitly represents the wake is physically motivated. But auxiliary losses can also hurt by diluting gradient signal.

**Confidence:** Speculative but physically grounded. FINE validates vorticity decomposition for flow prediction; auxiliary loss benefit depends on the specific task structure.

## Idea 7: Multi-Resolution Hash Encoding of Mesh Coordinates — Replace or Augment DSDF with Spatial Hash Grid

### What it is
Replace or augment the existing DSDF geometric features with a **multi-resolution hash grid** encoding of (x, y) coordinates. The hash grid (as used in NeRF/Instant-NGP) encodes spatial position at multiple resolutions: coarse (global shape) and fine (local boundary layer gradients). This directly mitigates **spectral bias** — the network can access high-frequency spatial features without requiring the backbone to learn them from scratch.

### Why it might help here
The DSDF features encode distance/direction to foil surfaces but are fixed-resolution. The network must learn to map from this low-frequency input to high-frequency pressure gradients through all its layers. Multi-resolution hash encoding provides explicit high-frequency spatial features at no extra model cost — the network just has better raw inputs.

**Parametric hash encoding for physics surrogates (arXiv:2403.15652, UC Irvine, 2024)** showed that hash grid encodings applied to mesh coordinates for PDE surrogates reduce error by 30-50% on problems with fine spatial structure. The key: hash grids are trained end-to-end alongside the network.

**Key difference from Fourier PE:** Fourier features are fixed-frequency; hash grid features are learned and can represent arbitrary spatial frequencies by discovering which grid cells matter.

### Key papers
- arXiv:2403.15652 (2024): "Parametric hash encoding for physics surrogates" — UC Irvine, 30-50% error reduction on PDE surrogates
- Muller et al. (2022) Instant-NGP: original multi-resolution hash encoding — the mechanism
- Tancik et al. (2020): Fourier features baseline comparison

### Implementation plan
1. Implement a `HashGrid2D` module: 2D multi-resolution grid with L=8 levels, resolution doubling from 16 to 2048. At each level, bilinearly interpolate a learned 2D feature map at the (x, y) query position. Concatenate all levels: `L * features_per_level = 8 * 4 = 32` features.
2. Add hash grid features to the existing 24-dim node features: new input dim = 24 + 32 = 56.
3. Train hash grid parameters and model parameters jointly.
4. The hash grid resolution should be set to cover the CFD domain range [x_min, x_max] × [y_min, y_max], which is known from the dataset.

**Simplest first experiment:** `--hash_grid_encoding --hash_grid_levels 8 --hash_grid_features 4`. This requires implementing the `HashGrid2D` module in `train.py`.

**Key hyperparameters:** L=8 levels, feature_dim=4 per level, base_resolution=16, max_resolution=2048, learning rate for hash params can be higher (1e-3) than backbone (2e-4).

**Risk:** The hash grid must cover the full training domain. If inference is done at coordinates outside the training range (e.g., for transformed meshes), the grid will extrapolate poorly. The existing data augmentation (AoA rotation, DSDF perturbation) should not change the global coordinate range.

**Expected impact on p_tan:** Medium-high. If the binding constraint is the network's inability to represent high-frequency spatial features, hash grid encoding directly solves this.

**Confidence:** Moderate. The hash encoding paper validates on PDE surrogates. The integration into the Transolver architecture is straightforward. Main uncertainty: the DSDF features already encode geometric structure; the hash grid adds spatial structure. The two may be redundant.

## Priority Ranking and Next Assignments

Based on expected impact, implementation risk, and gap from existing experiments:

### Tier 1 — Highest priority, genuine departures from current approach

1. **Idea 1 (Panel Method Features)** — Most theoretically grounded, most direct attack on p_tan. Literature shows 88% OOD error reduction in adjacent settings. The physics-informed prior is hardest to get from any architecture change alone. Requires offline pre-computation of panel solutions (moderate implementation effort) and a new `--panel_cp_feature` flag.

2. **Idea 3 (Test-Time Adaptation)** — Addresses OOD gap without requiring any new training data. GEPS showed this works for PDE surrogates. The NACA6416 case is exactly the OOD scenario GEPS was designed for. Low implementation risk (small context parameters, gradient through existing forward pass).

### Tier 2 — High potential, harder to execute

3. **Idea 2 (Ensemble Distillation)** — We have the 45 checkpoints already. The main cost is generating soft predictions offline. DEGU evidence is solid for regression. However, the ensemble is already *worse* than the best single model on p_tan, which limits the distillation signal.

4. **Idea 7 (Hash Grid Encoding)** — Addresses spectral bias structurally. Relatively easy to implement in train.py. The 30-50% improvement from arXiv:2403.15652 is on problems with similar structure. Risk: redundancy with DSDF.

### Tier 3 — Speculative but valuable to explore

5. **Idea 5 (Frequency-Weighted Loss)** — Requires arc-length node ordering. Clean experiment if ordering is available. Addresses spectral bias at the loss level (complementary to Idea 7 at the input level).

6. **Idea 6 (Vorticity Auxiliary)** — Physically motivated but computationally expensive to precompute. The wake-vorticity connection to p_tan is a genuine insight. Worth trying if Tier 1/2 ideas don't break the plateau.

7. **Idea 4 (MAE Pretraining)** — Genuine literature gap for CFD surrogates. Highest uncertainty. Long training time (50 + 160 epochs). Lower priority given the cost and speculative nature. Worth trying if other ideas plateau.
