## Idea 1 (TOP PICK): MoE FFN in Last TransolverBlock — `moe-ffn-last-block`

**Target:** p_tan  
**Risk:** Low-Medium  
**Novel:** Yes — never tried in this form

**What it is:** Replace the shared FFN (`self.mlp`) in the last TransolverBlock's forward pass with a hard-dispatched pair of expert MLPs — one for single-foil samples, one for tandem samples. The dispatch is based on the existing `tandem_mask` flag already passed through every block.

**The hypothesis:** Single-foil and tandem-foil samples require fundamentally different nonlinear feature transformations at the final representation layer. The current shared FFN must generalize across these two physically distinct regimes. Giving tandem samples a dedicated FFN at the last block is the minimal change that could break this sharing without the destabilizing risk of conditioning all blocks (which failed in `adaln_all_blocks`).

**Why this is different from existing approaches:**
- `soft_moe` flag: soft gate on OUTPUT HEADS (post-attention, last-layer), not on FFN. Affects what is decoded, not what is processed in the backbone.
- `domain_velhead`: hard gate on OUTPUT HEADS for velocity specifically, not on FFN computation.
- `adaln_all_blocks`: modulates LayerNorm scale/shift (affine), not the FFN weights themselves. AdaLN with all-blocks was negative.
- `domain_layernorm`: per-block conditional LN (affine), not FFN.
- This is the first experiment that gives tandem samples a DEDICATED WEIGHT MATRIX for the core nonlinear transformation.

**Key distinction:** The FFN is where the model assembles its final representation before the output head. The attention mechanism produces tokens; the FFN transforms them. If tandem physics require different feature manifolds, the shared FFN is the bottleneck — not the attention routing and not the output head.

**Implementation sketch:**

In `TransolverBlock.__init__`, add a new flag `moe_ffn=False` and when enabled:
```python
if moe_ffn:
    self.mlp_tandem = MLP(hidden_dim, hidden_dim * mlp_ratio * 4, hidden_dim, n_layers=0, res=False)
```
The existing `self.mlp` becomes the single-foil expert.

In `TransolverBlock.forward`, replace:
```python
fx = _ln(self.ln_2_post, self.mlp(_ln(self.ln_2, fx)) + fx)
```
with:
```python
if self.moe_ffn:
    fx_ln = _ln(self.ln_2, fx)
    is_tan = (tandem_mask.view(-1) > 0.5)  # [B] boolean
    out = self.mlp(fx_ln)                   # default: single expert
    if is_tan.any():
        tan_idx = is_tan.nonzero(as_tuple=True)[0]
        out[tan_idx] = self.mlp_tandem(fx_ln[tan_idx])
    fx = _ln(self.ln_2_post, out + fx)
else:
    fx = _ln(self.ln_2_post, self.mlp(_ln(self.ln_2, fx)) + fx)
```

In `Transolver.__init__`, pass `moe_ffn=moe_ffn if (idx == n_layers - 1) else False` in the block construction loop, following the exact same pattern as `soft_moe`, `domain_velhead`, `pressure_first`, etc.

Add `moe_ffn: bool = False` to `Config`.

Add `--moe_ffn` flag to argparse.

**Initialization:** `mlp_tandem` initializes via `initialize_weights()` (orthogonal), same as `mlp`. This is safe — the tandem expert starts from a different random init, not a copy of `mlp`. The training signal will pull it toward tandem-optimal weights from the start.

**Parameter count:** +1 MLP (192→768→192) = ~592K params. Total model ~12M params. VRAM impact: negligible.

**Risk:** Low risk of regression on single-foil (p_in) because the single-foil path is unchanged. The only risk is that tandem expert diverges early — mitigate with standard LR and T_max=150.

**Expected impact:** If the hypothesis is correct, p_tan improvement of 1-3%. If incorrect (tandem and single-foil need the same FFN), the tandem expert will converge to similar weights as the single-foil expert and performance will be flat. No catastrophic failure mode.

**Exact training command addition:** `--moe_ffn --cosine_T_max 150`

## Idea 2: Hydrodynamic Circulation Feature — `circulation-input-feature`

**Target:** p_tan, p_in  
**Risk:** Low  
**Novel:** Yes — not in any prior PR

**What it is:** Compute the approximate circulation Gamma = integral(v . dl) around each foil from the input velocity field, and inject it as a per-foil scalar feature. For a lifting body, circulation is the single most predictive scalar for total lift (Kutta-Joukowski). In tandem configurations, the aft foil operates in the downwash of the fore foil's circulation — making circulation a direct proxy for the fore→aft aerodynamic interaction.

**Why it might help:** The wake deficit feature (#2213, +4.1% p_in) proved that encoding the fore foil's downstream influence as an input feature directly beats learned interactions. Circulation is a higher-order integral of that same influence — it captures the rotational component that wake deficit (axial velocity) misses.

**Implementation:** From input features, extract surface nodes (boundary_id > 0), compute dx/dy along the surface contour, compute line integral of (Ux*dx + Uy*dy) around each foil. This is a precomputed scalar per sample (fore circulation, aft circulation if tandem), broadcast to all nodes of that foil as a new feature dimension.

**Connection to prior wins:** Same lineage as TE coordinate frame and wake deficit feature — physics-grounded input enrichment. No architectural change. Implementable as a preprocessing step or inside the model's feature assembly.

**Risk:** If input velocity already encodes circulation implicitly, this is redundant. But explicit encoding as a global scalar has lower variance than implicit extraction from local features.

## Idea 3: Tandem-Specific Slice Temperature Annealing — `tandem-slice-temp`

**Target:** p_tan  
**Risk:** Low  
**Novel:** Yes

**What it is:** Apply a lower temperature (sharper routing) for tandem samples throughout training, not just via the existing `temp_anneal_epoch` global schedule. The hypothesis is that tandem samples need harder/more focused slice assignments to capture the physically distinct fore/aft wake regions, while single-foil samples benefit from softer routing for generalization.

**Mechanism:** In `Physics_Attention_Irregular_Mesh.forward`, the temperature `T` is already computed per-block from `zone_temp`. Extend this to use `is_tandem` as a binary modifier: `T_tandem = T * alpha` where `alpha < 1.0` (e.g., alpha=0.7) sharpens routing for tandem only. Single-foil uses T unchanged.

**Why:** Round 26 confirmed slice routing itself is not broken — but it may be suboptimal for tandem without failing catastrophically. A lower temperature for tandem forces more specialized slices per foil, potentially improving the fore/aft separation in representation space.

**Risk:** Very low. Single-foil path is unchanged. Alpha=0.7 is a mild perturbation. Try alpha=0.7 first.

## Idea 4: Fore-Foil Global Context Token — `fore-foil-context-token`

**Target:** p_tan  
**Risk:** Medium  
**Novel:** Yes — related to but distinct from cross-attention SRF ideas that failed

**What it is:** Compute a global summary token of the fore-foil hidden states (mean pooled over boundary_id=6 nodes) and prepend it to the aft-foil nodes' attention context in the last TransolverBlock. Unlike the failed fore→aft cross-attention in AftSRF (#2219), this operates at the backbone level and does not add a new head — it modifies how the last attention block sees the aft-foil context.

**Distinction from #2219:** AftSRF cross-attn was an ADDITIVE HEAD on top of the backbone's output. This modifies the backbone's LAST BLOCK INPUT by injecting a summary of fore-foil state before the attention computes. The representation has not yet been decoded — there is still a chance for the model to use this context for all output channels, not just pressure.

**Implementation:** Before the last block forward, compute `fore_ctx = hidden[boundary_id==6].mean(dim=0)` and concatenate it as an extra position to the sequence for tandem samples only. The last block attends to it via the existing slice attention mechanism. Remove it from the sequence after the block's attention.

**Risk:** Medium — modifying the sequence length in the last block requires careful masking. The fore context token does not have a spatial position, so the spatial bias must be handled (zero or learned position).

## Idea 5: Stochastic Depth for Tandem Blocks — `stochastic-depth-tandem`

**Target:** p_tan, p_oodc (generalization)  
**Risk:** Low-Medium  
**Novel:** Yes

**What it is:** Apply stochastic depth (layer drop) independently to each of the 3 TransolverBlocks, but with a HIGHER drop probability for non-last blocks on tandem samples only. This forces the last block to develop robust tandem representations independently, preventing over-reliance on shallow-block features that may be dominated by single-foil patterns (given 80%+ single-foil samples in training).

**Why:** Stochastic depth (Huang et al., 2016) was originally motivated by the observation that many layers in deep nets are redundant — the model should learn to route around them. In a mixed single/tandem training regime, the early blocks are primarily optimized by the majority class (single). Stochastic depth with higher rates on tandem for early blocks forces the last block to be the primary tandem processor.

**Implementation:** During forward, for each block `i < n_layers-1`, on tandem samples with probability `p_drop_tandem` (e.g., 0.3), skip the block (`fx = fx` instead of block forward). Always use block on single-foil. During eval, no drop.

**Risk:** Could hurt single-foil if gradients mix. Ensure tandem masking is clean.

## Idea 6: Hypernetwork Re/Gap/Stagger Continuous Conditioning — `hypernetwork-re-scaling`

**Target:** p_re, p_oodc, p_tan  
**Risk:** Medium  
**Novel:** Partially — precursor to this was `adaln_4cond` and `film_cond`, both neutral. This goes further.

**What it is:** A small hypernetwork (2-layer MLP with 32 hidden units) that takes (Re, gap, stagger) as input and outputs per-block scale/shift for every LayerNorm in the model. Unlike the existing DomainLayerNorm (binary single vs tandem) or AdaLN (fixed cond dim), the hypernetwork produces CONTINUOUS conditioning that varies smoothly across the physics parameter space.

**Why it might help here:** The p_re regression in PR #2251 (p_re=6.364 vs #2213 baseline of 6.300) is a structural tension — lower final LR benefits tandem but hurts Re generalization. A hypernetwork conditioned on Re would allow the model to scale its internal representations proportionally to Reynolds number, which is a physically meaningful operation (boundary layer thickness scales as Re^-0.5, pressure coefficient scales differently across regimes).

**Connection to literature:** Neural Process / conditional neural field approaches use hypernetworks for continuous function-space conditioning. In fluid mechanics, Re is the primary non-dimensional parameter — conditioning on it continuously is strictly more expressive than the binary `domain_layernorm`.

**Why different from prior AdaLN attempts:** `adaln_all_blocks` (all blocks, fixed 2-dim cond) was negative. `adaln_4cond` (4 cond dims) was neutral. This hypernetwork: (a) generates modulations from a learned nonlinear map rather than a linear projection, (b) uses (Re, gap, stagger) as three physically meaningful scalars rather than learned 2-dim features, (c) applies ONLY to the last block and the output head, not all blocks.

**Implementation:** `hyper_net = MLP(3, 64, n_hidden * 2)` → outputs gamma/beta for final-block LN. Condition: `torch.stack([re, gap, stagger], dim=-1)`. Zero-init output layer.

**Risk:** Medium. Careful about p_in regression. Start with last-block only.

## Idea 7: Tandem Geometry Interpolation Augmentation — `tandem-geom-interpolation`

**Target:** p_tan  
**Risk:** Low-Medium  
**Novel:** Yes — PR #2255 (aug annealing) is in-flight but this is a different augmentation strategy

**What it is:** Physics-preserving interpolation between tandem training samples to synthesize new gap/stagger configurations not in the training distribution. Given two tandem samples with gap values g1, g2 and stagger values s1, s2, generate a synthetic sample at (alpha*g1 + (1-alpha)*g2, alpha*s1 + (1-alpha)*s2) where alpha ~ Uniform(0.3, 0.7). The target fields are linearly interpolated with the same alpha.

**Why:** The tandem training set is sparse in (gap, stagger) space. The model must extrapolate to OOD tandem configurations at test time. Linear interpolation of the target fields is approximately valid when the flow is in a regime where the physics vary smoothly with gap/stagger (i.e., no sharp bifurcations like vortex shedding lock-in). This is the domain where our worst tandem errors occur.

**Connection to Mixup:** This is structured Mixup applied to tandem samples only, constrained to be physically meaningful by mixing on the geometry axes that define tandem configuration space. Standard Mixup (Mixup baseline tried, failed) mixed ACROSS samples indiscriminately — this mixes only samples that differ in one physically interpretable dimension.

**Implementation:** In the training loop, after batch assembly, with probability 0.3 for tandem batches, sample two tandem items, interpolate with random alpha, use as an additional synthetic sample in the batch.

**Risk:** Low. Linear interpolation of pressure fields is approximately valid for smooth Re regimes. The risk is that non-smooth regimes (extreme AoA, very small gaps) produce unphysical interpolated targets — mitigate by restricting alpha to [0.3, 0.7] to stay near the center of the interpolation space.

## Idea 8: Boundary-Layer-Aware Node Reweighting — `boundary-layer-reweight`

**Target:** p_tan, p_in  
**Risk:** Low  
**Novel:** Partially — `boundary_aware` flag exists in Config but appears to target volume nodes; this is surface-node-specific

**What it is:** Dynamically reweight the surface loss by estimated boundary layer proximity. Nodes on the suction side near the leading edge and in the wake attachment region have the highest prediction error and the highest physical sensitivity. Assign higher loss weight to nodes with high local curvature AND in the high-Cp region (suction peak), identified from the input pressure field or from the sdf gradient direction.

**Why:** The asymmetric surface loss (#2264, in-flight) applies a fixed 1.5x weight to all suction-side nodes. This is a blunt instrument — not all suction-side nodes are equally important. The suction peak (leading-edge separation bubble onset) is the highest-error region for both p_in and p_tan. Curvature-weighted reweighting concentrates gradient signal precisely where prediction error is largest.

**Distinction from #2264:** Frieren's asymmetric loss uses a global suction-side flag. This uses local curvature magnitude (already in the feature set as index 24) as a continuous weight — no new feature needed.

**Implementation:** `w_i = 1 + gamma * |kappa_i|` where kappa_i is the curvature at surface node i (from input feature index 24, already available). Apply as a per-node loss multiplier in `compute_surface_loss`. gamma=2.0 is a good starting point.

## Idea 9: Wake Pressure Gradient Input Feature — `wake-pressure-gradient`

**Target:** p_tan  
**Risk:** Low  
**Novel:** Partially — pressure gradient AUX HEAD (#2267, in-flight) is complementary; this is an INPUT feature

**What it is:** Compute the numerical pressure GRADIENT dp/dx at each mesh node from the input pressure field (which is included as a reference/freestream value in some feature channels), and inject it as two new input features (dp/dx, dp/dy). The pressure gradient is the primary quantity determining velocity (via momentum equation), so providing it explicitly may help the model learn the p→v mapping.

**Distinction from #2267:** Thorfinn's PR #2267 predicts dp/dx, dp/dy as auxiliary OUTPUT targets (supervision signal). This idea provides dp/dx, dp/dy as INPUT features (direct physics information injection). These are complementary hypotheses about the same quantity — one teaches the model to predict gradients, the other gives the model the gradient directly.

**Implementation:** Numerical gradient from finite differences on the unstructured mesh (using KNN connectivity, already available from AftFoilRefinementContextHead's logic). Add 2 new input dimensions to x, update fun_dim accordingly.

**Risk:** Low. If the input pressure is already sufficient, this will be redundant (zero gradient response will be learned). The added dimensions add negligible VRAM cost.

## Idea 10: Spectral Mixing Loss — `spectral-mixing-loss`

**Target:** p_tan, p_oodc  
**Risk:** Medium  
**Novel:** Yes — DCT frequency loss (#2184, -1.5% p_re) succeeded; this extends it to 2D spectral mixing

**What it is:** After computing the spatial MAE loss, additionally compute a Fourier-space consistency loss on the surface pressure distribution. For each foil, project the surface pressure values onto a 1D DFT along arc-length and penalize high-frequency errors more heavily than the current flat spatial loss. This is a spectral re-weighting of the surface loss.

**Why:** The current DCT frequency loss (#2184) applied to volume fields gave p_re improvement. The surface pressure distribution along arc length is a quasi-1D signal well-suited to Fourier analysis. The suction peak is a sharp, high-frequency feature; the pressure recovery is a smooth, low-frequency feature. Standard MAE weight both equally. Spectral mixing upweights the sharp feature errors that matter most physically.

**Connection to prior work:** DCT loss (#2184) operated on the full volume field. This operates on the surface-only arc-length profile — much smaller computation, and directly targets the surface MAE metric rather than the volume field.

## Idea 11: Persistent Memory Tokens — `persistent-memory-tokens`

**Target:** p_tan, p_oodc  
**Risk:** Medium  
**Novel:** Yes — not tried

**What it is:** Prepend a small set of learned "memory" tokens (e.g., 8 tokens, each of size n_hidden=192) to the sequence before the last TransolverBlock's attention. These tokens are global, trainable parameters (not input-dependent), shared across all samples. They act as a learned "working memory" for the attention mechanism — they can store cross-sample statistics that help with OOD generalization.

**Connection to literature:** Persistent memory tokens were introduced in "All-MLP Mixer" variants and later in "Memory Transformers" (Burtsev et al., 2020). The key insight: global persistent memory tokens allow the model to store task-relevant information that doesn't fit in any individual token's context window. For tandem aerodynamics, these tokens can learn to represent "typical tandem configuration archetypes" without being explicitly supervised to do so.

**Why for p_oodc specifically:** OOD generalization suffers when individual tokens carry insufficient context to recognize the distribution shift. Global memory tokens act as distribution-level anchors — during attention, each node queries not just its spatial neighbors but also the global memory, which has been trained to represent the full distribution of seen configurations.

**Implementation:** Add `self.memory_tokens = nn.Parameter(torch.randn(n_mem, n_hidden) * 0.02)` to the last TransolverBlock. In forward, concatenate memory tokens to fx before attention, remove them after. Memory tokens receive no spatial bias (zero or learned scalar offset). Gradient flows through them normally.

**Risk:** Medium. Memory tokens add parameters and can overfit on small tandem datasets. Regularize with dropout on memory token positions.

## Idea 12: Confidence-Weighted Gradient Surgery — `conf-weighted-pcgrad`

**Target:** p_re, p_oodc  
**Risk:** Medium  
**Novel:** Yes — extends current PCGrad 3-way

**What it is:** The current PCGrad projects conflicting gradients between Groups A (single), B (tandem-normal-Re), C (tandem-extreme-Re). Extend this by weighting each group's gradient contribution by inverse prediction uncertainty — groups where the model is more uncertain (higher per-sample loss variance) contribute more gradient. This is a dynamic form of the currently static PCGrad balance.

**Why for p_re:** The p_re regression (#2251) reflects structural tension: lower final LR helps tandem but hurts Re generalization. If Re-extreme samples (Group C) have higher uncertainty than normal-Re tandem samples, confidence weighting will automatically upweight Re gradient contributions, restoring p_re without hurting p_tan.

**Implementation:** At each step, compute running variance of per-sample surface loss within each PCGrad group. Invert and normalize as group weights. Apply weights to the gradient magnitudes before conflict resolution. EMA smoothing with decay=0.99 to avoid per-step noise.
