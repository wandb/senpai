## Data Generation / Augmentation Ideas

### Idea 1: PIDA Chord-Scaling Augmentation (P0)

**What it is:** Physics-consistent data augmentation by scaling airfoil chord length. Reynolds number scales linearly with chord (Re = V·c/ν at fixed V and ν), so tripling chord at fixed velocity triples Re. This means a single CFD solution can be re-used at a different Re by relabelling the operating conditions — no new simulation needed. The velocity field is dimensionless (in terms of freestream), pressure coefficients are also Re-independent in the inviscid limit, but viscous effects scale correctly with the new Re.

**Why it might help:** p_re is the current hardest metric (7.232, regressed from ANP). The Re OOD validation uses Reynolds numbers far from the training distribution. PIDA augmentation directly creates synthetic training examples at extreme Re values by re-labelling existing solutions, making the model see a broader Re distribution without any new CFD runs. This is fundamentally different from FFD augmentation (#2365 failed due to label mismatch) — chord scaling preserves the CFD labels exactly via Reynolds similarity.

**Mechanism:** For each batch sample, with probability p_aug=0.5, randomly sample a scale factor s ~ Uniform(0.5, 2.0). Multiply Re by s (the equivalent of chord scaling). Leave Ux, Uy, p unchanged — they are already dimensionless. Adjust any Re-dependent features (log_re embedding, panel Cp if Re-dependent) accordingly. The key insight: a flow at Re=1e6 on a chord-1 airfoil is physically identical to Re=2e6 on a chord-2 airfoil at the same Cp distribution.

**Implementation:**
- Add `--pida_chord_aug` flag, `--pida_aug_prob 0.5`, `--pida_scale_range 0.5 2.0`
- In the data collation step, resample Re = Re * s for augmented samples
- Do NOT change velocity/pressure labels (they are already in coefficient form)
- Apply the same scale to all Re-dependent input features

**Suggested command suffix:** `--pida_chord_aug --pida_aug_prob 0.5`

**Confidence:** Strong — Reynolds similarity is exact physics for incompressible flow in the inviscid limit. Modest viscous correction, but should dramatically improve Re OOD distribution.

### Idea 2: Exact Flip-Symmetry Architectural Augmentation (P0)

**What it is:** At zero angle of attack, an airfoil's pressure distribution is exactly symmetric top-to-bottom. NeuralFoil (Drela group, MIT) uses this: run the neural network on both the original and the vertically-flipped geometry, then merge predictions with physically-correct sign conventions. For Ux: average. For Uy: negate and average. For p: average. This is an architectural augmentation — NOT data duplication — so it costs one extra forward pass per sample but provides exact symmetry enforcement at α=0 and strong regularization at all other angles.

**Why it might help:** Enforcing exact physical symmetry acts as a strong inductive bias that reduces the effective hypothesis space for the model. This should help generalisation (p_oodc and p_re), where the model must extrapolate to unseen conditions. At α=0 this is exact; at small α it provides a strong regularization signal. Cost: one extra forward pass during training (can be done with gradient only on one branch if memory-constrained).

**Mechanism:** 
```
x_flip = flip_geometry(x)  # negate y-coords of all nodes
pred_orig = model(x)
pred_flip = model(x_flip)
# Merge with physics-correct sign conventions:
Ux_merged = 0.5 * (pred_orig.Ux + pred_flip.Ux)
Uy_merged = 0.5 * (pred_orig.Uy - pred_flip.Uy)  # sign flip for y-velocity
p_merged  = 0.5 * (pred_orig.p  + pred_flip.p)
# Apply consistency loss between merged and individual predictions
loss_sym = MSE(pred_orig, flip_back(pred_flip))
```

**Implementation:**
- Add `--flip_sym_aug` flag, `--flip_sym_weight 0.1`
- In training loop: for each batch, create flipped version, compute both predictions, apply consistency loss
- Only add consistency loss term (not forward-pass averaging) during training to keep training cost manageable
- At inference: optionally average both passes for improved accuracy

**Suggested command suffix:** `--flip_sym_aug --flip_sym_weight 0.1`

**Confidence:** Strong — exact physics for incompressible flow; empirically proven in NeuralFoil to improve OOD generalization.

### Idea 3: Synthetic Panel-Method Re-Augmentation (P0)

**What it is:** Use the existing vortex-panel solver (already in the codebase via `--vortex_panel_velocity`) to generate synthetic training data at new Reynolds numbers not present in the TandemFoilSet. The panel method is fast (seconds per case) and produces physically meaningful inviscid pressure distributions. While panel predictions are not ground truth, they can be used as augmented training targets for the inviscid component, with the model learning residuals. Key: this is data GENERATION, not just a feature.

**Why it might help:** The Re OOD split exposes the model to Reynolds numbers far from training. If we generate 10,000+ panel-method solutions at extreme Re values and use them as low-fidelity training signal (or as augmentation targets for the pressure field), the model will see the correct qualitative behavior at OOD Re even without expensive CFD. This directly targets p_re.

**Mechanism:**
1. Sample 10k (geometry, AoA, Re) tuples with Re drawn from the OOD distribution (outside training range)
2. Run vortex-panel solver on each to get inviscid Cp distribution
3. Train with a dual-fidelity loss: high-weight on CFD-labelled data, low-weight (λ=0.1) on panel-labelled data for pressure only
4. The model learns: "at this Re range, pressure looks approximately like this panel solution"

**Implementation:**
- Add `--panel_synth_data` flag, `--panel_synth_weight 0.1`, `--panel_synth_n 10000`
- Pre-generate the synthetic dataset before training (one-time cost)
- Include in dataloader as a low-weight auxiliary task — pressure only, surface nodes only
- Key difference from failed Panel Oracle (#2365): panel oracle tried to replace CFD labels entirely. This uses panel as a SUPPLEMENTARY signal at OOD conditions.

**Suggested command suffix:** `--panel_synth_data --panel_synth_weight 0.05`

**Confidence:** Moderate — novel for this setting; conceptually sound. Risk: panel predictions may be too inaccurate at high AoA (separation) to help. Mitigation: use only at α < 5°.

### Idea 4: AoA Interpolation Augmentation (P1)

**What it is:** Given two CFD solutions at the same geometry and Re but different angles of attack (α1 and α2), physically interpolate to create a synthetic solution at α_interp = α1 + t*(α2-α1) for t ~ Uniform(0,1). For small ΔΑ, flow fields vary nearly linearly with AoA in attached flow. The interpolated field is approximately physically consistent.

**Why it might help:** Increases effective training data density in the AoA dimension with zero new CFD runs. The OOD condition split likely includes AoA extrapolation. By augmenting with interpolated solutions, we train the model to understand the smooth AoA dependence of flow fields, which should improve interpolation generalization (p_oodc).

**Mechanism:**
- Find pairs of solutions (same geometry, same Re, different AoA) in the training set
- For each pair, linearly interpolate all field values (Ux, Uy, p) at every node
- Label the interpolated solution with the interpolated AoA
- Filter to small ΔΑ < 3° to ensure physical validity (linearity assumption)

**Implementation:**
- Add `--aoa_interp_aug` flag, `--aoa_interp_pairs_per_batch 4`
- Pre-compute eligible pairs offline; store as index into dataset
- Sample pairs each epoch, interpolate in __getitem__ with random t

**Suggested command suffix:** `--aoa_interp_aug --aoa_interp_pairs_per_batch 4`

**Confidence:** Moderate — interpolation is physically exact only for linear (potential flow) regime. In separated flow at high AoA, interpolation error increases. Use only for |ΔΑ| < 3°.

## Radical Architecture Ideas

### Idea 5: MARIO-Style Hypernetwork Re Conditioning (P0)

**What it is:** Replace AdaLN (additive layer norm conditioning) with a full hypernetwork that generates layer-wise weight modulations conditioned on Re. Inspired by MARIO (Modulated Aerodynamic Resolution Invariant Operator, 3rd place NeurIPS ML4CFD), the hypernetwork takes the operating conditions (log_Re, AoA, gap/stagger) and outputs additive shift vectors for each transformer layer: η_l = ReLU(W_l · z + b_l) where z = concat(log_Re, AoA, gap, stagger). This is fundamentally different from AdaLN: instead of a single scale+shift on the layer norm, the hypernetwork generates separate modulation vectors per layer with full nonlinear capacity.

**Why it might help:** p_re regressed +16% after ANP because cross-attention aggregates fore-foil representations that are unreliable at OOD Reynolds numbers. AeroDiT (NeurIPS 2024) showed that models WITHOUT explicit Re conditioning degrade at Re > 1.2×10^7. A hypernetwork with dedicated Re-pathway gives the model an explicit mechanism to adapt ALL layers (not just layer norms) to the current Reynolds regime. This is the most direct architectural fix for the p_re regression.

**Mechanism:**
```python
class HypernetworkConditioner(nn.Module):
    def __init__(self, cond_dim=4, hidden=64, n_layers=3, layer_dim=192):
        # cond_dim: log_Re, AoA, gap, stagger
        self.net = nn.Sequential(
            nn.Linear(cond_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, n_layers * layer_dim)  # one shift per layer
        )
    def forward(self, cond):
        shifts = self.net(cond)  # [B, n_layers * layer_dim]
        return shifts.chunk(n_layers, dim=-1)  # list of [B, layer_dim]
```
Each transformer block then receives its layer-specific additive shift, applied after FFN (residual add).

**Implementation:**
- Add `--hypernetwork_re` flag, `--hypernetwork_hidden 64`
- Replace or augment AdaLN with hypernetwork output
- Condition on: log10(Re), sin(AoA), cos(AoA), gap_ratio, stagger_ratio
- Initialize hypernetwork output near zero (zero-init the last linear layer) to preserve pre-trained behavior

**Suggested command suffix:** `--hypernetwork_re --hypernetwork_hidden 64`

**Confidence:** Strong — MARIO won 3rd place at ML4CFD with exactly this mechanism; addresses the known p_re weakness directly.

### Idea 6: AeroDiT-Style Diffusion Head with Explicit Re Conditioning (P0)

**What it is:** Replace the deterministic surface refinement head with a lightweight Diffusion Transformer (DiT) that generates pressure/velocity as a conditional diffusion process. AeroDiT (NeurIPS 2024 workshop) demonstrated RANS flow prediction with diffusion at ~0.1 relative L2. The key difference from our prior diffusion attempt (#2349, pre-ANP): (1) the ANP backbone provides a much richer conditioning signal, and (2) we add EXPLICIT Re conditioning to the DiT via class-free guidance (CFG) on Re bins. Prior attempt lacked both.

**Why it might help:** Diffusion models can represent multimodal uncertainty in the pressure field — critical for tandem configurations where the aft-foil wake interaction creates sharp gradients that deterministic models smooth out. The ANP cross-attention already provides the inter-foil context; the diffusion head provides the generative capacity to sample from the sharp posterior. Explicit Re conditioning (CFG style) directly attacks p_re.

**Mechanism:**
- ANP backbone produces latent z_anp ~ [B, N_surface, d]
- DiT diffusion head: 4-step DDIM with 3 DiT blocks (lightweight)
- Conditioning: concat(z_anp, Re_embed, AoA_embed) as class token
- CFG: 10% training drop of Re_embed, use guidance scale γ=3.0 at inference
- Train diffusion head only (freeze backbone) for first 20 epochs, then joint

**Implementation:**
- Add `--diffusion_head` flag, `--diffusion_steps 4`, `--diffusion_cfg_scale 3.0`
- Use DDIM (deterministic) at inference for reproducible metrics
- DiT blocks: 3 layers, d=128, 4 heads (lightweight to stay within VRAM)
- Loss: simple MSE on noise prediction (not ELBO)

**Suggested command suffix:** `--diffusion_head --diffusion_steps 4 --diffusion_cfg_scale 3.0`

**Confidence:** Moderate-Strong — AeroDiT validates diffusion for RANS; prior attempt (#2349) was pre-ANP and lacked Re conditioning. Retry with both fixes is well-motivated.

### Idea 7: Surface-to-Volume Bipartite Graph Message Passing (GeoMPNN-style) (P1)

**What it is:** Replace the volume decoder with a bipartite graph message passing network (GeoMPNN-style, 4th place NeurIPS ML4CFD competition). The key idea: the surface nodes form a "source graph" and volume nodes form a "target graph". Messages propagate from surface to nearby volume nodes using learned edge weights based on (geometric distance, wall-normal direction, boundary-layer thickness estimate). This provides a physically-motivated inductive bias: surface conditions propagate into the volume following boundary-layer physics.

**Why it might help:** The volume decoder is currently a standard Transolver slice attention. It has no explicit mechanism for encoding how surface conditions influence nearby volume nodes. In real aerodynamics, the velocity deficit and pressure perturbations decay in a known way from the surface (BL theory: exponential decay in wall-normal direction). GeoMPNN won because it hardcoded this structural prior. In our setting, improved volume prediction may indirectly benefit surface MAE via better gradient signal.

**Mechanism:**
1. Build k-NN bipartite graph: surface → volume (k=8 volume neighbors per surface node)
2. Edge features: [distance, wall-normal dot product, log(distance), BL thickness estimate]
3. 2 rounds of message passing: surface → volume, then volume → volume (local)
4. Output: volume field corrections added to Transolver output (residual)

**Implementation:**
- Add `--bipartite_mpnn` flag, `--bipartite_k 8`, `--bipartite_layers 2`
- Pre-compute k-NN graph from mesh topology (can be done in prepare.py preprocessing)
- Edge MLP: 3 layers, hidden=64
- Apply as residual correction on top of field_decoder output

**Suggested command suffix:** `--bipartite_mpnn --bipartite_k 8`

**Confidence:** Moderate — GeoMPNN succeeded in the ML4CFD competition; direct applicability to tandem foil uncertain since we have a more complex multi-body topology.

### Idea 8: AB-UPT Anchor Token Decoupling (P1)

**What it is:** From AB-UPT (Anchored-Branched Universal Physics Transformers, arXiv 2024): introduce a small set of learnable "anchor tokens" (M=16-32) in the transformer latent space that serve as geometry-condition summaries. The anchors attend to the full geometry input, compress it into a compact representation, and then feed into the field decoder. This decouples geometry understanding from field prediction — the anchors learn a reusable geometry vocabulary. In the tandem setting: two sets of anchors (one per foil), with ANP-style cross-attention between them.

**Why it might help:** The ANP cross-attention currently operates at the full surface-node level (N_surface ~ 512 nodes per foil). This is expressive but may overfit to specific training geometries. Anchor tokens provide a compact bottleneck that forces the model to learn abstract geometric features (camber, thickness, leading-edge radius) rather than memorizing surface node patterns. This should improve OOD generalization (p_oodc, p_re).

**Mechanism:**
```python
class AnchorTokenModule(nn.Module):
    def __init__(self, M=16, d=192):
        self.anchors = nn.Parameter(torch.randn(M, d))  # learnable
    def forward(self, geometry_tokens):
        # anchors cross-attend to geometry tokens
        a = self.anchors.unsqueeze(0).expand(B, -1, -1)
        return cross_attention(q=a, kv=geometry_tokens)  # [B, M, d]
```
The M anchor tokens then replace the full surface tokens in the ANP cross-attention.

**Implementation:**
- Add `--anchor_tokens` flag, `--anchor_n 16`
- Insert anchor token module between SRF surface encoder and ANP cross-attention
- Fore-foil anchors (M=16) attend to aft-foil anchors via existing ANP mechanism
- Initialize anchors with PCA of training geometry embeddings (warm start)

**Suggested command suffix:** `--anchor_tokens --anchor_n 16`

**Confidence:** Moderate — AB-UPT validated for single-body aerodynamics; extending to tandem with ANP cross-attention is novel. The bottleneck may help or may lose fine-grained surface information.

### Idea 9: Divergence-Free Velocity Hard Constraint (P1)

**What it is:** For incompressible flow, the velocity field must satisfy ∇·u = 0 everywhere. Enforce this as a hard constraint on the model output by projecting the predicted velocity onto the divergence-free subspace using the Helmholtz decomposition: u_corrected = u_pred - ∇φ where φ solves the Poisson equation ∇²φ = ∇·u_pred. This can be approximated on the mesh using a learned Poisson solver (2-layer GNN). The constraint is exact physics — not a soft loss penalty.

**Why it might help:** The current model predicts Ux and Uy independently with only a soft loss coupling them. In real flow, continuity (∇·u = 0) provides a strong coupling that constrains the solution space dramatically. Enforcing this constraint should improve both the volume field accuracy and — via the pressure-velocity coupling in the NS equations — the surface pressure accuracy.

**Mechanism:**
1. Model predicts u_pred = (Ux, Uy) at all nodes
2. Compute nodal divergence: div_u = Σ (u_pred · n_ij) * |e_ij| (finite volume approximation on mesh)
3. Solve Poisson equation for φ using 3 iterations of a Gauss-Seidel GNN
4. Project: u_corrected = u_pred - ∇φ (gradient computed on mesh)
5. Use u_corrected for loss computation and output

**Implementation:**
- Add `--div_free_constraint` flag, `--div_free_weight 0.1`
- Start with soft version (add divergence term to loss) before hard projection
- GNN Poisson solver: 2 layers, message passing on mesh edge graph
- Apply projection in the output layer of the field decoder

**Suggested command suffix:** `--div_free_constraint --div_free_weight 0.1`

**Confidence:** Moderate — divergence-free constraints are well-motivated physics; implementation on unstructured mesh is non-trivial. Suggest starting with soft penalty version first.

### Idea 10: Flow Matching Generative Head on ANP Baseline (P1)

**What it is:** Replace the deterministic SRF head with a flow matching (Rectified Flow / Conditional Flow Matching) generative model. Flow matching trains a vector field that continuously interpolates between a Gaussian prior and the target distribution in 1-10 steps (much faster than DDPM's 1000 steps). Prior attempt (#2271 pre-ANP) ran without the ANP backbone and without explicit conditioning — retry with ANP context as the conditioning signal.

**Why it might help:** Flow matching can represent the full posterior distribution over flow fields given geometry — critical for tandem configurations where small geometry changes cause large aft-foil flow changes (high uncertainty in the wake). The ANP backbone provides the conditioning context; the flow matching head provides the generative flexibility. Unlike diffusion, flow matching uses straight-line trajectories in function space, which are more efficient to learn and sample.

**Mechanism:**
- Backbone: full ANP model → latent z_anp [B, N_surface, d]
- Flow matching head: learn v(z_t, t) = z_target - z_noise (Rectified Flow)
- At inference: 4-step ODE integration from z~N(0,I) to prediction
- Conditioning: z_anp concatenated to each token at each flow step

**Implementation:**
- Add `--flow_matching_head` flag, `--flow_matching_steps 4`
- Use Rectified Flow (simplest version): v = (x1 - x0) directly
- Head architecture: 3 transformer blocks, d=128 (lightweight)
- Loss: MSE on vector field v_theta(z_t, t) vs target (z1 - z0)
- Train jointly with backbone (no frozen stage)

**Suggested command suffix:** `--flow_matching_head --flow_matching_steps 4`

**Confidence:** Moderate — flow matching improves on diffusion for speed; prior attempt was pre-ANP. ANP conditioning should dramatically improve the conditioning signal.

### Idea 11: Displacement Thickness δ* Boundary Layer Feature (P1)

**What it is:** Compute the integral boundary layer displacement thickness δ*(s) along each airfoil surface as a physics-derived input feature. δ* = ∫₀^∞ (1 - u/U_e) dn, approximated from the vortex-panel velocity field (already computed) using the integral momentum method (Thwaites' method). This gives a surface-node feature representing how thick the viscous layer is at each chordwise station — a direct predictor of viscous pressure losses.

**Why it might help:** The viscous pressure distribution deviates from inviscid panel predictions primarily due to displacement effect: the boundary layer acts like a thickening of the airfoil, shifting the effective geometry. δ* encodes this effect explicitly. At high Re, δ* is thin and viscous effects are small; at low Re, δ* is thick and significant. This directly encodes the Re-dependent correction needed, targeting p_re.

**Mechanism:**
1. Given vortex-panel velocity field U_e(s) along surface (already computed via `--vortex_panel_velocity`)
2. Apply Thwaites' method: θ(s) = (0.45ν/U_e^6) ∫₀^s U_e^5 ds (momentum thickness)
3. Estimate δ* ≈ H · θ where H ≈ 2.6 (shape factor for attached BL)
4. Add δ*(s) as an additional scalar feature on surface nodes

**Implementation:**
- Add `--displacement_thickness_feature` flag
- Compute in the surface feature preprocessing (after vortex panel solve)
- Normalize: log(1 + δ*/chord) to handle large dynamic range across Re
- Apply only to surface nodes (not volume)
- Requires `--vortex_panel_velocity` to already be active

**Suggested command suffix:** `--displacement_thickness_feature` (requires `--vortex_panel_velocity`)

**Confidence:** Moderate — physically motivated; Thwaites' method is only valid for laminar attached BL. May degrade at high AoA (separation). But the feature itself signals "where separation is likely" even if the method breaks down there.

### Idea 12: Boundary-Layer Adaptive Node Weighting (P0)

**What it is:** Reweight the training loss by local boundary-layer membership. Nodes in the boundary layer (wall distance < δ_99 estimate) receive higher loss weight; nodes in the freestream receive lower weight. The estimated δ_99 ~ 5 * δ* (from displacement thickness above, or from a heuristic wall-distance threshold). This is inspired by OB-GNN (2nd place ML4CFD NeurIPS), which used hierarchical sampling: 8× weight on surface nodes, 2× on near-wall nodes, 1× on freestream.

**Why it might help:** The model sees equal gradient signal from all volume nodes, but surface and near-wall accuracy matters much more for engineering quantities (Cp, drag, lift). Reweighting toward the boundary layer concentrates learning capacity on the high-value regions. This is a training efficiency improvement — same model, better use of gradient budget.

**Mechanism:**
- Compute wall distance d_wall for each mesh node (can be precomputed from mesh)
- Estimate BL thickness: δ_est = max(0.1·chord, 5·δ* from panel method)
- Loss weight w_i:
  - Surface nodes: w = 10.0
  - Near-wall (d_wall < δ_est): w = 3.0
  - Wake region: w = 2.0
  - Freestream: w = 1.0

**Implementation:**
- Add `--bl_adaptive_weighting` flag, `--bl_surface_weight 10.0`, `--bl_nearwall_weight 3.0`
- Precompute node distances from mesh in data loading
- Apply as element-wise weight to the loss tensor before reduction
- Compatible with PCGrad 3-way (weight inside each task's loss)

**Suggested command suffix:** `--bl_adaptive_weighting --bl_surface_weight 10.0 --bl_nearwall_weight 3.0`

**Confidence:** Strong — directly validated by OB-GNN (ML4CFD 2nd place); the mechanism is clear and the implementation is low complexity. Best bang-for-buck idea in this round.
