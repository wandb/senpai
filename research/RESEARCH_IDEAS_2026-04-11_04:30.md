## Idea 1: Cp-Space ANP Decoding (HIGHEST PRIORITY — targets p_re regression)

**Hypothesis**: The ANP's p_re regression is caused by the cross-attention mechanism learning attention patterns correlated with absolute pressure magnitude, which scales as Re² (pressure ∝ ρU² ∝ Re²). By transforming the ANP decoder targets into pressure coefficient space (Cp = p / (0.5 ρ U²)) before computing the ANP correction, we force the attention mechanism to learn geometry-conditioned patterns that are independent of Reynolds number. The asymmetric cross-foil attention currently learns "how much pressure from the fore foil?" in absolute terms — which generalizes poorly across Re. In Cp space, the coupling is Re-invariant and should generalize to the OOD Reynolds split.

**Implementation**:
1. Before the ANP decoder computes its correction, normalize the context node pressures to Cp using the per-sample dynamic pressure head (0.5 * rho * U^2), derivable from the flow conditions features already present in x.
2. The ANP context encoder receives Cp values (not raw p).
3. The ANP decoder outputs a Cp correction.
4. Post-decode: multiply the Cp correction by the per-sample dynamic pressure to get back to physical pressure units.
5. The SRF head remains in raw p space (additive residual structure unchanged).
6. Add a new flag `--anp_cp_space` that enables this normalization.

**Key detail**: Dynamic pressure requires rho and U. The flow conditions in TandemFoilSet include Re and chord; U can be inferred from Re = rho*U*c/mu using known fluid properties. Alternatively, use the asinh-normalized pressure scale already in the pipeline as a proxy. Check how the existing asinh pressure normalization interacts — you may need to apply Cp normalization AFTER the asinh transform's inverse, then re-apply asinh after Cp scaling.

**Expected impact**: HIGH — directly targets the physical reason for p_re regression. Cp normalization is standard in aerodynamics for exactly this reason: collapse of Re-dependent pressure onto geometry-dependent coefficient curves.

**Risk**: Medium. The Cp calculation requires accurate dynamic pressure, which depends on knowing freestream velocity. If this is not cleanly available in the feature vector, the normalization may introduce noise. Also, Cp space near stagnation points can be numerically extreme (Cp approaches 1 at stagnation). The asinh transform already in use may need to be applied to Cp as well.

**Suggested command addition**: `--anp_cp_space`

## Idea 2: Reynolds-Conditional ANP Attention (AdaLN in Q/K projections)

**Hypothesis**: The ANP's p_re regression occurs because the query and key projections (Q, K matrices) learn fixed linear transformations that are implicitly calibrated to the training Re distribution. At OOD Reynolds numbers, the attention map (Q^T K / sqrt(d)) produces incorrect similarity scores, causing the decoder to weight context nodes incorrectly. By conditioning the Q and K projections on the Reynolds number via AdaLN (already used in the Transolver backbone), we give the model an explicit mechanism to adjust its attention patterns for different Re — exactly analogous to how AdaLN is used for flow condition conditioning in the backbone.

**Implementation**:
1. In the ANP decoder, add a Re embedding (log(Re) projected to a scalar or small vector) as conditioning signal.
2. Apply AdaLN to the Q and K projections: scale/shift their outputs conditioned on Re.
3. Use the same AdaLN conditioning infrastructure already present in the Transolver backbone — just pass the Re embedding through.
4. Keep V projections unconditioned (or add a separate AdaLN on V for ablation).
5. Total parameter overhead: 2 * attn_dim * 2 (scale + shift) per Re embedding dim — minimal.
6. Add flag `--anp_adaln_re`.

**Expected impact**: HIGH for p_re specifically. This is a principled surgical fix. The AdaLN already works well in the backbone for exactly this kind of conditioning.

**Risk**: Low-medium. The implementation is a small, targeted change. Risk of harming p_in/p_oodc is low since AdaLN is data-driven (at in-distribution Re, the scale/shift converges to near-identity). Main risk: if Re is already encoded adequately in the context features, this may add parameters without benefit.

**Suggested command addition**: `--anp_adaln_re`

## Idea 3: ANP Context Dropout (Neural Process Robustness Training)

**Hypothesis**: Neural Processes are designed to generalize across context sizes and distributions by training with randomly subsampled context sets. The current ANP decoder always sees ALL surface nodes as context — it never has to function with partial context. This is equivalent to training a dropout-free network: the model becomes fragile because it memorizes full-context patterns. At OOD Reynolds numbers, the context node features are shifted (different pressure magnitudes), effectively presenting a corrupted context. Training with random context masking (20–50% of context nodes dropped per forward pass) would force the ANP to aggregate information robustly, improving generalization to distribution-shifted contexts including OOD Re.

**Implementation**:
1. During training only, randomly mask a fraction p_drop (try 0.3) of context nodes before they are passed to the context encoder.
2. At inference, use full context (all surface nodes).
3. This is analogous to DropPath / stochastic depth, but applied to context set membership.
4. Can also be combined with idea 2 (Re-conditional AdaLN).
5. Add flag `--anp_context_dropout 0.3`.

**Expected impact**: Medium-high for p_re, potentially positive for p_oodc as well. This is well-motivated by the Neural Process literature (Garnelo et al. 2018; Kim et al. 2019 ANPI). The technique is used in CNP/ANP training to ensure proper marginalization — we are currently skipping this step.

**Risk**: Low. Dropout variants rarely hurt significantly. The main risk is that with 0.3 dropout, training becomes noisier and may need a slight lr reduction or longer warmup. Start with p_drop=0.2, then try 0.3.

**Suggested command addition**: `--anp_context_dropout 0.3`

## Idea 4: Stochastic Cross-Foil Attention (Aft-Foil Robustness)

**Hypothesis**: The cross-foil attention (aft-foil queries attending to fore-foil context) is the mechanism that drives p_tan improvement but may be hurting p_re. When cross-foil attention is always active, the aft-foil decoder becomes dependent on fore-foil context signals. At OOD Reynolds, the fore-foil signals are also shifted, compounding the OOD error. By randomly disabling cross-foil attention during training (stochastic depth, p_skip=0.3), the aft-foil decoder is forced to sometimes solve the problem with self-attention alone — making it more robust. At inference, always use full cross-foil attention.

**Implementation**:
1. In the ANP decoder's cross-attention block, add a training-time flag that randomly (p_skip=0.3) replaces the aft-foil cross-attention (attend to all nodes) with aft-foil self-attention (attend only to aft-foil nodes).
2. This is a single line addition: `if training and random() < p_skip: use self-attention else: use cross-attention`.
3. Fore-foil path remains unchanged (always self-attention).
4. Single-foil cases are unaffected.
5. Add flag `--anp_stochastic_cross 0.3`.

**Expected impact**: Medium for p_re, low risk to p_tan (cross-attention is still always used at inference). This is the "stochastic depth for attention paths" idea applied to the specific bottleneck.

**Risk**: Low-medium. The p_tan improvement came from cross-foil attention; any regularization of this path risks marginally reducing p_tan gains. However, stochastic depth empirically tends to improve both robustness and accuracy by acting as an ensemble. The key is to keep p_skip ≤ 0.4 so the model still sees plenty of cross-foil training signal.

**Suggested command addition**: `--anp_stochastic_cross 0.3`

## Idea 5: Spectral Normalization of ANP Attention (Lipschitz-Constrained Cross-Attention)

**Hypothesis**: The p_re regression signature of the ANP — excellent in-distribution, worse OOD — is the classic fingerprint of a high-Lipschitz function. The attention map Q^T K can have unbounded spectral norm, causing small input perturbations (OOD Re → slightly different node features) to produce disproportionately different attention weights. Spectral normalization of Q and K projections constrains the Lipschitz constant of the attention map, directly limiting sensitivity to input distribution shift. This is theoretically motivated: spectral normalization is used in GANs and robust networks precisely to prevent this failure mode.

**Implementation**:
1. Apply `torch.nn.utils.spectral_norm` to the Q and K linear layers in the ANP cross-attention.
2. Leave V and output projection unnormalized (or try normalizing V too as an ablation).
3. No new hyperparameters needed — spectral norm is parameter-free.
4. Add flag `--anp_spectral_norm`.
5. Optionally combine with idea 3 (context dropout) for a more aggressive robustness bundle.

**Expected impact**: Medium for p_re. Spectral normalization is well-understood; the mechanism is sound. The downside is it may reduce the sharpness of attention learned for tandem coupling, potentially slightly reducing p_tan gains.

**Risk**: Medium. Spectral norm adds a small per-step cost (power iteration). The main risk is over-constraining the attention, reducing its capacity to model cross-foil coupling accurately. Ablate with V normalization off first.

**Suggested command addition**: `--anp_spectral_norm`

## Idea 6: Multi-Resolution ANP Context (Coarse Statistics + Fine Nodes)

**Hypothesis**: The current ANP uses individual surface nodes as context. This is fine-grained but means the attention mechanism must simultaneously learn (a) which specific surface locations to attend to and (b) global flow patterns from context. Hierarchical context — where coarse aggregate statistics (mean/std of Cp, suction peak location, stagnation point pressure) are provided alongside fine-grained node-level context — would give the ANP a structured global picture before attending to fine details. This is analogous to coarse-to-fine attention in NLP (document-level then token-level). The coarse statistics are less Re-sensitive (geometry-driven), potentially improving p_re.

**Implementation**:
1. Compute 8–16 coarse summary tokens from the surface nodes: segment the airfoil arc into 4–8 zones (LE, upper-fore, upper-aft, TE-upper, TE-lower, lower-aft, lower-fore), average the features in each zone → coarse tokens.
2. Concatenate coarse tokens with fine node-level tokens as context: context = [zone_tokens, node_tokens].
3. Queries attend to this extended context set normally.
4. Coarse tokens: 8 tokens × attn_dim=192 → negligible parameter overhead.
5. Add flag `--anp_hierarchical_context 8` (controls number of zones).

**Expected impact**: Medium-high. Structured context is a well-established improvement in attention-based architectures. The coarse statistics are inherently more Re-robust (geometry drives Cp distribution shape; Re drives magnitude). This addresses both p_re and potentially p_tan.

**Risk**: Medium. The zone segmentation requires arc-length parameterization of the surface nodes, which needs to be implemented correctly. If the zone boundaries don't align well with aerodynamic features (suction peak, transition), the coarse tokens may not be meaningful. Pre-defined aerodynamic zones (rather than uniform arc-length segments) would be better but more complex.

**Suggested command addition**: `--anp_hierarchical_context 8`

## Idea 7: Tandem Wake Superposition Prior for ANP (Physics-Initialized Decoder)

**Hypothesis**: The ANP decoder currently starts from a zero prior and learns the full pressure field from scratch via attention. For tandem configurations, aerodynamic theory gives us a strong prior: aft-foil pressure ≈ isolated-foil pressure + interference correction from fore-foil wake. The panel-method Cp features (already merged, `--cp_panel --cp_panel_tandem_only`) encode some of this, but they are pre-computed and fixed. A stronger idea: initialize the ANP decoder's output (before residual addition) as the sum of the fore-foil Cp (from panel method) scaled and shifted by a learned function of geometry — giving the model a physics-based cross-foil coupling prior that is inherently Re-normalized (Cp space). The ANP then predicts the viscous/nonlinear correction on top of this prior.

**Implementation**:
1. In the ANP decoder output layer, add a side-path that computes: `prior = MLP(cp_panel_features) * dynamic_pressure` where cp_panel_features is the panel-method Cp from the fore foil (already in the feature vector).
2. The ANP attention output is added to this prior: `final = prior + anp_attention_output`.
3. This is a 2-layer MLP (cp_panel_dim → 64 → 1) for each of Ux, Uy, p — trivial parameter count.
4. The prior is in physical units after the dynamic pressure multiplication.
5. Add flag `--anp_physics_prior`.

**Expected impact**: HIGH for p_tan (physically motivated cross-foil coupling prior), MEDIUM for p_re (Cp normalization in the prior path is Re-invariant). This is a principled physics-ML hybrid.

**Risk**: Low-medium. The panel-method Cp features are already in the model — this just routes them more explicitly into the decoder. Main risk: if the panel-method prior is poor (which it sometimes is for high-AoA or thick-boundary-layer cases), the model may be hurt by the prior. A small learnable gate on the prior weight would mitigate this.

**Suggested command addition**: `--anp_physics_prior`

## Idea 8: Reynolds Mixup in Cp Space (Data Augmentation for p_re)

**Hypothesis**: The p_re split (OOD Reynolds) fails because the model sees only 5 discrete Reynolds numbers during training and has to extrapolate to held-out Re. Standard Mixup in pressure space creates samples between existing Re values, but the pressure scaling is quadratic in Re, making naive interpolation incorrect. Mixing in Cp space (Re-normalized pressure) creates valid intermediate Cp fields that are geometrically consistent — then converting back to pressure with an intermediate Re gives a synthetic sample at a new Reynolds number that the model has never seen. This directly targets the extrapolation gap in p_re.

**Implementation**:
1. For each training batch, with probability 0.3, sample two examples with similar geometry but different Re values (within the same configuration type — single or tandem).
2. Compute Cp for each: Cp = p / (0.5 * rho * U^2).
3. Mix Cp fields: Cp_mix = lambda * Cp_1 + (1-lambda) * Cp_2, lambda ~ Beta(0.4, 0.4).
4. Choose Re_mix = lambda * Re_1 + (1-lambda) * Re_2.
5. Convert back to pressure: p_mix = Cp_mix * (0.5 * rho * U_mix^2).
6. Use (geometry from sample 1, Re_mix, p_mix) as synthetic training sample.
7. Velocity fields: mix directly (velocity is linear in U → mix in normalized units, scale by U_mix).
8. Add flag `--re_mixup --re_mixup_alpha 0.4`.
9. Implementation in `data/prepare.py` collation step (or as an augmentation wrapper in train.py).

**Expected impact**: HIGH for p_re specifically. Mixup in Cp space is physically motivated — this is essentially creating synthetic intermediate Reynolds number training samples. This directly closes the distribution gap.

**Risk**: Medium. The key assumption is that Cp fields mix linearly, which is only approximately true (real aerodynamics has nonlinear Re effects like transition location, separation bubble size). The mixing should be applied conservatively (lambda near 0.5, not extreme values) and limited to pairs with geometrically similar configs. Boundary condition consistency (geometry) needs care — only mix same-geometry-class pairs.

**Suggested command addition**: `--re_mixup --re_mixup_alpha 0.4`

## Idea 9: Boundary Layer Thickness Feature (Physics Feature for p_re)

**Hypothesis**: Reynolds number affects pressure primarily through its effect on boundary layer thickness and transition location. The current model receives Re as a scalar conditioning input, but the model must learn internally how Re maps to pressure changes. A more powerful encoding: add a pre-computed boundary layer thickness estimate (delta ~ c * Re^(-0.5) for laminar, delta ~ c * Re^(-0.2) for turbulent) and displacement thickness (delta* ~ 1.72c/sqrt(Re)) as node-level features, distributed along the surface arc length. These physics-derived features directly encode the Re-dependent pressure correction mechanism, giving the model interpretable input that linearizes the Re-pressure relationship.

**Implementation**:
1. For each surface node at arc-length s, compute: `delta(s) = 1.72 * sqrt(nu * s / U)` (Blasius laminar estimate).
2. Also compute shape factor H = delta*/theta (Thwaites approximation) and displacement thickness delta*.
3. Add these 3 scalar features per surface node to the node feature vector (currently 24-dim → 27-dim).
4. For tandem configs, compute separately for fore- and aft-foil using their respective local s coordinates.
5. This is purely a feature engineering addition — no architectural changes needed.
6. Add flag `--bl_thickness_feature`.

**Expected impact**: MEDIUM for p_re — provides the model with the physical quantity that mediates the Re-pressure relationship. The boundary layer directly causes pressure deviations from inviscid theory. Medium for general surface pressure accuracy.

**Risk**: Low. Feature addition is low risk. Main concern: Blasius estimate assumes flat plate boundary layer, which is a rough approximation for airfoil surfaces with pressure gradients. The feature is still useful as a Re-dependent signal even if imprecise. The model will learn to weight it appropriately.

**Suggested command addition**: `--bl_thickness_feature`

## Idea 10: Separated ANP Heads per Flow Regime (Single-Foil vs. Tandem)

**Hypothesis**: The current ANP uses a single set of attention parameters for both single-foil and tandem configurations. Single-foil cases have no cross-foil coupling (fore-foil attention is self-attention only, aft-foil is self-attention as a degenerate case). The network must simultaneously be a "simple self-attention surface smoother" for single-foil and a "cross-foil interference detector" for tandem. These are geometrically and physically different problems. Specialized lightweight ANP heads — a small single-foil ANP (self-attention only, fewer params) and a tandem ANP (full cross-attention, more params) — would allow each head to be optimized for its task without compromise.

**Implementation**:
1. Keep the existing full ANP (`--anp_srf`) as the tandem head (used when `n_foils == 2`).
2. Add a smaller single-foil ANP head: self-attention only, attn_dim=96 instead of 192, 2 heads instead of 4. This is a ~4× parameter reduction for the single-foil path.
3. Route forward pass: `if is_single_foil: use sf_anp else: use full_anp`.
4. The single-foil ANP processes only the single foil's surface nodes. This is exactly analogous to how the current ANP reduces to self-attention for single-foil — but now with dedicated parameters and smaller capacity.
5. This allows the tandem ANP to specialize on cross-foil coupling while the single-foil ANP specializes on Re generalization without interference from the tandem task.
6. Add flag `--anp_split_heads`.
7. Expected parameter count: current ANP ~262K; split heads: ~196K (tandem) + ~49K (single-foil) = ~245K — actually fewer total params.

**Expected impact**: MEDIUM for p_re (single-foil ANP no longer "polluted" by tandem cross-attention gradients), MEDIUM for p_tan (tandem ANP can specialize without needing to also work for single-foil). This is a classic multi-task learning specialization idea.

**Risk**: Low. The routing logic is simple (single vs. tandem is a known property of each sample). The main risk is reduced parameter sharing loses beneficial regularization from multi-task learning. The split head idea should be tested after ideas 1-4 to confirm it adds value beyond the Re-conditional approaches.

**Suggested command addition**: `--anp_split_heads`
