### Idea 1: Panel Method as Cheap Data Oracle (Multi-Fidelity Data Generation)

**Hypothesis:** The tandem configuration space is severely undersampled in the training set. Using an inviscid panel solver (XFOIL or a custom panel implementation already partially present in the codebase via `--cp_panel`) to generate 5000+ synthetic tandem configurations at negligible compute cost, then training with these as low-fidelity auxiliary data alongside the high-fidelity CFD dataset, will reduce p_tan by 10–20% by improving coverage of the tandem interference region.

**Why bold:** This is not a model change — it is a data generation strategy that changes the effective training distribution. The panel method is already used as an input feature (`--cp_panel`); using it as a data oracle to generate labels is a qualitative leap. Unlike FFD augmentation (PR #2365, which failed due to label mismatch), here the labels come from the panel solver itself and are self-consistent. The AIAA 2024 multi-fidelity aerodynamics paper showed that 6M+ XFOIL datapoints alongside 575K high-fidelity samples improved OOD generalization substantially. The Springer Nature 2025 inviscid transfer learning paper showed 10² MSE improvement with only 2 high-fidelity samples when pre-training on inviscid data.

**Expected target metric:** p_tan primary (tandem coverage), p_oodc secondary. Conservative estimate: -10% p_tan, -5% p_oodc.

**Implementation sketch:**
1. Use the existing panel code path (`cp_panel` logic in `train.py`) to compute inviscid surface Cp for arbitrary tandem configurations.
2. Generate a grid of (chord ratio, gap, stagger, AoA_1, AoA_2, Re) configurations — ~5000 samples — covering the tandem parameter space more densely than the CFD dataset.
3. For each configuration, compute panel Cp as the label (not the CFD pressure). Train with these as auxiliary data at reduced weight (0.1× loss scaling) alongside the primary CFD dataset. The inviscid Cp is systematically wrong near the TE and in the viscous wake, but is structurally correct for the pressure distribution shape.
4. Alternatively: use the panel Cp as a "prior prediction" and train the model to predict the CFD-panel residual — aligning directly with the existing `--residual_prediction` flag logic.
5. Key hyperparameter: auxiliary data weight (try 0.1, 0.3, 1.0). The model must not overfit to inviscid labels.

**References:**
- "Multi-fidelity aerodynamic surrogate with XFOIL + CFD," AIAA 2024 — 6M XFOIL datapoints alongside 575K CFD samples; confirms scale of benefit from inviscid data augmentation.
- "Inductive transfer learning from inviscid to viscous aerodynamics," Springer Nature 2025 — 10² MSE improvement from inviscid pre-training; only 2 high-fidelity samples needed after pre-training.
- Existing codebase: `--cp_panel`, `--cp_panel_tandem_only`, `--cp_panel_scale` flags — panel infrastructure already present.

### Idea 2: Mamba/SSM Surface Sequence Model (Replace SRF Head)

**Hypothesis:** The surface refinement head (SRF) uses attention over arc-length-ordered surface nodes. Replacing this with a selective state-space model (Mamba/S4) operating over the same arc-length sequence will capture long-range pressure dependencies more efficiently at O(n) cost, allowing a deeper (6–8 layer) surface decoder within the same VRAM budget, reducing p_in and p_tan by 5–10%.

**Why bold:** Attention over surface nodes has a fundamental O(n²) cost that limits the depth of the SRF head. Mamba processes sequences in O(n) time and has demonstrated state-of-the-art performance on PDE tasks (Mamba Neural Operator, ScienceDirect 2025). The arc-length ordering of surface nodes is a natural sequence — the pressure field has genuine 1D structure along the surface (leading edge → upper surface → trailing edge → lower surface) that an SSM can exploit via its recurrent inductive bias. This is architecturally distinct from the 1D FNO decoder in PR #2371 (frequency domain) — Mamba operates in the time/sequence domain with selective gating.

**Expected target metric:** p_in and p_tan. Surface pressure prediction has the most structure as a 1D signal; SSM should capture boundary layer transition locations and trailing-edge separation better than attention.

**Implementation sketch:**
1. Install `mamba-ssm` (pip install mamba-ssm; requires CUDA 11.6+; 96GB A100 GPUs should support).
2. Replace the SRF MLP/attention block with a Mamba layer stack: `MambaBlock(d_model=192, d_state=16, d_conv=4, expand=2)` × 4 layers.
3. Input: surface node features from the main Transolver backbone, ordered by arc-length (already computed for `--surface_refine`). Append panel Cp as an additional channel.
4. Output: delta pressure prediction added to the backbone surface predictions (residual formulation, consistent with `--residual_prediction`).
5. If `mamba-ssm` has install issues, fall back to S4/S5 (available via `state-spaces` package) or a simple linear recurrence (LRU).
6. Critical: arc-length ordering must be consistent across foils. Use per-foil sequences (fore foil arc-length sequence, aft foil arc-length sequence) processed independently, then concatenated before the output projection.

**References:**
- "Mamba Neural Operator," ScienceDirect 2025 — Mamba outperforms Transformers on PDE benchmarks; selective state spaces capture multi-scale dynamics.
- "Mamba: Linear-Time Sequence Modeling with Selective State Spaces," Gu & Dao 2023 (arXiv:2312.00752) — original Mamba paper.
- "S4: Efficiently Modeling Long Sequences with Structured State Spaces," Gu et al. 2022 — predecessor; well-tested on scientific time series.

### Idea 3: Neural Process for Pressure Fields (Meta-Learning Context Encoder)

**Hypothesis:** Training the model as an Attentive Neural Process (ANP) — where context points are randomly sampled surface nodes with their panel Cp values, and target points are the remaining surface nodes — will give the model an explicit mechanism to condition on partial observations and generalize to new configurations by design. This targets p_oodc (OOD conditioning) directly, with expected -10–15% on p_oodc.

**Why bold:** Neural Processes are designed precisely for the problem of amortized inference over function spaces — exactly what a surrogate model does. No existing work (as of 2025) applies NPs to CFD pressure field prediction. The key novelty is using panel Cp as the context signal: at test time, the model sees `{surface_coord_i, cp_panel_i}` for all surface nodes as context and predicts `p_CFD_i` at each node as target. This is a complete reformulation of the inference problem — instead of a feedforward mapping from geometry features to pressure, it is a conditional distribution over pressure fields given observed (inviscid) pressures. The model learns to "correct" the panel prediction in a distribution-aware way.

**Expected target metric:** p_oodc primary (-10–15%), p_tan secondary. OOD generalization is the explicit design goal of Neural Processes.

**Implementation sketch:**
1. ANP architecture: (a) Deterministic path: cross-attention encoder from context to target node positions; (b) Latent path: set encoder (DeepSets) over context → latent z → sampled at target points.
2. Context points: all surface nodes with features `[x, y, arc_length, cp_panel, foil_role]`. Target points: same surface nodes (self-prediction task), with the model predicting a Gaussian over CFD pressure `N(mu_i, sigma_i)`.
3. Training: NLL loss on Gaussian predictions + KL divergence for latent path. At inference: take mean prediction (no sampling).
4. Integration: keep the Transolver backbone as the feature extractor; replace only the SRF head with the ANP decoder. The backbone hidden states serve as the "global context encoding" feeding into the ANP's cross-attention.
5. Implementation: use `neuralprocess` Python package or implement from scratch using `torch.nn.MultiheadAttention` for cross-attention encoder (~150 lines).
6. Dropout 50% of context points during training (standard NP training protocol) to force the model to handle partial observations.

**References:**
- "Attentive Neural Processes," Kim et al. 2019 (arXiv:1901.05761) — adds cross-attention to NPs; dramatically improves function reconstruction quality.
- "Neural Processes," Garnelo et al. 2018 (arXiv:1807.01622) — foundational paper.
- No existing CFD application confirmed as of 2025 — this is a genuinely novel application domain.

### Idea 4: Hard Bernoulli/Kutta Constraint Output Layer

**Hypothesis:** Enforcing exact physical constraints at the output layer — specifically the Kutta condition (pressure continuity at the trailing edge) and the Bernoulli relation (Cp = 1 - (V/V_inf)²) as hard projection steps — will reduce systematic errors in the high-curvature trailing edge region where current predictions are worst, targeting p_tan -5–8%.

**Why bold:** All current loss terms are soft — they penalize violations but allow the model to violate physics at inference time. A hard constraint layer enforces zero violation regardless of what the backbone predicts, by construction. The Kutta condition for a sharp trailing edge says the pressure must be equal on the upper and lower surfaces at the TE node — this can be enforced as a simple averaging step on the TE node pair. The Bernoulli relation connects velocity magnitude to pressure coefficient — if the velocity predictions are better-constrained, the pressure predictions follow. This is the Lagrangian constraint approach from physics-informed deep learning, applied as a post-processing layer rather than a loss term.

**Expected target metric:** p_tan (tandem TE interaction region), p_in secondary. TE pressure errors are typically the largest surface errors.

**Implementation sketch:**
1. Identify the trailing edge node pair (upper TE, lower TE) for each foil using the `te_coord_frame` logic already in the codebase.
2. Hard Kutta constraint: `p_TE = (p_upper_TE + p_lower_TE) / 2` applied as a differentiable post-processing step after the model output. Backpropagation flows through this averaging operation.
3. Bernoulli consistency: given predicted `(Ux_i, Uy_i)`, compute `V_i² = Ux_i² + Uy_i²` and add a loss term `||p_i - (1 - V_i²/V_inf²)||_2²` at surface nodes. This is softer than a hard constraint but enforces the coupling between velocity and pressure predictions.
4. Stagnation point: the stagnation point (Cp = 1.0) location can be identified as the node with maximum Cp on the leading-edge region. Add a penalty for Cp > 1.0 elsewhere on the surface (physically impossible for inviscid flow, approximately true for viscous).
5. Weight: start with Kutta constraint weight = 1.0 (hard, via projection); Bernoulli weight = 0.1 (soft, via loss). The hard Kutta step is parameter-free.

**References:**
- "Physics-Informed Neural Networks," Raissi et al. 2019 (JCP) — foundational constraint enforcement via loss terms.
- "Hard Constraints for Neural Networks in Physics," various 2023–2025 — projection-based constraint layers.
- Existing codebase: `--te_coord_frame` flag — TE node identification already implemented, providing the infrastructure for hard TE constraints.

### Idea 5: Koopman Operator Lifting for Surface Pressure Dynamics

**Hypothesis:** Lifting the surface pressure distribution to a Koopman eigenfunction space — where nonlinear pressure dynamics become linear — and learning the Koopman operator via a deep autoencoder will enable the model to discover a coordinate system in which tandem interference effects are linearly separable, targeting p_tan -8–12%.

**Why bold:** Koopman operator theory provides a rigorous mathematical framework for linearizing nonlinear dynamical systems by working in an infinite-dimensional function space. For tandem foil aerodynamics, the interference between fore and aft foil pressure fields is a nonlinear superposition problem — exactly the class of problem Koopman methods are designed for. The key insight: learn an encoder φ(Cp_surface) → z ∈ R^d such that the Koopman operator K is linear (K·z_fore + Kz_aft ≈ z_tandem). This decomposition is not achievable with the current additive feature approach. Recent Koopman Neural Operator work (arXiv:2301.10022) shows this is feasible for PDE systems.

**Expected target metric:** p_tan primary (-8–12%), p_oodc secondary. Tandem interference is the canonical nonlinear superposition problem.

**Implementation sketch:**
1. Train a Koopman autoencoder: encoder f: R^N_surf → R^d (d=64–128), Koopman matrix K ∈ R^{d×d}, decoder g: R^d → R^N_surf.
2. Loss: reconstruction loss + Koopman linearity loss (||K·f(Cp_t) - f(Cp_{t+1})||²) where t indexes flow condition steps (AoA sweep).
3. Integration: use the Koopman embedding of single-foil panel Cp as an additional conditioning signal for the tandem prediction. The tandem Koopman embedding is predicted as a function of the two single-foil embeddings.
4. Alternatively (simpler): train the main model with an auxiliary Koopman loss on the panel Cp embeddings as a regularizer that enforces linearity in latent space.
5. Key risk: the AoA sweep structure needed for Koopman training may not be well-represented in the dataset — check dataset structure before committing.

**References:**
- "Koopman Neural Operator," arXiv:2301.10022 — Koopman operators for PDE systems; learns linear dynamics in latent space.
- "Deep Learning for Universal Linear Embeddings of Nonlinear Dynamics," Lusch et al. 2018 (Nature Communications) — foundational Koopman autoencoder.
- "Koopman Operator-based Model Reduction for Switched-System Control of PDEs," various 2024 — application to fluid dynamics.

### Idea 6: SE(2)-Equivariant Architecture for Geometric Generalization

**Hypothesis:** Replacing the coordinate-based input features with SE(2)-equivariant representations — where the model's predictions transform correctly under 2D rotation and translation of the geometry — will allow the model to generalize across different global orientations (including tandem stagger and gap configurations) without memorizing orientation-specific patterns, targeting p_oodc -8–12% and p_tan -5%.

**Why bold:** The current Transolver takes raw (x, y) coordinates as input — meaning rotating the tandem configuration by 5° produces a completely different input representation, forcing the model to re-learn geometry at each orientation. An SE(2)-equivariant model knows by construction that rotating the geometry should produce a correspondingly rotated output, halving the effective dimensionality of the learning problem. Recent work (arXiv:2504.07741 — equivariant GNN for turbulence; arXiv:2511.09769 — equivariant NN for RANS) demonstrates 10–30% improvements from equivariance in fluid dynamics settings.

**Expected target metric:** p_oodc primary (OOD configurations with unusual orientations), p_tan secondary (tandem stagger angle generalization).

**Implementation sketch:**
1. Represent each node with SE(2)-equivariant features: (r, θ) relative to the airfoil centroid instead of (x, y). Pairwise relative coordinates instead of global coordinates.
2. Use `e2cnn` library (equivariant neural networks) or implement steerable convolutions for the surface decoder.
3. Minimal change path: replace only the coordinate input features with SE(2)-invariant scalars (distance to centroid, local curvature) and SE(2)-equivariant vectors (outward normal). This preserves the Transolver backbone but makes the inputs invariant to global rotation.
4. The `--te_coord_frame` flag already uses a local coordinate frame at the TE — extend this to a per-node local frame defined by the surface normal and tangent.
5. Reference implementation: `e2cnn` Python library provides SE(2)-equivariant convolutions; `torch_geometric` provides equivariant GNN layers.

**References:**
- "Equivariant GNN for Turbulence," arXiv:2504.07741 — equivariant architecture for fluid dynamics; 15-20% improvement over standard GNN.
- "Equivariant Neural Network for RANS," arXiv:2511.09769 — SE(2) equivariance for Reynolds-averaged Navier-Stokes prediction.
- "RotEqNet," Marcos et al. 2017 — foundational equivariant CNN; shows equivariance benefit for orientation-variable tasks.
- `e2cnn` library: https://github.com/QUVA-Lab/e2cnn — PyTorch-native SE(2)-equivariant layers.

### Idea 7: Flow Matching on Pressure Field (Generative Surrogate)

**Hypothesis:** Training a flow matching model (conditional normalizing flow with straight-line interpolation trajectories) that maps Gaussian noise to the pressure field distribution — conditioned on geometry and flow conditions — will produce sharper, physically-consistent pressure distributions compared to the current regression approach, targeting p_tan -10–15% and p_in -5%.

**Why bold:** The current model predicts a point estimate (MAP) of the pressure field. Flow matching learns the full conditional distribution p(Cp | geometry, Re, AoA) and samples from it — this is fundamentally different from regression. The CFO paper (arXiv:2512.05297) showed 87% error reduction on PDE right-hand-side learning using flow matching. The key insight: pressure fields in tandem configurations are multi-modal (the same geometry can produce different wake states depending on initialization) — a regression model averages over modes, while a generative model can capture them. At inference, take the mode (multiple samples + pick the one with lowest panel residual).

**Expected target metric:** p_tan primary (multi-modal tandem wake states), p_in secondary.

**Implementation sketch:**
1. Use `torchdyn` or `torchcfm` (PyTorch conditional flow matching library).
2. Architecture: keep Transolver as the conditioning encoder; add a small UNet/MLP operating on the pressure field as the flow matching velocity field model.
3. Training: sample time t ~ U(0,1), interpolate between Gaussian noise and true pressure field, train the velocity field model to predict the interpolation direction.
4. Inference: integrate the ODE from t=0 (noise) to t=1 (prediction) using a fast ODE solver (10-20 NFE sufficient for flow matching).
5. Key constraint: inference must be fast enough for the validation loop — use flow matching (not diffusion) specifically because it supports 1-step distillation.
6. Critical: 30-min training budget limits the number of ODE function evaluations; use 5-10 NFE at inference, no more.

**References:**
- "Continuous Flow Operator (CFO)," arXiv:2512.05297 — flow matching for PDE right-hand-side learning; 87% error reduction; time-resolution invariant.
- "Flow Matching Meets PDEs," arXiv:2506.08604 — direct application of flow matching to PDE surrogate learning.
- "Flow Marching," arXiv:2509.18611 — flow matching with marching scheme for sequential spatial domains; relevant to surface-ordered predictions.
- `torchcfm`: https://github.com/atong01/conditional-flow-matching — clean PyTorch implementation.

### Idea 8: Contrastive Geometry Pretraining for OOD Generalization

**Hypothesis:** Self-supervised pretraining on a large synthetic airfoil library (UIUC + NACA 4-digit series, ~1500 geometries) using contrastive learning — where augmented views of the same geometry (random AoA perturbation, small thickness scaling) are treated as positive pairs — will learn a geometry embedding that generalizes across the tandem configuration space, targeting p_oodc -8–12%.

**Why bold:** The current model learns geometry representations supervised entirely by pressure labels. Contrastive pretraining creates a rich geometry encoder without needing any CFD labels — and with access to arbitrary synthetic geometries. The geometry encoder then provides a strong prior for the downstream pressure prediction. This is directly analogous to BERT pretraining for NLP and JEPA pretraining for vision. The PI-JEPA paper (arXiv:2604.01349) demonstrates this approach for subsurface flow, showing strong OOD generalization. The key difference from failed DPOT pretraining (PR #2364): DPOT used the same training set (no inductive bias); this approach uses a different, larger synthetic dataset with genuine geometric diversity.

**Expected target metric:** p_oodc primary (-8–12%), p_tan secondary. Contrastive pretraining is specifically designed for distribution shift.

**Implementation sketch:**
1. Generate synthetic airfoil library: NACA 4-digit series (0006–9415, ~200 geometries) + UIUC database (~1300 geometries); parameterize surfaces at 256 arc-length points.
2. Augmentations for positive pairs: (a) random AoA rotation ±5°, (b) chord scaling ±10%, (c) random surface point dropout 20%.
3. Contrastive objective: SimCLR-style NT-Xent loss, or BYOL (no negative pairs needed). Encoder: 4-layer MLP over surface point cloud features → 64-dim embedding.
4. Downstream: freeze the geometry encoder, fine-tune only the pressure prediction head on the TandemFoilSet. Or: use the geometry embedding as an additional conditioning signal (concatenate with Re embedding).
5. Key implementation detail: the pretraining is done offline before the main training run; the pretrained weights are loaded via `--pretrained_geometry_encoder path/to/checkpoint.pt`.
6. Training budget: pretraining is fast (geometry only, no CFD labels) — 100 epochs on 1500 geometries takes ~5 minutes; well within the 30-min budget.

**References:**
- "PI-JEPA: Physics-Informed Joint Embedding Predictive Architecture," arXiv:2604.01349 — contrastive/self-supervised pretraining for subsurface flow surrogate; strong OOD generalization.
- "SimCLR: A Simple Framework for Contrastive Learning of Visual Representations," Chen et al. 2020 — foundational contrastive pretraining method.
- "BYOL: Bootstrap Your Own Latent," Grill et al. 2020 — contrastive pretraining without negative pairs; more stable than SimCLR.
- UIUC Airfoil Coordinates Database: https://m-selig.ae.illinois.edu/ads/coord_database.html — 1600+ free-to-use airfoil geometries.
