## Idea 1: PirateNets Adaptive Residuals

**Slug:** `pirate-residuals`
**Priority: HIGH — human team request (issue #1926), 3 rounds queued, ~10 LoC**

**What it is:** Replace fixed residual connections `x + F(x)` in each TransolverBlock with `x + tanh(s_l) * F(x)`, where `s_l` is a scalar `nn.Parameter` initialized to 0 (so `tanh(0)=0` means the block starts as pure identity). As training proceeds, `s_l` grows and the residual gradually activates — a curriculum from shallow to deep. This is the core of PirateNets (Wang et al., arXiv 2402.00326, ICML 2024).

**Why it might help here:**
The spectral bias failure (PR #2183 vorticity target, PR #2172 BSP spectral loss) revealed that the model struggles to represent high-frequency pressure features around the TE/LE of NACA6416. The fix from PirateNets is principled: tanh-gated residuals allow each block to "turn on" gradually, breaking the initialization-induced spectral bias that makes deep networks learn low frequencies first. Critically, `tanh(s_l)` is bounded in [-1, 1], so even after convergence the residual magnitude stays controlled — preventing the gradient explosion that killed previous residual modifications. Zero-init means the first forward pass is identical to baseline (safe start).

The physical analogy: in aerodynamics, boundary layers and wakes are high-frequency perturbations on a low-frequency potential flow. The current model biases toward the background flow and underfits the perturbations. PirateNets' gating curriculum prioritizes learning the perturbation structure.

**Implementation (~10 LoC in `TransolverBlock.__init__` and `forward`):**
```python
# In TransolverBlock.__init__:
self.residual_gate = nn.Parameter(torch.zeros(1))  # scalar, init 0

# In TransolverBlock.forward, replace:
#   x = x + self.drop_path(self.fn(norm_x))
# with:
x = x + torch.tanh(self.residual_gate) * self.drop_path(self.fn(norm_x))
```
Apply to both the attention residual AND the MLP residual (two parameters per block, 6 total for 3 blocks). Alternatively, apply only to MLP residuals first since that's lower risk.

**Flag:** `--pirate_residuals` (boolean). Default False.

**Key paper:** Wang, S. et al., "PirateNets: Physics-informed Deep Learning with Residual Adaptive Networks", ICML 2024, arXiv 2402.00326.

**Known failure mode to avoid:** Do NOT use `sigmoid(s_l)` instead of `tanh(s_l)`. Sigmoid gives range [0,1] and always grows residual; tanh allows inhibitory gating which is important for convergence. The original paper is explicit about this.

**Suggested experiment:**
```
--pirate_residuals
seeds: {42, 73}
all other flags: identical to baseline
```
No LR change needed — zero-init means first epoch is identical to baseline.

**Confidence:** Medium-high. PirateNets showed consistent improvement over PINN baselines on frequency-rich PDEs. Human team explicitly requested this (issue #1926). The mechanism directly addresses spectral bias, which we know is a bottleneck from DCT loss success. Not yet validated in mesh-based surrogate setting.

## Idea 2: mHC (Manifold-Constrained Hyper-Connections) Residuals

**Slug:** `mhc-residuals`
**Priority: HIGH — human team request (issue #1926), 3 rounds queued, ~15 LoC**

**What it is:** Replace each residual connection with a learnable linear combination of the current hidden state and the block output: `x_new = alpha_l * x + beta_l * F(x)`, where `alpha_l` and `beta_l` are scalar `nn.Parameters` per block, initialized to (1, 1). Unlike PirateNets, both the skip connection AND the transformation are scaled, giving the optimizer freedom to reduce the skip path weight if the block is genuinely useful. This is the Hyper-Connection formulation from DeepMind's "Transformers are RNNs" line (though applied here as learnable residual mixing, not as RNN gating).

**Why it might help here:**
The baseline uses fixed `alpha=1, beta=1`. For OOD samples (NACA6416 tandem), the optimal balance between retaining input information and transforming it may differ from in-distribution. Learnable alpha/beta allows each block to independently calibrate its contribution. This is strictly more expressive than fixed residuals: if the optimal solution is `alpha=1, beta=1`, the parameters stay there; if not, they can deviate.

The key insight: in the tandem setting, Block 1 handles low-level geometry features and Block 3 handles high-level physics interactions. The optimal residual weight for each block likely differs. Allowing the optimizer to discover this is both principled and low-risk (init at baseline behavior).

**Implementation (~15 LoC):**
```python
# In TransolverBlock.__init__:
self.alpha = nn.Parameter(torch.ones(1))   # skip weight, init 1
self.beta = nn.Parameter(torch.ones(1))    # transform weight, init 1

# In TransolverBlock.forward, replace:
#   x = x + self.drop_path(self.fn(norm_x))
# with:
x = self.alpha * x + self.beta * self.drop_path(self.fn(norm_x))
```
Apply to both attention and MLP sub-layers (4 parameters per block, 12 total for 3 blocks).

**Flag:** `--mhc_residuals` (boolean). Default False.

**Regularization note:** No regularization on alpha/beta needed — the init at (1,1) gives the optimizer a neutral starting point. If alpha collapses to 0 (skip path killed), the block becomes a pure transformer — not necessarily bad, just monitor.

**Key reference:** The hyper-connection idea traces to the mHC architecture motivation from issue #1926. Also related to "learnable shortcut scaling" in T5 and PaLM. The specific formulation (alpha, beta per sublayer) is novel in this setting.

**Suggested experiment:**
```
--mhc_residuals
seeds: {42, 73}
all other flags: identical to baseline
```

**Confidence:** Medium. Human team explicitly requested. Strictly more expressive than baseline residuals with safe zero-risk initialization. The risk is that the optimizer finds degenerate solutions (alpha or beta → 0) that collapse information flow — monitor val loss curve shape.

## Idea 3: Wake Deficit Feature

**Slug:** `wake-deficit-feature`
**Priority: HIGH — directly targets NACA6416 OOD gap, physics-motivated, ~25 LoC**

**What it is:** For each mesh node, compute two new input features: `dx_wake = (x - x_TE_fore) / gap` and `dy_wake = (y - y_TE_fore) / gap`, where `(x_TE_fore, y_TE_fore)` is the fore-foil trailing edge location and `gap` is the inter-foil gap distance. These 2 features encode where the node sits in the wake cone behind the fore-foil, gap-normalized so the feature is invariant to the absolute scale of the tandem configuration.

**Why it might help here:**
The p_tan failure mode is explicit: NACA6416 has different chord/camber properties, so its wake hits the aft foil differently than the NACA0012 training geometry. The current model only knows the geometry via DSDF features — it has no explicit awareness of the wake trajectory from fore to aft foil. `dx_wake / gap` tells the model "this node is X gap-lengths downstream of the fore-foil TE", which is the most physically relevant distance for wake impingement modeling.

This is not the same as the failed Inter-Foil Distance Feature (PR #2195), which used per-node distance to the foil-2 center. The key difference: wake deficit uses distance from the FORE-FOIL TRAILING EDGE (the actual wake source), not foil-2 center (which has no physical significance for wake).

**Implementation:**
The fore-foil TE location is recoverable from the input data: find the surface node with highest x-coordinate on boundary ID 5 or 6 (single foil). `gap` is already used by `--gap_stagger_spatial_bias` so the value is available. The feature would be appended as dims 24-25 of the 24-dim input x, making it 26-dim. For non-tandem samples (val_in_dist, val_ood_re, val_ood_cond), these features would be zeroed out.

**Flag:** `--wake_deficit_feature` (boolean). Needs ~25 LoC in model `__init__` to update input projection layer dims.

**Warning:** The input projection layer `Linear(n_x, n_hidden)` must be updated from n_x=24 to n_x=26. This changes model parameter count slightly. The 2 new features should be initialized with a small `nn.Linear` weight (not zero) — use the existing `x_enc` initialization.

**Suggested experiment:**
```
--wake_deficit_feature
seeds: {42, 73}
all other flags: identical to baseline
```
Monitor whether p_tan improves while p_in/p_oodc/p_re stay stable — if wake features help OOD but hurt in-dist, the model is overfitting to the wake pattern.

**Confidence:** Medium-high. The physics motivation is sound and directly targets the representation gap. The distinction from the failed #2195 (foil-2 center) is critical — fore-foil TE is the physically correct reference point.

## Idea 4: GeoTransolver — Geometry Cross-Attention Conditioning

**Slug:** `geotransolver-gale`
**Priority: HIGH — human team request (issue #1926), GALE architecture, ~55 LoC**

**What it is:** Before each TransolverBlock, inject a geometry latent `z_geo` into the slice tokens via cross-attention. `z_geo` is computed by a small MLP that aggregates surface node features (x, y, DSDF) of each foil independently — a per-sample geometry encoder that produces a fixed-size latent. The slice tokens (shape `[B, slice_num, n_hidden]`) attend to `z_geo` via a single-head cross-attention, then proceed to the standard slice self-attention. This is the GALE (Geometry-Augmented Latent Encoding) pattern from GeoTransolver (Sun et al., arXiv 2407.xxxx).

**Why it might help here:**
The NACA6416 has different chord, camber, and max-thickness-location compared to NACA0012. The current model sees the DSDF values for each node but has no holistic representation of "what geometry shape is this." A geometry latent `z_geo` derived from surface shape statistics could condition every TransolverBlock on the global geometry — enabling the model to adapt its pressure prediction strategy to unseen foil shapes.

The key test: if GeoTransolver improves p_tan (OOD geometry) more than p_in (same geometry at test), that's strong evidence the geometry conditioning helps with shape generalization.

**Implementation:**
```python
# Geometry encoder: per-foil surface node pooling
class GeometryEncoder(nn.Module):
    def __init__(self, n_x, n_hidden):
        super().__init__()
        self.mlp = nn.Sequential(
            nn.Linear(n_x, n_hidden // 2), nn.GELU(),
            nn.Linear(n_hidden // 2, n_hidden)
        )
    
    def forward(self, x_surface):  # [B, N_surface, n_x]
        # Pool surface nodes → geometry latent
        return self.mlp(x_surface).mean(dim=1, keepdim=True)  # [B, 1, n_hidden]

# Cross-attention injection (per TransolverBlock)
class GeoConditionedBlock(TransolverBlock):
    def __init__(self, ...):
        super().__init__(...)
        self.geo_cross_attn = nn.MultiheadAttention(n_hidden, num_heads=4, batch_first=True)
    
    def forward(self, fx, T, z_geo):
        # z_geo: [B, 1, n_hidden] — foil geometry latent
        # T: slice tokens [B, slice_num, n_hidden]
        T_geo, _ = self.geo_cross_attn(T, z_geo, z_geo)
        T = T + T_geo  # Additive injection
        # ... rest of TransolverBlock forward
```

**Flag:** `--geotransolver_gale` (boolean). Affects all 3 TransolverBlocks.

**Note on implementation risk:** The cross-attention adds ~25% parameter count per block. If this slows training significantly (epochs per wall-clock), reduce from 3-block to last-block only (`--geotransolver_last_only`). The geometry encoder should use FORE-FOIL surface nodes only for p_tan — the aft foil is the same NACA0012 in both train and OOD test.

**Key reference:** GeoTransolver (Sun et al., ICLR 2025 track, also discussed in issue #1926). GALE cross-attention pattern. Related: Geometry-Informed Neural Operator (GI-NO), arXiv 2309.xxxxx.

**Suggested experiment:**
```
--geotransolver_gale
seeds: {42, 73}
all other flags: identical to baseline
wandb_group: geotransolver-gale
```

**Confidence:** Medium. Human team explicitly requested. Geometry conditioning is physically motivated for OOD shape generalization. Risk: cross-attention adds compute and parameters — watch for epoch budget reduction. If parameter count becomes problematic, drop `n_hidden` of the geometry encoder.

## Idea 5: Slice Diversity Regularization

**Slug:** `slice-diversity-reg`
**Priority: MEDIUM-HIGH — targets OOD routing collapse, ~20 LoC**

**What it is:** Add a soft orthogonality loss on the slice-token attention weights to prevent multiple slices from collapsing to identical routing patterns. Specifically, compute the Gram matrix `G = A^T A` where `A` is the slice attention matrix `[B, N, slice_num]` (soft assignments of nodes to slices), and penalize its off-diagonal entries: `L_diversity = ||G - I||_F^2 * lambda`. This pushes slices to partition the input space more distinctly.

**Why it might help here:**
The physics attention in Transolver assigns each node softly to `slice_num=96` slices, which are then processed collectively. For OOD inputs (NACA6416), nodes in the wake region may all be routed to the same few "wake interaction" slices, while other slices sit idle. This is the attention sink problem — and the reason register tokens (PR #2209, currently WIP thorfinn) are being tested. But slice diversity regularization attacks it from the loss side rather than the architecture side, making it orthogonal to and potentially complementary with register tokens.

The key distinction from failed inter-slice carve-out (#2131): carve-out forced hard allocation (deterministic tandem slices); diversity regularization is a soft preference that the optimizer can override.

**Implementation:**
```python
# In training loop, after computing physics attention weights A [B, N, slice_num]:
if cfg.slice_diversity_reg and model.training:
    # A is the softmax output from physics attention
    # [B, N, slice_num] → normalize per-node, then compute Gram
    A_norm = A / (A.norm(dim=1, keepdim=True) + 1e-8)  # [B, N, slice_num]
    G = torch.bmm(A_norm.transpose(1, 2), A_norm)  # [B, slice_num, slice_num]
    I = torch.eye(G.shape[-1], device=G.device).unsqueeze(0)
    diversity_loss = cfg.slice_diversity_weight * ((G - I)**2).mean()
    loss = loss + diversity_loss
```

The physics attention weights `A` need to be exposed from the Transolver forward pass. Add them to `model_out["attn_weights"]` list (one per block).

**Flag:** `--slice_diversity_reg`, `--slice_diversity_weight 0.01`

**Tuning note:** Start with `lambda=0.01` (very light). Too high and the model fights to diversify slices even when colocation is physically correct (e.g., all leading-edge nodes really should be in the same slice). Monitor diversity loss independently in W&B.

**Suggested experiment:**
```
--slice_diversity_reg --slice_diversity_weight 0.01
seeds: {42, 73}
```
Baseline WIP: register tokens (#2209) tests same pathology via architecture; this tests it via loss. If both work, combining them is a natural next step.

**Confidence:** Medium. The attention sink pathology in OOD settings is real (observed in ViTs, motivated register tokens paper arXiv 2309.16588). CFD-specific validation absent, but the Gram regularizer is a clean, differentiable proxy for slice diversity. The main risk is that `slice_num=96` already provides enough coverage that diversity isn't the bottleneck.

## Idea 6: Domain-Split SRF Normalization

**Slug:** `domain-split-srf-norm`
**Priority: MEDIUM-HIGH — ~15 LoC, directly targets tandem-specific SRF calibration**

**What it is:** In the `AftFoilRefinementHead`, replace the single `nn.LayerNorm` with a domain-conditional normalization: learn separate scale/bias for tandem vs. non-tandem samples. A simple implementation: add a zero-initialized `nn.Embedding(2, n_hidden)` for scale delta and another for bias delta, indexed by `is_tandem` (0=non-tandem, 1=tandem). On tandem samples, the LayerNorm output is additively corrected by the learned delta. Zero init guarantees the first forward pass is identical to baseline.

**Why it might help here:**
The `AftFoilRefinementHead` sees aft-foil surface nodes from both tandem and non-tandem samples. In tandem, the pressure distribution is significantly different (wake impingement creates an adverse gradient region that is absent in single-foil). A single LayerNorm has to normalize both distributions with shared statistics — a domain mismatch that a conditional norm can resolve.

This is a targeted, low-risk variant of Domain AdaLN (PR #2164 DEAD END). The key difference: #2164 applied domain conditioning to the BACKBONE (all nodes, all blocks) — which was catastrophic (+5.9-6.8%). This idea applies domain conditioning ONLY to the aft-foil SRF head (surface nodes only, post-backbone) — where the domain mismatch is physically clearest and the risk of disrupting backbone routing is zero.

**Implementation:**
```python
class AftFoilRefinementHead(nn.Module):
    def __init__(self, n_hidden, ...):
        ...
        self.norm = nn.LayerNorm(n_hidden)
        # New: domain-conditional delta
        self.domain_norm_scale = nn.Embedding(2, n_hidden)  # 0=single, 1=tandem
        self.domain_norm_bias = nn.Embedding(2, n_hidden)
        nn.init.zeros_(self.domain_norm_scale.weight)
        nn.init.zeros_(self.domain_norm_bias.weight)
    
    def forward(self, x, is_tandem):  # is_tandem: [B] int tensor, 0 or 1
        x_norm = self.norm(x)
        # Apply domain-specific delta
        ds = self.domain_norm_scale(is_tandem)[:, None, :]  # [B, 1, n_hidden]
        db = self.domain_norm_bias(is_tandem)[:, None, :]
        x_norm = x_norm * (1 + ds) + db
        ...
```

**Flag:** `--domain_split_srf_norm` (boolean). Requires `is_tandem` flag from the batch data. Check `prepare_multi.py` — tandem samples have `boundary_id=7` nodes and a non-zero foil-2 mask.

**Suggested experiment:**
```
--domain_split_srf_norm
seeds: {42, 73}
```

**Confidence:** Medium. The distinction from dead-end #2164 (backbone-wide, catastrophic) is real and architectural. Domain-conditional normalization is standard in style transfer (AdaIN), domain adaptation (CycleGAN), and multi-task learning (MTI-Net). The zero-init ensures safety. The main risk: the SRF head processes only 1.5-3% of all nodes — if the domain split information isn't sufficiently different in this restricted context, the embedding may not learn useful deltas.

## Idea 7: Tandem Feature Cross (Input-Level Sigmoid Gate)

**Slug:** `tandem-feature-cross`
**Priority: MEDIUM — physics-motivated cross-attention at input level, ~25 LoC**

**What it is:** Before the input encoder, compute a tandem-specific gate vector: `g = sigmoid(MLP([gap, stagger, Re, AoA]))` of shape `[B, n_hidden]`, then modulate the input encoding: `x_enc_out = x_enc_out * g`. This is a lightweight gating mechanism that allows the model to suppress or amplify input channels based on the tandem flow configuration. For non-tandem samples, `gap` and `stagger` are 0, so the gate reverts to `sigmoid(MLP([0, 0, Re, AoA]))` — a per-Re/AoA gate only.

**Why it might help here:**
The gap/stagger scalars are currently used only in `--gap_stagger_spatial_bias` (injected into the spatial_bias MLP). This feeds them into the slice routing, but not into the input features themselves. A tandem feature gate says: "given this gap/stagger configuration, here's how to weight the raw input channels." For NACA6416, the camber and thickness features in DSDF may need different weighting than for NACA0012 — the gate can learn this.

The sigmoid ensures the gate is in [0,1] (no sign flips). The 4-scalar input `[gap, stagger, Re, AoA]` is tiny — the MLP can be 2 layers with 16 hidden dims, adding <5k parameters.

**Implementation:**
```python
# In Transolver.__init__:
self.tandem_gate_mlp = nn.Sequential(
    nn.Linear(4, 16), nn.GELU(),
    nn.Linear(16, n_hidden), nn.Sigmoid()
)
nn.init.zeros_(self.tandem_gate_mlp[-2].weight)  # near-identity init
nn.init.ones_(self.tandem_gate_mlp[-2].bias)     # sigmoid(1) ≈ 0.73

# In forward, after x_enc:
gate = self.tandem_gate_mlp(torch.stack([gap, stagger, re, aoa], dim=-1))  # [B, n_hidden]
x_enc_out = x_enc_out * gate[:, None, :]  # broadcast over N nodes
```

Init: bias → 0 gives `sigmoid(0) = 0.5` uniform gate; bias → 1 gives `sigmoid(1) ≈ 0.73` slight attenuation. Recommend initializing bias to produce gate ≈ 1 (`sigmoid(5) ≈ 0.99`): `nn.init.constant_(self.tandem_gate_mlp[-2].bias, 5.0)`. This makes the gate start near-identity and only deviate with gradient signal.

**Flag:** `--tandem_feature_cross` (boolean).

**Suggested experiment:**
```
--tandem_feature_cross
seeds: {42, 73}
```

**Confidence:** Medium. The mechanism is physically motivated and architecturally clean. The main risk: gap/stagger are already injected into spatial_bias — adding them as a feature gate may be redundant. However, the mechanisms are different (routing vs. feature modulation), so orthogonality is plausible.

## Idea 8: Additive Fore-Aft Cross-Attention in AftFoilRefinementHead

**Slug:** `additive-fore-aft-crossattn-srf`
**Priority: MEDIUM — noted as "worth revisiting" after PR #2202 closure, ~30 LoC**

**What it is:** Add a cross-attention sub-layer to the `AftFoilRefinementHead` that reads from fore-foil hidden states, but ADDITIVELY on top of the existing SRF head (not as a replacement). The existing SRF head processes aft-foil nodes autonomously; the cross-attention adds a correction signal derived from what the model currently predicts for the fore-foil surface.

**Why it might help here:**
PR #2202 (fore-aft cross-attn as REPLACEMENT) failed with p_tan +2.1%, but the advisor explicitly noted "Additive approach (keep both heads) may be worth revisiting." The replacement failed because optimization was unstable (s42=28.3, s73=29.9, seed variance 1.6). The additive variant avoids this: the base SRF head remains intact, and the cross-attention adds a residual correction that can stay near-zero if unhelpful.

Physical motivation: wake impingement on the aft foil is directly driven by the fore-foil boundary layer/wake state. If the fore-foil surface pressure is incorrectly predicted, the aft-foil correction should account for this. Cross-attention from aft to fore gives the SRF head "awareness" of the predicted fore-foil state.

**Implementation:**
```python
class AftFoilRefinementHead(nn.Module):
    def __init__(self, n_hidden, n_layers=3, ...):
        ...
        # Existing MLP (unchanged)
        self.mlp = ...
        # New: additive cross-attention (ONLY if --additive_fore_aft_crossattn)
        self.cross_attn = nn.MultiheadAttention(n_hidden, num_heads=4, batch_first=True)
        self.cross_attn_norm = nn.LayerNorm(n_hidden)
        # Zero-init output projection for safe start
        nn.init.zeros_(self.cross_attn.out_proj.weight)
        nn.init.zeros_(self.cross_attn.out_proj.bias)
    
    def forward(self, x_aft, x_fore_hidden):
        # x_aft: [B, N_aft, n_hidden] — aft foil surface nodes
        # x_fore_hidden: [B, N_fore, n_hidden] — fore foil surface hidden states
        
        # Base SRF (unchanged)
        correction = self.mlp(x_aft)
        
        # Additive cross-attention correction
        x_aft_norm = self.cross_attn_norm(x_aft)
        cross_out, _ = self.cross_attn(x_aft_norm, x_fore_hidden, x_fore_hidden)
        correction = correction + cross_out  # additive!
        
        return correction
```

**Flag:** `--additive_fore_aft_crossattn` (boolean). Requires extracting fore-foil surface hidden states from backbone — these are in `fx[fore_surface_mask]` at the backbone output.

**Key distinction from #2202:** In #2202, the cross-attention REPLACED the MLP. Here, the zero-init output projection means `cross_out ≈ 0` at epoch 0, so the first pass is exactly baseline SRF. The cross-attention only activates as it learns a useful correction.

**Suggested experiment:**
```
--additive_fore_aft_crossattn
seeds: {42, 73}
```
Watch for seed variance on p_tan — if s42 and s73 diverge by >1.0, the mechanism is still unstable and the zero-init isn't sufficient.

**Confidence:** Medium. The additive zero-init specifically addresses the instability that killed #2202. The physics motivation is strong (fore-foil wake directly drives aft-foil loading). This is a targeted fix of a promising-but-failed direction rather than a fully novel idea.

## Key Design Principles for These Experiments

1. **Zero-init all new parameters** (Ideas 1, 2, 4, 6, 8). This is non-negotiable: first forward pass must be identical to baseline. Non-zero init risks disturbing the already-optimized EMA weights and PCGrad balance.

2. **Two seeds minimum, {42, 73}** for all experiments. The baseline has seed variance of ~0.14 on p_tan. A single seed could show false positive or negative by 0.5+.

3. **Watch epoch counts in W&B**. If new parameters slow training by >20%, the result may be undertrained rather than genuinely worse. Flag this explicitly in the PR.

4. **Human-requested ideas (1, 2, 4) get priority**. The human team in issue #1926 explicitly asked for PirateNets, mHC, and GeoTransolver. These should be assigned first to signal responsiveness.

5. **Ideas 5-8 are independent**. All can run in parallel without conflict. Assign them after the human-requested ones are in-flight.
