## Research Framework

The 7 active WIP PRs are already working the most obvious levers: attention routing (slice carve-out,
spatial bias), local context (KNN volume context), conditioning (FiLM on SRF, spatial bias with
gap/stagger), and auxiliary losses (pressure gradient). What they have NOT tried:

1. **Global geometry shape encoding** — the model sees DSDF channels and coordinates, but never
   a compact global shape descriptor for the front foil. OOD generalization should improve if the
   model understands "this is a NACA6416-like shape" even if it never trained on NACA6416.

2. **Inter-node message passing** — Transolver uses soft-assignment attention (nodes → slices → nodes),
   but there is no direct node-to-node communication. Near-surface nodes on the aft foil receive
   only indirect signal from the fore foil through shared slice participation. Explicit graph
   message passing between surface nodes could improve aft-foil prediction.

3. **Tandem-specific geometric features** — the channel geometry between fore and aft foil
   (gap size, stagger, relative position) determines the inter-foil jet. These are encoded as
   scalars (gap=x[:,0,21], stagger=x[:,0,22]) but not as spatial fields that vary node-by-node.

4. **Inductive bias via coordinate-frame normalization** — the model uses absolute x,y coordinates.
   Tandem OOD generalization would improve if aft-foil nodes were expressed relative to the
   fore-foil trailing edge, which is the physically meaningful reference frame for wake effects.

5. **Output-space ensemble diversity** — the 8-seed ensemble reduces variance but not bias.
   A dropout-based epistemic uncertainty head during training could identify high-uncertainty
   aft-foil nodes for targeted loss upweighting.

## Ranked Hypotheses

### 1. foil1-relative-coords — Fore-Foil Trailing-Edge Relative Coordinate Frame for Aft Nodes

**Priority: HIGH. ~20 LoC. Expected -3% to -7% p_tan. MEDIUM risk.**

**What it is:** Add 2 extra input features to aft-foil surface nodes (boundary ID=7): the x,y
coordinates of each aft-foil node expressed **relative to the fore-foil trailing edge** (TE).
Non-aft-foil nodes get zeros for these features.

**Why it helps p_tan:** The aft-foil pressure distribution is physically controlled by the inter-foil
channel jet — the jet exits the fore-foil TE and impinges on the aft foil. The model currently has
no coordinate-frame that makes this physically obvious. If we express aft-foil node positions as
(delta_x_from_fore_TE, delta_y_from_fore_TE), the model sees where each aft-foil node sits in
the wake frame, which is a much more informative quantity for predicting wake-driven pressure.

This is different from the gap/stagger spatial bias (#2130) which extends raw_xy uniformly. This
idea adds a PHYSICALLY MEANINGFUL offset that changes the coordinate origin for aft nodes only.

**Why it might generalize OOD:** The fore-foil TE location is a structural landmark that encodes
the wake source. For the OOD NACA6416 shape, the TE position shifts compared to training, and
this shift directly matters for aft-foil pressure. Expressing coordinates in the TE-relative frame
makes this invariant: regardless of fore-foil shape, the aft-foil leading edge is always at some
(dx, dy) from the fore-foil TE. This geometric invariance should reduce the effective OOD shift.

**Implementation — specific code locations:**

The fore-foil TE needs to be estimated from the input. A robust proxy: the surface node with the
maximum x-coordinate among boundary ID=6 nodes (fore-foil surface). The fore-foil surface nodes
can be identified by: `_is_tandem & (saf_norm <= 0.005) & is_surface` (same logic as in the
`_aft_foil_mask` computation already in the training loop).

```python
# After line 1580 (where _aft_foil_mask is computed):
# Build fore-foil mask (boundary ID=6): tandem, surface, saf norm SMALL
if aft_srf_head is not None or cfg.foil1_rel_coords:
    _fore_foil_mask = is_surface & (_raw_saf_norm <= 0.005) & _is_tandem.unsqueeze(1)  # [B, N]
    # Fore-foil trailing edge: max x-coord among fore-foil surface nodes (per batch)
    _fore_x = x[:, :, 0].clone()  # [B, N] -- still pre-standardization here
    _fore_x_masked = _fore_x.masked_fill(~_fore_foil_mask, float('-inf'))
    _fore_te_x = _fore_x_masked.max(dim=1).values  # [B]
    # y-coord at TE: argmax over x among fore-foil nodes
    _fore_te_idx = _fore_x_masked.argmax(dim=1)   # [B]
    _fore_te_y = x[torch.arange(B, device=device), _fore_te_idx, 1]  # [B]
    # Build relative coords for aft-foil nodes
    _rel_x = (x[:, :, 0] - _fore_te_x.unsqueeze(1)) * _aft_foil_mask.float()  # [B, N]
    _rel_y = (x[:, :, 1] - _fore_te_y.unsqueeze(1)) * _aft_foil_mask.float()  # [B, N]
    _foil1_rel = torch.stack([_rel_x, _rel_y], dim=-1)  # [B, N, 2]
    # For non-tandem samples, these are all zeros -- no signal leakage
```

Then append `_foil1_rel` to x AFTER standardization but BEFORE the Fourier PE:
```python
x = torch.cat([x, _foil1_rel, fourier_pe], dim=-1)  # 2 extra dims, appended after foil2_dist
```

Add config flag `foil1_rel_coords: bool = False`. Input dim increases by 2.

**Key distinction from dead ends:** This is NOT a new spatial bias variant (that's #2130). It adds
a physically motivated coordinate system that encodes wake-source geometry directly in the input,
available to all Transolver layers, not just the slice attention routing.

**Interaction with aft_foil_srf:** The aft-foil SRF head already uses `aft_hidden` from the trunk.
If the trunk receives better TE-relative coordinates, the aft_hidden will be richer, and the SRF
head correction will also improve. These are synergistic.

**Caveats:**
- The TE estimation (max x among fore-foil surface nodes) is a proxy that may break for highly
  swept geometries. The NACA series in this dataset is all relatively symmetric, so this is
  acceptable.
- For non-tandem samples, _foil1_rel is all zeros, which is a valid null signal.
- The relative coords are NOT standardized — they have physical scale (chord lengths ≈ 1.0).
  May need to scale by 1/gap to normalize. Try without normalization first.

**Suggested experiment:**
```
--foil1_rel_coords
--aft_foil_srf           (keep in baseline)
```
Run 2 seeds. Target: p_tan < 29.5.

**Confidence:** Strong — physically motivated inductive bias, directly addresses the wake-source
geometry information gap. Similar coordinate-frame normalization tricks appear throughout NeurIPS
fluid dynamics papers (FNO, GINO, DeepONet variants).

### 2. tandem-self-distill — Tandem-Domain Self-Distillation via EMA Teacher

**Priority: HIGH. ~25 LoC. Expected -2% to -5% p_tan. MEDIUM risk.**

**What it is:** During training, use the EMA model (which is more accurate) as a "teacher" to
generate pseudo-labels for tandem samples. After epoch 40 (when EMA has warmed up), for each
tandem training batch, run a forward pass through the frozen EMA model to get teacher predictions,
then add a KD loss between student and teacher outputs on **tandem samples only**.

This is self-distillation: student and teacher are the same architecture, but the teacher is
weight-averaged (more stable, lower variance). The key is restricting the KD loss to **tandem
samples** where variance reduction matters most for p_tan.

**Why it helps p_tan:** The EMA model is already used for evaluation (best metric comes from EMA).
But during training, the student model doesn't learn from the EMA's smoother predictions. For
tandem samples (which are rarer in training due to sample weights and are OOD at test time), the
EMA teacher's predictions are less noisy than the raw ground truth corrupted by target noise.
Adding a KD loss on tandem predictions gives the student a second training signal that is
specifically better calibrated for the tandem domain.

**Distinction from knowledge distillation (PR #2090 fern, closed):** That PR used a fully trained
offline ensemble as teacher. This uses the EMA of the current run — online self-distillation.
No offline teacher needed. Conceptually similar to "Born-Again Networks" (Furlanello et al. 2018)
but restricted to the tandem domain where signal-to-noise is worst.

**Implementation:**

```python
# After EMA update (line ~1966), during training loop after epoch cfg.kd_start_epoch (default 40):
if (cfg.tandem_selfdistill and epoch >= cfg.kd_start_epoch and
        ema_model is not None and is_tandem_batch.any()):
    with torch.no_grad():
        _ema_model.eval()
        _ema_out = ema_model({"x": x[is_tandem_batch]})
        _teacher_pred = _ema_out["preds"].float()
        _ema_model.train()
    _student_pred_tandem = pred[is_tandem_batch]  # already computed above
    # KD on surface nodes of tandem samples only
    _tan_surf = surf_mask[is_tandem_batch]  # [B_tan, N]
    _kd_loss = F.l1_loss(
        _student_pred_tandem[_tan_surf],
        _teacher_pred[_tan_surf]
    )
    loss = loss + cfg.kd_weight * _kd_loss
```

New flags: `--tandem_selfdistill` (bool), `--kd_start_epoch 40` (int), `--kd_weight 0.3` (float).
Sweep kd_weight: {0.1, 0.3, 0.5}.

**Memory note:** The EMA forward pass runs on the tandem subset only (not full batch), so memory
overhead is low. The EMA model is already stored on device.

**Key subtlety:** The KD target is `ema_model(x[is_tandem_batch])` — it gets the SAME input the
student sees (with augmentations applied). If the EMA teacher's prediction is better than the
noisy ground truth target, the student learns from a cleaner signal for tandem.

**Confidence:** Moderate-strong. Online self-distillation is well-validated (Hinton et al. 2015,
Furlanello et al. 2018). The tandem-specific restriction is novel but well-motivated. The risk is
that the EMA teacher is not much better than the student on tandem (since it also trains on the
same distribution). However, the EMA's smoothing does reduce prediction variance by ~20-30%.

### 3. foil-shape-ae — Foil Shape Autoencoder Latent as Condition Feature

**Priority: HIGH. ~60 LoC. Expected -2% to -6% p_tan. MEDIUM-HIGH risk.**

**What it is:** Train a small convolutional/MLP autoencoder on the DSDF profile of surface nodes
to compress the foil shape into a compact latent vector (e.g., 16 or 32 dims). Use this latent
as an additional global condition alongside Re/AoA in the AdaLN layers. Crucially, train the AE
JOINTLY with the main model so the latent adapts to what matters for flow prediction.

**Why it helps p_tan:** The fundamental challenge is that NACA6416 (front foil shape in val_tandem
transfer) was never seen during training. The model currently identifies shape through DSDF
channels at each node (local geometry) and through the Fourier PE (global position). But there
is no mechanism that says "this is a thin, cambered foil with NACA64xx characteristics" as a
GLOBAL shape feature. A shape AE latent provides exactly this: a compressed global representation
of the foil's cross-sectional geometry that can generalize to unseen shapes via interpolation in
latent space.

**Connection to OOD generalization:** The NACA6416 shape is different from training shapes, but
its DSDF profile interpolates smoothly in the space of NACA profiles. An AE trained on all
training foil shapes will embed NACA6416 somewhere in latent space — and the downstream model
can use this interpolated embedding to make better predictions than if it had no global shape
representation at all.

**Implementation plan:**

Step 1: Shape representation. For each sample, extract surface node DSDF values (x[:,:,2:4] for
fore foil, x[:,:,6:8] for aft foil). Sort by arc-length (approximately by angle around the foil).
Use an MLP set-encoder (DeepSets style, permutation-invariant) to map from surface DSDF values
to a fixed-dim shape latent:

```python
class ShapeEncoder(nn.Module):
    def __init__(self, in_dim=4, latent_dim=16, hidden=128):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(in_dim, hidden), nn.GELU(),
            nn.Linear(hidden, hidden), nn.GELU(),
            nn.Linear(hidden, latent_dim)
        )
    def forward(self, dsdf_surface):  # [M, 4] surface nodes
        node_feats = self.encoder(dsdf_surface)  # [M, latent_dim]
        return node_feats.mean(dim=0)  # permutation-invariant pooling [latent_dim]
```

Step 2: Per-batch, build shape latent:
```python
# Extract fore-foil surface DSDF for this batch
_fore_sdf = x[b, _fore_foil_mask[b], 2:6]  # [M_fore, 4]
_shape_lat = shape_encoder(_fore_sdf)  # [latent_dim]
```

Step 3: Inject into model. The simplest path: expand shape latent to [B, latent_dim], then
concatenate with `cond_2` (Re/AoA) in the AdaLN computation. The Transolver's AdaLN already
takes `cond_2 = x[:, 0, 13:15]` as a 2-dim condition. Extend this to `(latent_dim + 2)` dims by
concatenating the shape latent, and update AdaLN weight/bias MLP input dim.

New flags: `--shape_encoder` (bool), `--shape_latent_dim 16` (int).

**Alternative simpler approach (recommend trying first):** Skip the AE entirely. Instead of
learning a latent, compute a fixed-function shape summary from surface DSDF: mean, std, skewness,
kurtosis of each DSDF channel over surface nodes → 4 channels × 4 moments = 16 features.
These are differentiable and capture global shape statistics without the AE overhead.
Add `--shape_stats_cond` flag for this simpler variant.

**Experiment order:**
1. First: `--shape_stats_cond` (fixed moments, ~10 LoC, low risk)
2. If that works: `--shape_encoder` (learned latent, ~60 LoC)

**Confidence:** Moderate. Global shape conditioning is conceptually compelling and is how most
aerodynamics surrogates handle geometry generalization (CST coefficients, shape mode decomposition).
The implementation complexity is moderate. Risk: the shape AE may overfit to training shapes and
not generalize the OOD NACA6416 latent.

### 4. interfoil-channel-aug — Inter-Foil Channel Flow Virtual Samples via Coordinate Morphing

**Priority: MEDIUM-HIGH. ~35 LoC. Expected -1% to -4% p_tan. MEDIUM risk.**

**What it is:** A training-time augmentation that generates synthetic tandem configurations with
interpolated gap/stagger values by MORPHING the mesh coordinates between two real tandem samples.
Specifically, given two tandem samples (gap1, stagger1) and (gap2, stagger2), create a virtual
sample with gap = alpha*gap1 + (1-alpha)*gap2 by linearly interpolating the spatial coordinates
of ALL nodes (not just the condition scalar, which is what the existing gap/stagger aug does).

**Why this is different from existing gap/stagger aug (PR #2126, now dead end):** The existing
`aug_gap_stagger_sigma` perturbs the scalar features x[:,0,21:23] (gap and stagger) without
changing the node coordinates. This creates an inconsistency: the coordinates don't match the
perturbed gap/stagger values. The DSDF channels are computed from actual geometry, so they stay
consistent with the original coordinates, not the perturbed gap. This is physically inconsistent
and probably why the gap/stagger aug causes issues when combined with aft_foil_srf.

This new augmentation instead morphs the actual spatial coordinates of aft-foil surface and
volume nodes by translating them by (delta_gap, delta_stagger), so the ENTIRE geometry is
consistent with the interpolated configuration. The DSDF channels for the aft foil then need
to be recomputed or approximately corrected.

**Implementation sketch:**

```python
# For tandem samples only, with probability cfg.channel_aug_prob:
# 1. Sample a second tandem sample from the batch (or from a buffer)
# 2. Interpolate gap and stagger
# 3. Shift aft-foil nodes (x[:,:,0:2] for nodes where saf_norm > threshold) by delta gap/stagger
# 4. Update x[:,0,21:23] to match the new gap/stagger
# 5. The DSDF channels for aft foil (x[:,:,6:10]) need approximate correction:
#    shift the SAF distance field by the same delta vector (first-order correction)

# Approximate DSDF correction for aft-foil nodes:
_aft_node_mask = (_raw_saf_norm > 0.005) & _is_tandem.unsqueeze(1)
x[b, _aft_node_mask[b], :2] += delta_xy   # shift spatial coords
x[b, _aft_node_mask[b], 6:8] -= delta_xy  # approximate SAF correction (sign flip for signed dist)
```

**Why it might generalize OOD:** The val_tandem_transfer split has a different FRONT FOIL shape
(NACA6416), not just different gap/stagger. But augmenting the range of gap/stagger values seen
during training should make the model more robust to aft-foil geometric variation in general.

**Caution:** The DSDF correction is approximate (first-order, ignores curvature). For large
delta_xy this will introduce inconsistencies. Keep delta small (< 10% of gap size).

New flags: `--channel_aug` (bool), `--channel_aug_prob 0.3` (float), `--channel_aug_max_delta 0.05`.

**Confidence:** Moderate. The physical motivation is solid. The implementation complexity is
moderate and the DSDF correction is approximate. Recommend limiting to small deltas initially.

### 5. surf-node-dropout — Surface Node Dropout: Forcing Volume-to-Surface Generalization

**Priority: MEDIUM. ~15 LoC. Expected -1% to -3% p_tan. LOW risk.**

**What it is:** During training, randomly drop a fraction of surface nodes from the attention
computation (set their features to zero and exclude them from the surface loss). The remaining
surface nodes must then be predicted well using information from nearby volume nodes. At inference,
all surface nodes are present.

**Why it helps p_tan:** The model may be overfit to the specific surface node layout of training
foils. For the OOD NACA6416 front foil, the surface node distribution will be in different
relative positions. By randomly hiding surface nodes during training, the model is forced to
develop more robust volume-to-surface information routing. This is analogous to DropPath/Dropout
but applied to the spatial input distribution rather than the neural network connections.

**Connection to known wins:** The AftFoilRefinementHead (merged) works well precisely because
it specializes the volume-level hidden states for surface prediction. Surface node dropout would
force the trunk to develop richer intermediate representations even without the SRF head.

**Implementation:**

```python
# After building is_surface mask and before model forward (line ~1682):
if model.training and cfg.surf_node_dropout > 0.0:
    _drop_mask = torch.rand(x.shape[:2], device=device) < cfg.surf_node_dropout
    _surf_drop = is_surface & _drop_mask  # only drop surface nodes, not volume
    # Zero out features of dropped surface nodes
    x = x.clone()
    x[_surf_drop] = 0.0
    # Remove from surface mask so they don't contribute to surface loss
    is_surface = is_surface & ~_surf_drop
```

New flag: `--surf_node_dropout 0.1` (fraction of surface nodes to drop). Sweep: {0.05, 0.1, 0.2}.

**Key subtlety:** Dropped nodes still participate in the attention computation (their features are
zeroed, but they still occupy attention slots). This is MASKING not REMOVAL. The model learns
that zero-feature surface-position nodes should be interpolated from their neighborhood.

**Alternative formulation:** Instead of zeroing, add large Gaussian noise to dropped nodes' features
(noise injection). This is closer to DropConnect behavior and may be more stable.

**Confidence:** Moderate. Input dropout is well-established (Dropout in Hinton et al. 2012, Feature
Dropout in various domain adaptation works). The mesh-specific surface-only variant is novel but
the mechanism is sound. Risk: if surface nodes are the main source of accurate prediction, dropping
them may destabilize training, especially for the merged SRF heads which depend on accurate
surface hidden states.

### 6. tandem-contrastive-geom — Geometry-Conditioned Tandem Contrastive Pretraining Auxiliary

**Priority: MEDIUM. ~40 LoC. Expected -1% to -3% p_tan. MEDIUM risk.**

**What it is:** Add a contrastive auxiliary loss that pulls the hidden representations of tandem
samples with SIMILAR gap/stagger closer together in representation space, and pushes samples with
DIFFERENT gap/stagger apart. This is a geometry-conditioned InfoNCE loss applied to the global
pooled representation of tandem samples.

**Why this is different from the closed contrastive reg PR:** The earlier contrastive experiment
(confirmed dead end) was a symmetric contrastive loss across all samples. This is gap/stagger-CONDITIONED
and applies to tandem samples ONLY. The hypothesis is different: rather than general representation
smoothness, we want the model to develop a latent geometry space that is metrically meaningful —
samples close in gap/stagger space should have close representations, enabling interpolation to the
OOD NACA6416 configuration.

**Why it might help OOD:** Even though NACA6416 was never seen, if the model's tandem representation
space is organized by gap/stagger geometry, the model can interpolate to the OOD geometry's
expected hidden state by analogy with similar-gap-stagger training samples.

**Implementation:**

```python
# After trunk forward pass, during training only:
if cfg.tandem_contrastive and is_tandem_batch.any() and is_tandem_batch.sum() >= 2:
    _tan_hidden = hidden[is_tandem_batch].mean(dim=1)  # [B_tan, n_hidden] global pool
    _tan_gap_stag = x[is_tandem_batch, 0, 21:23]      # [B_tan, 2]
    # Pairwise gap/stagger similarity
    _gs_dist = torch.cdist(_tan_gap_stag, _tan_gap_stag)  # [B_tan, B_tan]
    _threshold = cfg.contrastive_gs_thresh  # e.g., 0.1 in standardized units
    _pos_mask = _gs_dist < _threshold
    _neg_mask = _gs_dist >= _threshold
    # InfoNCE: for each sample, positives = similar gap/stagger, negatives = different
    _sim = torch.mm(_tan_hidden, _tan_hidden.T) / (cfg.contrastive_temp * _tan_hidden.norm(dim=1).outer(_tan_hidden.norm(dim=1)) + 1e-8)
    _loss_ct = (- (_sim * _pos_mask.float()).sum(dim=1) / _pos_mask.float().sum(dim=1).clamp(1)
               + (_sim * _neg_mask.float()).exp().sum(dim=1).log()).mean()
    loss = loss + cfg.contrastive_weight * _loss_ct
```

New flags: `--tandem_contrastive` (bool), `--contrastive_gs_thresh 0.15`, `--contrastive_weight 0.01`,
`--contrastive_temp 0.1`. Sweep contrastive_weight: {0.005, 0.01, 0.05}.

**Confidence:** Speculative. The mechanism is theoretically sound (metric learning for geometry space
structure) but the implementation is moderately complex and the hyperparameter sensitivity is high.
The earlier contrastive reg failure is a red flag — needs careful weight tuning to avoid it
dominating the main loss.

### 7. adaptive-tandem-boost — Adaptive Tandem Loss Weight via Val-Ratio Schedule

**Priority: MEDIUM-LOW. ~10 LoC. Expected -0.5% to -2% p_tan. LOW risk.**

**What it is:** The current `adaptive_boost` (line 1793 in train.py) is computed from the TRAINING
running_tandem_loss / running_nontandem_loss ratio, clamped to [1.0, 4.0]. This is a proxy for
the true generalization gap. Instead, use the **validation ratio** p_tan / p_in (evaluated every
N epochs) as the signal for tandem upweighting: if val p_tan / p_in is high, increase tandem
weight more aggressively.

**Why it helps:** The training ratio running_tandem_loss / running_nontandem_loss may not track
the validation gap well, especially in the tandem_ramp phase where tandem samples ramp up slowly.
The actual val_tandem surface MAE is the true signal for how much the model needs tandem-focused
training. Using it directly as the adaptive weight signal creates a tighter feedback loop.

**Implementation:**

```python
# In the validation loop, compute val_ratio = val_surf_tan_mae / val_surf_in_mae
# Use this to set a global tandem_weight_scale:
if epoch % cfg.adaptive_tandem_interval == 0 and epoch > 0:
    val_ratio = last_val_metrics.get("p_tan", 30.0) / max(last_val_metrics.get("p_in", 13.0), 1e-8)
    # Map ratio [1.0, 3.0] → boost scale [1.0, 3.0]
    cfg_tandem_boost_scale = min(3.0, max(1.0, val_ratio))

# In training loop, multiply adaptive_boost by cfg_tandem_boost_scale
adaptive_boost = max(1.0, min(4.0, running_tandem_loss / max(running_nontandem_loss, 1e-8))) * cfg_tandem_boost_scale
```

New flags: `--adaptive_tandem_val` (bool), `--adaptive_tandem_interval 10` (eval every N epochs).

**Distinction from loss reweighting (known dead end):** Previous loss reweighting was a FIXED
multiplier. This is a DYNAMIC schedule driven by the observed generalization gap — much more
nuanced. It amplifies what the training loop already does adaptively rather than replacing it.

**Confidence:** Moderate-low. The mechanism is sound (feedback control of loss weight) but the
validation-driven adjustment may cause training instability if the val metric is noisy. The
benefit is likely small but the risk is also low.

**Why this is #1:** p_tan is uniquely driven by OOD NACA6416 front foil shape. The physically
most informative thing about the aft-foil wake is WHERE each aft-foil node sits relative to the
fore-foil trailing edge — this is the source of the inter-foil jet. Currently the model has NO
coordinate frame that encodes this. Adding it requires ~20 LoC, works synergistically with the
existing aft_foil_srf head, and is a physically motivated inductive bias backed by CFD theory.
Analogous ideas (wake-relative frames) are standard in analytical aerodynamics.

**Implementation risk:** Moderate. The TE estimation (max x-coord among fore-foil surface nodes)
is a proxy that works for NACA series foils. The 2 extra features are physically scaled (chord
units), no standardization needed. The only edge case is non-tandem samples (all zeros, fine).

### Pick 2: `tandem-selfdistill` (Online EMA Self-Distillation on Tandem Samples)

**Why this is #2:** The EMA model is already computed every batch for evaluation, but is never
used as a training signal. On tandem samples (which are both rarer and OOD), the EMA prediction
is measurably smoother than the noisy student. Using the EMA as a teacher specifically for tandem
samples adds a second, higher-quality training signal at near-zero extra cost (EMA is already on
device). This is conceptually the cleanest idea on the list. ~25 LoC, very low memory overhead.

**Implementation risk:** Low. The EMA model forward pass runs on tandem subset only. Key caution:
use `torch.no_grad()` for the teacher pass and don't backprop through it.

### Pick 3: `foil-shape-ae` simple stats variant — `shape_stats_cond`

**Why this is #3:** Global shape statistics (mean/std/skewness/kurtosis of DSDF over surface nodes)
are a zero-AE-training way to give the model a global shape fingerprint. In ~10 LoC, this injects
16 features (4 DSDF channels × 4 moments) into the AdaLN condition. For OOD NACA6416, these
statistics will be in-distribution even if the shape is not directly trained on. Start with this
simpler variant before the full AE. The risk is that fixed statistics don't capture shape in
the right basis for flow generalization — but this is the minimal-code test of the hypothesis.
