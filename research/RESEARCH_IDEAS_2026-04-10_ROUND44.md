#### Idea #1: HyPINO Hypernetwork — Per-Sample SRF Weight Generation
**Category:** Architecture (hypernetwork)  
**Priority:** 1 — Morgan requested, strong OOD generalization case  
**Risk:** Medium

**Hypothesis:**
The Surface Refinement Head (SRF) uses global AdaLN conditioning (--adaln_output) on Re/AoA, but this is an affine modulation applied after the fact. A hypernetwork that generates the *weights* of the SRF MLP from the global flow parameters (Re, AoA, gap, stagger) can adapt the entire function, not just its scale/bias. This is the core insight of HyPINO (Zhao et al., NeurIPS 2025 Spotlight): the hypernetwork directly generates the target network's weight matrices via low-rank decomposition. For tandem configurations, the gap and stagger ratio are the most variable parameters that the current SRF cannot adapt to — a hypernetwork that sees these inputs can produce completely different pressure field computations per-sample, which is exactly what p_tan needs.

**Implementation (changes to train.py):**

Add a new flag `--hyper_srf`. When enabled:

1. Construct a small "hyper-encoder" that takes the global flow condition vector `[Re, AoA, gap, stagger, is_tandem]` (5 scalars) and outputs low-rank weight factors for the SRF.
2. The SRF becomes a "dynamic MLP" where each linear layer's weight matrix is `W = W_base + U @ V` where `U` (n_hidden × r) and `V` (r × n_hidden) are generated by the hyper-encoder. Use rank r=8 to keep parameter count small.
3. The hyper-encoder itself is a 2-layer MLP (5 → 64 → 2 * n_srf_layers * n_hidden * r), initialized so U=0 (base weights dominate at init).
4. Only apply to the SRF (--surface_refine path), not the backbone — this keeps complexity contained.

```
# Suggested flags
--hyper_srf --hyper_srf_rank 8 --hyper_srf_hidden 64
```

Use the same base command as the baseline (all existing flags). The hypernetwork adds ~200K parameters (trivial vs. 96GB VRAM).

**Expected impact:** p_tan improvement (tandem gap/stagger sensitivity is the axis of p_tan variance). p_re may also improve (Re generalization). Neutral on p_in/p_oodc.

**Key reference:** HyPINO — "Hypernetwork for Physics-Informed Neural Operators" (Zhao et al., NeurIPS 2025, ETH Zurich). Also: Cho et al. 2023 arXiv:2310.09528 "Hypernetwork-based Meta-Learning for Low-Rank Physics-Informed Neural Adaptation."

#### Idea #2: Multi-Scale Slice Attention v2 — Hierarchical Slice Aggregation
**Category:** Architecture (attention mechanism)  
**Priority:** 2 — Morgan requested; prior attempt (PR #2373) used additive mixing which failed  
**Risk:** Medium

**Hypothesis:**
PR #2373 (Multi-Scale Slice Attention) failed because it added the coarse and fine slice outputs additively, creating gradient interference. The correct formulation is *hierarchical*: first compute coarse-scale slice tokens (few large physics zones) to build global context, then refine with fine-scale slices (many local physics patches) that attend to the coarse tokens as context. This mirrors the Swin Transformer's shifted-window hierarchy but in physics-state space rather than image space. The coarse slices capture tandem inter-foil coupling (the main source of p_tan error), while fine slices capture local pressure peaks at leading/trailing edges.

**Implementation:**

Add `--multiscale_hierarchical` flag. In `Physics_Attention_Irregular_Mesh`, instead of parallel coarse+fine, use a two-stage attention:

1. Stage 1 (coarse): 16 slice tokens, standard Transolver attention → output coarse_tokens [B, H, 16, D].
2. Stage 2 (fine): 96 slice tokens (current default), but Q attends to own slice tokens while K,V come from coarse_tokens concatenated with fine slice tokens. This is a form of cross-scale attention.
3. The coarse stage uses a separate but smaller projection: `in_project_slice_coarse = nn.Linear(dim_head, 16)`.
4. Only apply to the LAST Transolver layer (n_layers=3; only layer 3 is hierarchical). Early layers use standard 96-slice attention.

This differs from PR #2373 (parallel additive) and avoids the gradient interference.

```
# Suggested flags
--multiscale_hierarchical --coarse_slices 16
```

**Expected impact:** Better global context for tandem inter-foil pressure communication. Target: p_tan -3% to -5%.

#### Idea #3: Hypernetwork Meta-Conditioning of ALL Transolver Layers (mHC)
**Category:** Architecture (global conditioning)  
**Priority:** 3 — Morgan requested; complements Idea #1 which targets only SRF  
**Risk:** Medium-High

**Hypothesis:**
The current AdaLN (--adaln_output) conditions only the output projection. FiLM-style conditioning at every TransolverBlock layer has been tried and merged (DomainLayerNorm). The next step is a *hypernetwork* that generates all FiLM scale/shift vectors simultaneously from a richer conditioning signal — not just Re/AoA but also learned latent embeddings of geometry. This is the "mHC" (meta-Hypernetwork Conditioning) idea from Morgan. Unlike affine conditioning, the hypernetwork can generate *different* conditioning for each layer based on the same input, learning which layers need to adapt for tandem vs. single-foil vs. OOD-Re.

**Implementation:**

Add `--hyper_film_backbone` flag. Build a "global context encoder" (separate from backbone):

1. Pool the input features globally: mean-pool x_embed across all nodes → [B, n_hidden].
2. Concatenate with scalar flow params: [Re, AoA, gap, stagger, is_tandem] → [B, n_hidden+5].
3. Project via 2-layer MLP → [B, n_layers * 2 * n_hidden] (scale + shift for each layer).
4. At each TransolverBlock, after the standard attention+FFN, apply layer-specific FiLM: `out = out * (1 + scale_l) + shift_l`.
5. Initialize scale=0, shift=0 so initial behavior is identical to baseline.

This is different from DomainLayerNorm (which uses a hard is_tandem binary split) — the hypernetwork can interpolate and generate continuous conditioning.

```
# Suggested flags  
--hyper_film_backbone --hyper_film_hidden 128
```

**Expected impact:** Better OOD generalization (p_oodc, p_re). Potential p_tan improvement by allowing the backbone to adapt its computation to tandem geometry parameters.

#### Idea #4: GeoMPNN-Inspired Latent Surface Graph with Bipartite Volume Propagation (GeoSolver)
**Category:** Architecture (graph neural network decoder)  
**Priority:** 4 — Morgan requested; NOTE: previous "GeoSolver" PRs #2009/#2027 failed but used only geometry features, not the full architectural approach  
**Risk:** High (significant implementation)

**Hypothesis:**
The failed GeoSolver PRs (#2009, #2027) only added geometry-derived input features. The actual GeoMPNN contribution (Helwig et al., NeurIPS 2024 ML4CFD Best Student Paper, arXiv:2412.09399) is architectural: build a *latent graph* exclusively on airfoil surface nodes with surface-specific message passing, then use bipartite directed propagation from surface to volume nodes. The surface graph has a direct inductive bias for boundary layer pressure distribution. The key differences from the baseline: (1) surface nodes form their own graph with arc-length-weighted edges, (2) surface→volume message passing carries learned "influence" from the surface into the volume field. This addresses the core known limitation in train.py: "SURFACE_IDS=(5,6) misses boundary ID 7 (foil 2 surface in tandem data). Tandem surface loss is therefore underweighted." A surface-latent-graph explicitly models the foil-2 surface even without the correct boundary ID.

**Implementation:**

Add `--geo_surface_graph` flag. After the Transolver backbone:

1. Identify surface nodes using existing `is_surface` mask.
2. Build a fixed k-NN graph (k=8) among surface nodes using their (x,y) coordinates.
3. Apply 2 rounds of message passing (GCN-style) on the surface graph: `h_surf[i] = MLP(h_surf[i] || mean_j∈N(i) h_surf[j])`.
4. For each volume node, compute attention-weighted aggregation from the nearest K=4 surface nodes (bipartite: volume attends to surface).
5. Add the aggregated surface context to the volume node's hidden state before the output projection.

Use sparse operations; the surface graph is small (~500 nodes per foil surface) so it's cheap.

```
# Suggested flags
--geo_surface_graph --geo_surf_knn 8 --geo_surf_mp_layers 2 --geo_vol_surface_knn 4
```

**Expected impact:** p_tan could improve significantly if surface-to-surface message passing enables better fore→aft pressure communication. The aft foil surface (foil-2) is the main source of p_tan error.

#### Idea #5: Stagnation Point Pressure Constraint
**Category:** Physics constraint  
**Priority:** 5 — extends the proven Kutta TE constraint direction (PR #2374 beat baseline)  
**Risk:** Low-Medium

**Hypothesis:**
Kutta TE constraint (PR #2374) showed p_oodc=-2.9% — proving that physics constraints flowing INTO the backbone outperform physics features added ON TOP. The next natural physics constraint is the stagnation point condition: at the leading edge stagnation point, Cp = 1.0 exactly (by definition, for the inviscid incompressible limit). The neural network currently has no signal forcing Cp ≈ 1 at the stagnation point. Adding a soft constraint: `L_stag = ||p_pred(x_stag) - 0.5*rho*Umag^2||` (in physical units) directly suppresses leading-edge pressure errors. For tandem configurations, both fore and aft foil stagnation points must satisfy this constraint — adding two separate stagnation constraints may force the model to correctly resolve the aft-foil stagnation which is the main source of p_tan error.

**Implementation:**

Add `--stagnation_constraint --stagnation_weight 0.5` flags:

1. Find stagnation node per foil: the surface node with maximum predicted Cp (or, more robustly, the surface node closest to the theoretical stagnation point given AoA — use `x_coord * cos(AoA) + y_coord * sin(AoA)` as a scoring function to find the leading-edge node facing the flow).
2. Constraint loss: `L_stag = mean(|p_pred[stag_node] / (0.5 * Umag^2) - 1.0|)` over batch.
3. Compute separately for fore and aft foil. Set aft stagnation weight to 2× fore (since aft foil is harder).
4. Add to total loss: `loss = loss_main + stagnation_weight * L_stag`.

Combine with Kutta TE constraint if that PR is merged, otherwise test independently.

```
# Suggested flags
--stagnation_constraint --stagnation_weight 0.5 --stagnation_aft_weight_mult 2.0
```

**Expected impact:** p_tan, p_in improvement. Low risk — constraint adds information without replacing anything.

#### Idea #6: Bernoulli Velocity-Pressure Consistency Constraint
**Category:** Physics constraint  
**Priority:** 6 — tests the Bernoulli equation as a soft constraint  
**Risk:** Low

**Hypothesis:**
The model predicts Ux, Uy, and p jointly but has no explicit loss enforcing the Bernoulli relationship: `p + 0.5*rho*(Ux^2 + Uy^2) = const` along streamlines. In the irrotational inviscid approximation, this constant equals `0.5*rho*Umag^2` everywhere (total pressure conservation). A soft Bernoulli loss penalizes inconsistencies between the velocity and pressure predictions: `L_bernoulli = ||p_pred + 0.5*(Ux_pred^2 + Uy_pred^2) - 0.5*Umag^2||^2`. This is a *multi-output consistency constraint* — the only kind of constraint that connects velocity and pressure predictions together. Current PCGrad only handles gradient conflicts between dataset splits, not between output channels.

**Implementation:**

Add `--bernoulli_constraint --bernoulli_weight 0.1` flags:

1. After computing the raw predictions (before asinh denormalization), compute: `q_pred = 0.5 * (Ux_pred^2 + Uy_pred^2)` and `L_bern = mean(|p_pred + q_pred - 0.5 * Umag^2|)` on surface nodes only.
2. Scale by `--bernoulli_weight` (start at 0.1; if training becomes unstable, reduce to 0.01).
3. Apply only to surface nodes (volume nodes have viscous effects that break Bernoulli).
4. Combine with PCGrad: add as a fourth PCGrad component alongside the three dataset-split losses.

Note: Bernoulli is strictly inviscid. The viscous correction is of order Re^-1/2 in the boundary layer, which is ~5% for Re=500K. This is the tolerance the model needs to beat, so Bernoulli as a SOFT constraint is appropriate — it provides signal without overconstrained physics.

```
# Suggested flags
--bernoulli_constraint --bernoulli_weight 0.1
```

**Expected impact:** Better velocity-pressure consistency especially on volume nodes. Potential surface improvement from cleaner representations.

#### Idea #7: Circulation-Conserving Tandem Pressure Normalization
**Category:** Physics features / loss formulation  
**Priority:** 7  
**Risk:** Low-Medium

**Hypothesis:**
The Kutta-Joukowski theorem states that lift L = rho * Umag * Gamma, where Gamma is the circulation (integral of velocity around the airfoil). For tandem configurations, the total circulation of the fore foil modifies the effective angle of attack seen by the aft foil via the downwash velocity. This is the main inter-foil interaction that makes p_tan hard. Currently the model sees only static geometry features for the aft foil; it has no explicit signal about the *expected circulation interaction*. We can compute an approximate circulation from the panel method velocity field (already available via --vortex_panel_velocity): `Gamma_fore = sum_panel(vortex_panel_strength * panel_length)`. Adding this as a scalar conditioning feature gives the model information about the strength of the fore foil's wake effect on the aft foil.

**Implementation:**

Add `--kj_circulation_feature` flag:

1. In the input feature construction, compute the panel-method-derived total circulation for the fore foil: sum over panels of `(gamma_panel * ds_panel)` where `gamma_panel` is the vortex strength from the vortex panel solve (already available in the pipeline).
2. Normalize by `Umag * chord` to get a dimensionless circulation coefficient `cl_panel ≈ Gamma / (0.5 * Umag * chord)`.
3. Append this scalar to the global conditioning vector alongside Re, AoA, gap, stagger.
4. For tandem samples: compute separate fore and aft circulation coefficients. For single-foil: aft=0.

The circulation coefficient is a richer signal than AoA alone because it captures the nonlinear lift saturation and the actual wake strength impacting the aft foil.

```
# Suggested flags
--kj_circulation_feature
```

**Expected impact:** p_tan improvement by providing explicit inter-foil wake coupling information.

#### Idea #8: Learned Arc-Length Positional Encoding for Surface Nodes
**Category:** Architecture (positional encoding)  
**Priority:** 8  
**Risk:** Low

**Hypothesis:**
The Wavelet-Fourier PE (a major merged breakthrough) applies to all nodes. However, surface nodes have a natural 1D ordering: arc-length from the leading edge, going clockwise. The current PE treats surface nodes like any other 2D-coordinate nodes. A *surface-specific arc-length PE* would encode this 1D topology explicitly, helping the model learn smooth Cp distributions along the surface. This is analogous to how 1D sinusoidal PE in transformers enables smooth sequence understanding — but for the airfoil surface. The arc-length PE is orthogonal to the 2D spatial PE (they capture different geometric information) and can be added as an additional feature without replacing anything.

**Implementation:**

Add `--arc_length_pe --arc_pe_dim 16` flags:

1. For surface nodes, compute arc-length from the leading edge (min-x node) in the positive (clockwise) direction using cumulative sum of inter-node distances along the surface boundary.
2. Normalize arc-length to [0, 1] per foil.
3. Encode with `n` Fourier features: `[sin(2pi*k*s), cos(2pi*k*s)]` for k=1..8 → 16-dim feature.
4. Concatenate to the input feature vector for surface nodes only (zero-pad volume nodes).
5. For tandem: compute separately for fore and aft foil, using different frequency offsets to distinguish them.

The arc-length PE is cheap (no learned parameters beyond the existing input projection) and directly addresses the surface node ordering that the 2D PE cannot capture.

```
# Suggested flags
--arc_length_pe --arc_pe_dim 16
```

**Expected impact:** Smoother surface Cp predictions, fewer leading/trailing edge oscillations. Target p_in, p_tan improvement.

#### Idea #9: Tandem Configuration Augmentation via Gap/Stagger Interpolation
**Category:** Data augmentation  
**Priority:** 9  
**Risk:** Low-Medium

**Hypothesis:**
The current data augmentation (--re_stratified_sampling) addresses Re distribution imbalance. But for p_tan, the main source of variance is gap and stagger ratio. If the training set has sparse coverage of certain gap/stagger combinations, the model will extrapolate poorly to unseen configurations (which is exactly what val_tandem_transfer tests). Mixup-style interpolation between nearby tandem configurations in gap/stagger space would create synthetic training samples covering the gaps. This is different from FFD geometry augmentation (PR #2365, failed due to label mismatch) because we are mixing *existing labeled samples* with each other — no need for re-simulation.

**Implementation:**

Add `--tandem_gap_stagger_mixup --mixup_alpha 0.2` flags:

1. During training, for each tandem batch, find pairs of samples with similar AoA and Re but different gap/stagger.
2. Apply Mixup: `x_mix = lambda * x_a + (1-lambda) * x_b` and `y_mix = lambda * y_a + (1-lambda) * y_b`, where `lambda ~ Beta(alpha, alpha)`.
3. IMPORTANT: Only mix the node *features* (physics inputs) and *labels*, not the mesh topology. This is valid because all tandem samples share the same mesh topology (tandem mesh is fixed).
4. Only apply to tandem samples (is_tandem=True); do not mix tandem with single-foil.
5. Use `alpha=0.2` (weak mixing to stay near valid configurations).

This directly augments the sparse gap/stagger space that p_tan is sensitive to.

```
# Suggested flags
--tandem_gap_stagger_mixup --mixup_alpha 0.2
```

**Expected impact:** p_tan improvement via better interpolation between gap/stagger configurations. May hurt p_in slightly (don't mix single-foil samples).

#### Idea #10: Boundary Layer Thickness Feature via Re-Dependent Scaling
**Category:** Physics features  
**Priority:** 10  
**Risk:** Low

**Hypothesis:**
The boundary layer thickness scales as `delta ~ chord / sqrt(Re)`. At Re=500K, delta/chord ≈ 0.0014. At Re=4.4M, delta/chord ≈ 0.00047. For surface nodes, the distance from the wall determines whether a node is inside or outside the boundary layer — information the model currently lacks (it only knows Re globally). Adding a per-node "boundary layer significance" feature `bl_feature = exp(-d_wall / delta_BL)` where `d_wall` is the distance-to-wall feature (already available in the surface-adjacent flag) and `delta_BL = chord / sqrt(Re)` explicitly tells the model which nodes are in the viscous-dominated region. This could resolve the Re-dependent pressure gradient at the aft foil surface that drives p_re errors.

**Implementation:**

Add `--bl_thickness_feature` flag:

1. Compute `delta_BL = 1.0 / sqrt(Re_value)` (normalized units, since chord is already normalized to 1).
2. Compute `d_wall` as the minimum distance to any surface node (this is effectively what `sdf` already provides in the input features).
3. Add feature `bl_sig = exp(-sdf_normalized / delta_BL)` for all nodes. For surface nodes, sdf≈0 → bl_sig≈1.0. For far-field nodes, sdf>>delta_BL → bl_sig≈0.
4. Concatenate as a single additional input feature dimension.

This is computationally trivial and uses existing information (sdf is already computed). The innovation is the Re-dependent normalization that makes the feature physically meaningful.

```
# Suggested flags
--bl_thickness_feature
```

**Expected impact:** p_re improvement (OOD Reynolds number). Potentially p_tan improvement for viscous interactions near the aft foil surface.

#### Idea #11: Mirror Symmetry Augmentation for Zero-AoA Single-Foil Cases
**Category:** Data augmentation  
**Priority:** 11 — was in "Never Ran" queue as PR #2300  
**Risk:** Very Low

**Hypothesis:**
For single-foil cases at AoA=0°, the flow field is symmetric about the chord line. Reflecting the mesh and labels about the y=0 axis gives a valid additional training sample at AoA=0°. This doubles the effective training data for zero-AoA cases and directly constrains the model to predict symmetric pressure distributions for symmetric inputs. PR #2300 was never run. This is the lowest-risk idea in Round 44 and should be attempted as a sanity check.

**Implementation:**

Add `--mirror_aoa_zero_aug` flag:

1. During training, for samples where `|AoA| < 1.0°`, add a mirrored copy: negate all y-coordinates, negate Uy, keep Ux and p unchanged.
2. Include mirrored samples in the same batch (doubles these samples in the batch).
3. Only for single-foil samples (mirroring tandem configurations changes the geometry semantics).

```
# Suggested flags
--mirror_aoa_zero_aug
```

**Expected impact:** Small p_in improvement from better zero-AoA generalization. Very low risk.

#### Idea #12: Effective Angle of Attack Feature for Aft Foil
**Category:** Physics features  
**Priority:** 12 — was in "Never Ran" queue as PR #2287  
**Risk:** Low

**Hypothesis:**
The aft foil in a tandem configuration experiences a modified effective AoA due to the downwash from the fore foil wake. This effective AoA is `AoA_eff = AoA - arctan(w_downwash / Umag)` where `w_downwash` is the wake-induced velocity at the aft foil leading edge position. We can estimate `w_downwash` from the vortex panel velocity (already computed via --vortex_panel_velocity) evaluated at the aft foil stagnation point. Adding this as a scalar feature encodes the critical inter-foil coupling that makes tandem pressure prediction harder than single-foil.

**Implementation:**

Add `--effective_aoa_aft_feature` flag:

1. From the vortex panel velocity field (already computed), extract the induced velocity `[Ux_panel, Uy_panel]` at the aft foil stagnation point (leading edge node with saf_norm > 0.005, minimum x-coord).
2. Compute: `Ux_eff = Umag + Ux_panel`, `Uy_eff = Uy_panel`. Then `AoA_eff = arctan(Uy_eff / Ux_eff)`.
3. Add `(AoA_eff - AoA) / pi` as a scalar global conditioning feature.
4. For single-foil samples: set to 0.

```
# Suggested flags
--effective_aoa_aft_feature
```

**Expected impact:** p_tan improvement by explicitly encoding the fore→aft wake coupling that the model must currently infer from geometry alone.

#### Idea #13: Progressive Layer-Wise Learning Rate (LLRD) for Fine-Tuning
**Category:** Optimization  
**Priority:** 13  
**Risk:** Low

**Hypothesis:**
Layer-wise learning rate decay (LLRD) — applying smaller learning rates to earlier layers — is standard in fine-tuning pretrained transformers (BERT, ViT). For Transolver, the lower layers capture general physics representations while upper layers adapt to specific geometries. Using LLRD means: layer 3 (output) gets full lr=2e-4, layer 2 gets lr=1e-4, layer 1 gets lr=5e-5. This prevents catastrophic forgetting of early-layer representations when training on the scarce tandem data. It's a training optimization, not an architecture change — very low risk.

**Implementation:**

Add `--llrd --llrd_decay 0.5` flags:

1. Create separate param groups for each TransolverBlock layer.
2. Set lr multipliers: `layer_k_lr = base_lr * (llrd_decay ** (n_layers - 1 - k))`.
3. For n_layers=3, decay=0.5: layer0_lr=0.5e-4, layer1_lr=1e-4, layer2_lr=2e-4.
4. SRF and head parameters use base_lr.

```
# Suggested flags
--llrd --llrd_decay 0.5
```

**Expected impact:** Small but consistent improvement across all metrics. Particularly likely to help p_oodc and p_re (OOD generalization benefits most from stable early representations).

## Strategic Notes

**Why p_tan remains so hard:** The val_tandem_transfer split tests configurations where the fore foil geometry is unseen during training. The model must generalize across fore-foil shape × gap × stagger × AoA interactions. The Transolver's slice attention pools globally across all mesh nodes — it has no inductive bias that specifically models the fore→aft pressure communication. Ideas #1, #2, #4 all attack this from different angles.

**Why physics constraints keep winning:** Kutta TE (PR #2374, p_oodc -2.9%) proved that constraints flow gradients INTO the backbone, shaping all representations. Features and conditioning only add signals ON TOP of existing representations that the model must choose to use. Ideas #5 and #6 extend this proven strategy.

**What to avoid:** All decoder replacements (B-GNN, FNO, multi-scale) have failed. The Transolver + SRF is a strong local optimum. Changes should inject signals INTO the existing backbone or add lightweight modules ON TOP (hypernetworks, constraints), not replace the backbone itself.

**Assign priority order:** Given 8 idle students, assign Ideas #1-#8. Hold #9-#13 for Round 45 or as fallbacks if early experiments finish quickly.
