## Ranked Hypotheses

### 1. SWAD — Dense Checkpoint Averaging from Validation Plateau

**Priority: HIGH. Flag `swad=True` exists in train.py, NOT in dead-ends, distinct from snapshot ensemble (#2028).**

SWAD (Cha et al., NeurIPS 2021) averages checkpoints collected from within a stable validation-loss plateau — the phase where val_loss has stopped improving but hasn't diverged. This is fundamentally different from snapshot ensemble (#2028, DEAD END), which averaged checkpoints taken at each LR cycle restart. SWAD averages within ONE stable descent into a flat basin; snapshot ensemble averaged across multiple different basins. The mechanism is that the flat basin average lands in a region of lower curvature with better generalization.

The infrastructure in `train.py` is already implemented: `swad`, `swad_initial_val`, `swad_prev_val`, `swad_checkpoints`, `swad_collecting`, `swad_done`. Collection triggers when val_loss enters a stable window.

**Why it might help:** The existing 8-seed ensemble shows variance reduction is the single biggest lever we have found. SWAD gives a single model similar benefit in weight space without needing separate seeds, which means it compounds with (rather than replaces) the ensemble strategy.

**Key paper:** Cha et al., "SWAD: Domain Generalization by Seeking Flat Minima", NeurIPS 2021.

**Experiment design:**
```
--swad True
--scheduler_type sequential
--seed 42
```
Run single seed with SWAD, compare to single-seed p_in for seed 42. Do NOT compare to the 8-seed ensemble — single-seed SWAD has a different reference. Use `--wandb_group phase6/swad-single-seed`.

**Implementation note:** The existing collection window triggers at `val_loss < swad_initial_val * 1.01`. If the window is too narrow (e.g., only 1–2 checkpoints collected), SWAD degenerates to a single model. The student should log `swad_checkpoints` count at the end of training to confirm collection is happening. If fewer than 5 checkpoints are collected, try adding `--swad_start_epoch 130` to force earlier collection in the final cosine descent.

**Confidence:** Strong from domain generalization literature. The mechanism (flat-basin averaging) is orthogonal to Lion/EMA and does not conflict with any active experiment.

### 2. Multi-Scale Output Supervision via Intermediate Hidden Features

**Priority: HIGH. Model exposes `hidden` (fx_deep) in output dict. ~15 lines of code, not tried.**

The model's forward pass saves `fx_deep = fx` just before the final TransolverBlock and returns it in `model_out["hidden"]`. This intermediate representation at `[B, N, n_hidden]` receives gradients only indirectly, through the chain from the final layer's loss.

Adding an auxiliary head that directly predicts surface targets from `fx_deep` creates an explicit gradient path into the pre-final-block representations. This is standard deep supervision (HED, nnUNet, multi-scale FNO) applied to the Transolver's intermediate layers.

The 3-block Transolver is relatively shallow. Direct supervision of intermediate layers forces the physics attention in blocks 1–2 to produce spatially discriminative representations, not just pass information through to block 3.

**Experiment design:**
```python
# In train.py, after model_out is returned:
if cfg.aux_supervision and "hidden" in model_out:
    aux_pred = model.aux_head(model_out["hidden"])  # [B, N, 3]
    aux_loss = cfg.aux_weight * F.l1_loss(
        aux_pred[surface_mask], targets[surface_mask]
    )
    loss = loss + aux_loss

# aux_head definition in Transolver.__init__:
self.aux_head = nn.Sequential(
    nn.Linear(n_hidden, n_hidden // 2), nn.GELU(),
    nn.Linear(n_hidden // 2, 3)
)
```

Flags: `--aux_supervision True --aux_weight 0.15`

The auxiliary head should NOT be used at inference. The `surface_mask` focuses the auxiliary gradient on the nodes that matter.

**Why it might help:** p_tan=29.1 is by far the weakest metric. Tandem foil geometry has qualitatively different pressure distributions (inter-foil slot flow, wake-foil leading edge stagnation). Deep supervision forces earlier layers to represent these local surface features explicitly, not rely on the final layer to extract them.

**Confidence:** Moderate-strong. Deep supervision reliably improves dense prediction tasks (segmentation, depth estimation) where early layers need to capture local structure. Not previously validated in Transolver-based surrogates.

### 3. Asymmetric Pressure Loss — Separate Weights for Over/Under Prediction

**Priority: HIGH. 3-line code change, directly targets p_tan which has physical asymmetry.**

The current loss treats over-prediction and under-prediction of pressure equally (L1). But in aerodynamics, under-predicting suction peaks (negative pressure coefficient) is physically more consequential than over-predicting: suction peak magnitude drives lift, drag, and separation onset. The suction surface of both airfoils in the tandem configuration is the primary discriminator between in-distribution and OOD conditions.

An asymmetric loss applies a higher weight to cases where the model under-predicts the pressure magnitude (i.e., predicted p > true p when true p < 0, or predicted p < true p when true p > 0):

```python
# Asymmetric pressure loss
residual = pred_p - target_p_transformed
alpha_over = 1.0   # weight on over-prediction
alpha_under = cfg.asym_weight  # e.g., 1.5–2.0, weight on under-prediction
weights = torch.where(residual > 0, alpha_over, alpha_under)  # shape [B, N]
asym_loss = (weights * residual.abs()).mean()
```

Flag: `--asym_pressure_weight 1.5` (ratio of under:over penalty)

**Why it might help:** The asinh transform already compresses the pressure range, but it does so symmetrically. Asymmetric loss additionally biases the model toward accurate suction peak prediction, which is the physically critical region for aerodynamic performance. p_tan=29.1 dominates our weakness — tandem foils have much stronger inter-foil aerodynamic interaction (Venturi-like slot flow) that creates extreme suction peaks on the aft element's leading edge.

**Implementation note:** The asymmetric loss should be applied AFTER the asinh transform (on the transformed pressure, not the raw pressure), so the asymmetry is in transformed space. The appropriate `alpha_under` needs a sweep: try 1.3, 1.5, 2.0. Start with 1.5 as the default.

**Confidence:** Moderate. Asymmetric loss functions (pinball loss, quantile regression) are well-established for skewed targets. The application to pressure suction peaks is physically motivated and novel in this setting. Similar to focal loss in object detection — focuses capacity on hard negatives.

### 4. Input Feature: Normalized Inter-Foil Gap and Stagger

**Priority: MEDIUM-HIGH. Adds 2–4 explicit geometric features that tandem prediction currently lacks.**

The 24-dim input features encode geometry (NACA 4-digit parameters, chord, AoA) and flow conditions (Re). For tandem configurations, the critical geometric parameters are the inter-foil gap (streamwise distance between trailing edge of foil 1 and leading edge of foil 2) and stagger (vertical offset). These determine whether the aft foil is in the wake of the fore foil and the strength of the slot flow.

Currently, the tandem geometry is implicitly encoded through the foil-2 SDF and boundary features (introduced in `prepare_multi.py`). But explicit scalar features for gap/stagger/overlap would give the model a direct low-frequency conditioning signal without requiring it to infer these from the SDF.

**Experiment design (requires train.py modification):**
```python
# New features to add to the 24-dim input (in prepare_multi.py — read only, 
# so compute from existing foil-2 features in train.py preprocessing):
# - gap_normalized: (x_le_foil2 - x_te_foil1) / chord_foil1  (streamwise gap)
# - stagger_normalized: (y_le_foil2 - y_te_foil1) / chord_foil1  (vertical stagger)
# These can be derived from the existing per-node foil-2 distance features.
```

However, since `prepare_multi.py` is read-only, the gap/stagger must be computed from the existing features in `train.py`. If the 24-dim input already includes foil-2 position scalars (check `prepare_multi.py` carefully), simply adding a learned linear projection of these scalars as an additional conditioning term to the AdaLayerNorm (alongside Re, AoA) would be the cleanest approach.

**Simpler alternative (no data modification):** Add a learned embedding for the inter-foil configuration as a domain flag: single-foil (gap=inf) vs. tandem with small/medium/large gap. Discretize gap into 3 bins. This is purely in `train.py`.

**Why it might help:** p_tan=29.1 — the tandem split is nearly 2.5x worse than p_in=12.2. This is the largest remaining gap in our metrics. Explicit gap/stagger conditioning gives the model a direct geometric handle on the slot flow regime, which determines pressure distribution shape on the aft foil.

**Confidence:** Moderate. Feature engineering has shown diminishing returns in Phase 3 R6 (advanced feature engineering PRs), but those were general flow features. Gap/stagger are specific to the tandem configuration weakness and haven't been tried explicitly.

### 5. Asinh Transform on Velocity Channels (Ux, Uy)

**Priority: MEDIUM. Extension of the successful pressure asinh to the velocity channels. Not tried.**

The current asinh transform applies only to the pressure channel (index 2): `asinh(p * 0.75)`. Ux and Uy are untransformed. However, wake velocity also has concentration effects: boundary layer nodes near the surface have steep velocity gradients (from zero-slip to freestream), while far-field nodes have nearly uniform velocity. The L1 loss on raw velocity is therefore dominated by the near-field high-gradient regions regardless of physical importance.

Asinh compression on velocity would reduce the effective dynamic range of the velocity loss, redistributing gradient weight more uniformly across the mesh. This is the same argument that motivated the pressure asinh — the velocity channels just haven't received the same treatment.

**Experiment design:**
```
--asinh_velocity True
--asinh_velocity_scale 0.3
```

(Separate scale from pressure, since velocity range differs. Start at 0.3 — smaller than 0.75 — since freestream velocity is already O(1) in normalized units.)

Alternatively, apply the same 0.75 scale to all 3 channels and compare.

**Why it might help:** If velocity residuals in high-gradient near-wall regions dominate the loss, the model may be under-fitting pressure in favor of velocity accuracy. Asinh on velocity could rebalance this, indirectly improving pressure prediction. Effect should be most visible in p_re (Reynolds OOD) since Re affects boundary layer thickness.

**Implementation note:** The asinh transform on velocity must be applied consistently to both training targets AND evaluation inverse-transform. Double-check that the eval code correctly inverts the velocity transform before computing surface MAE. A bug here would cause inflated-looking metrics that don't correspond to real improvement.

**Confidence:** Moderate. The mechanism is sound but velocity has directional structure that pressure lacks. Ux and Uy in the freestream far-field are already normalized to O(1), so the dynamic range issue may be less severe than for pressure.

### 6. Loss-Weighted Curriculum via Per-Sample Difficulty Tracking

**Priority: MEDIUM. Dynamic sample reweighting based on running average loss per sample.**

Our training set has 1,322 samples spanning a wide range of AoA (-5° to 25°), Re (1e5 to 2e6), and configurations (single/tandem). The model learns all samples with equal weight, but some configurations are likely harder and consistently high-error.

**Hard example mining:** Track a running exponential moving average of per-sample loss across epochs. Samples with persistently high loss receive higher weight in the loss (NOT in sampling — all samples still appear every epoch, but their loss contribution is upweighted). This is online hard example mining (OHEM), adapted from object detection.

```python
# In train.py training loop:
# Initialize: sample_ema_loss = torch.ones(N_samples) * initial_loss
# Each step:
with torch.no_grad():
    per_sample_loss = F.l1_loss(pred, target, reduction='none').mean(dim=(1,2))
sample_ema_loss[batch_indices] = (
    0.9 * sample_ema_loss[batch_indices] + 0.1 * per_sample_loss.cpu()
)
# Reweight: use percentile-based thresholding
weight = torch.where(per_sample_loss > percentile_75, 1.5, 1.0)
loss = (weight * per_sample_loss).mean()
```

Flags: `--ohem True --ohem_threshold_pct 75 --ohem_weight 1.5`

**Why it might help:** p_tan=29.1 vs. p_in=12.2 suggests the tandem configurations are persistently harder. If those samples are consistently high-error, OHEM will upweight them relative to easy single-foil samples, pushing the model to allocate more capacity to the hard regime. This is related to the tandem_ramp curriculum (already in baseline) but dynamically adapts throughout training rather than using a fixed schedule.

**Note on tandem_ramp interaction:** The existing `--tandem_ramp` already gives extra weight to tandem samples during training. OHEM is complementary — it reweights within all sample types based on which are currently hardest, not just which domain they belong to.

**Confidence:** Moderate. OHEM is well-validated in detection (SSD, Faster-RCNN) and segmentation. Transfer to regression on meshes is plausible but not directly validated. The main risk is instability from self-referential weighting — the exponential smoothing mitigates this.

### 7. Stochastic Weight Averaging (SWA) — Fixed Endpoint Averaging

**Priority: MEDIUM. NOT in dead-ends, NOT currently running (distinct from SWAD and snapshot ensemble).**

SWA (Izmailov et al., UAI 2018) averages two widely-separated checkpoints: the model at T_max-N and the final model. This is distinct from:
- Snapshot ensemble (#2028, DEAD END): averaged across multiple LR cycle restarts
- SWAD (idea #1 above): dense averaging from within a stable validation window
- EMA (active baseline): exponential moving average maintained throughout training

SWA's mechanism: the two endpoints of training span a wide region of weight space; averaging them lands in a flat basin that often generalizes better than either endpoint alone. The EMA in the baseline is an online average over all training steps, which emphasizes recent checkpoints heavily (ema_decay=0.999 means the window is ~1000 steps). SWA averages two specific checkpoints equally, giving equal weight to different training phases.

**Experiment design:**
```python
# In train.py, after training completes:
# Save checkpoint at epoch T_max - 20 = 140
# Final model = EMA at T_max = 160
# SWA model = (checkpoint_140 + ema_final) / 2  (parameter-wise mean)
# Evaluate SWA model
```

Flags: `--swa True --swa_checkpoint_epoch 140`

If already implemented, check for a `swa` flag. If not, it requires ~10 lines: save a checkpoint at the specified epoch, load both at the end, average their state_dicts.

**Implementation note:** SWA averaging must be done AFTER EMA model extraction — average the two EMA models (not the raw models). Also: batch norm buffers need updating after SWA averaging (if any BN layers exist — check for none in the Transolver).

**Confidence:** Moderate. SWA is well-validated on image classification and recently on some PDE tasks. The key risk is that our EMA already performs a form of temporal averaging, potentially reducing the marginal benefit of SWA. Worth one experiment to confirm or rule out.

### 8. Pressure-Conditioned Tandem Attention Bias

**Priority: MEDIUM. Targeted architectural change specifically for p_tan weakness.**

The tandem foil's inter-foil aerodynamic interaction is fundamentally a non-local spatial effect: the pressure field around the aft foil depends on the wake structure coming off the fore foil. The Transolver's slice attention processes all nodes together through slice tokens, but there is no explicit mechanism for the aft-foil nodes to condition on fore-foil surface information.

A targeted fix: add a learned scalar bias to the slice attention logits for aft-foil surface nodes (boundary ID 7), conditioned on a summary statistic of the fore-foil pressure (e.g., average surface pressure of boundary ID 6). This is a form of domain-aware attention bias, similar to ALiBi (attention with linear biases) but conditioned on physics rather than position.

**Minimal implementation:**
```python
# After existing slice attention computation, before softmax:
if cfg.tandem_attention_bias and is_tandem_sample:
    foil1_mean_p = hidden[foil1_surface_mask].mean(0)  # [n_hidden]
    bias_scale = self.tandem_bias_mlp(foil1_mean_p)    # [1] scalar
    attn_logits[foil2_surface_mask] += bias_scale
```

This is 10–15 lines. The tandem flag from the conditioning vector (`condition` in AdaLayerNorm) already provides an is-tandem signal. The new piece is the conditioning on fore-foil hidden state.

**Why it might help:** p_tan is our most persistent weakness across 1,700 experiments. The tandem configuration has a qualitatively different aerodynamic mechanism (wake-foil interaction) that the current model treats as just another geometry. An explicit inter-foil communication mechanism directly addresses this.

**Confidence:** Moderate. This is a novel idea without direct precedent in the existing experiment history. The risk is that the EMA absorbs the conditioning effect (as happened with HyperNetwork #1837). The distinction from #1837 is that this conditions on intermediate hidden state (data-dependent), not just on scalar flow conditions (fixed per sample).

## Key Distinctions from Prior Dead-Ends

- **SWAD vs Snapshot Ensemble (#2028):** Snapshot averaged across LR restarts (different basins). SWAD averages within one stable plateau (same basin, lower curvature). Fundamentally different mechanism.
- **SWA vs SWAD vs EMA:** SWA = two specific endpoints averaged equally. SWAD = many checkpoints from a plateau window. EMA = exponentially weighted online average (current baseline). All three are distinct.
- **Asymmetric loss vs Learnable Asinh (#2069):** Asinh scale addressed dynamic range compression. Asymmetric loss addresses prediction direction bias. Different axis of improvement.
- **OHEM vs hard example mining (#1890 "never ran"):** PR #1890 proposed but never ran. OHEM is not in the dead-ends list.
- **Tandem attention bias vs HyperNetwork (#1837, val=0.411):** HyperNet conditioned on flow condition scalars (Re, AoA) — static per sample. Tandem bias conditions on intermediate hidden state of the fore-foil surface — dynamic, data-dependent.
