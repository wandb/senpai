## Hypothesis 1 — Local Adaptive Slice Temperature (Transolver++ Eidetic States)

### One-line summary

Replace the global shared learnable temperature tensor with a per-node adaptive temperature predicted from each node's hidden features, preventing homogeneous state assignment in the physics attention mechanism.

### Background and motivation

The Transolver architecture assigns mesh nodes to "physics-meaningful" slice tokens via a soft routing mechanism gated by a temperature parameter. The current implementation uses a single global temperature `self.temperature = nn.Parameter(torch.ones(1, n_heads, 1, 1) * init_temp)` — one scalar per attention head, shared identically across ALL nodes in the mesh.

This is a known failure mode: nodes in qualitatively different flow regimes (surface boundary layer vs. far-field freestream vs. wake) are competing for slice assignments using the same routing sharpness. A high temperature pushes all nodes toward diffuse routing (soft averaging); a low temperature sharply clusters all nodes the same way. Neither is optimal across the full range of physical environments present in a single forward pass.

**Transolver++ (arxiv:2502.02414v2, "Eidetic States")** addresses this directly with "Node-Adaptive Temperature": each node's routing temperature is predicted from its own feature vector:

```
τ_i = τ₀ + Linear(x_i)   # shape: [B, N, n_heads]
```

With a Gumbel-Softmax reparameterization option for discrete-like routing:

```
Softmax((Linear(x) − log(−log(ε))) / τ_i)
```

The paper reports 41% drag coefficient improvement on DrivAerNet++ and 30.4% lift improvement on the AirCraft benchmark. Both are large-scale irregular-mesh aerodynamics tasks directly analogous to TandemFoilSet.

**Why this matters here specifically:** The tandem configuration has a known difficulty (p_tan = 29.1 vs p_in = 12.1 — 2.5x worse). The aft foil sits in the disturbed wake of the fore foil. Wake nodes, boundary-layer nodes, and freestream nodes all have fundamentally different flow physics and should receive different routing sharpness. A global temperature cannot simultaneously be optimal for all three.

Prior temperature experiments in this programme:
- PR #1168 (merged): Added a learnable scalar offset to temperature for the tandem domain — still global across nodes within a domain
- PR #1323 (merged): Per-head tandem temperature offset — still global across nodes

Neither experiment made temperature adaptive at the node level. This is the key untested axis.

### Key paper

**Transolver++: An Efficient Interpolating Solver for Multiphysics PDEs on Irregular Geometries**
Venue: ICML 2025 (follow-up to original Transolver, ICML 2024)
Link: https://arxiv.org/abs/2502.02414

Relevant section: Section 3.2 "Eidetic States: Node-Adaptive Temperature"

**New code — add to `__init__`:**
```python
# Per-node adaptive temperature (Transolver++ Eidetic States)
self.temperature = nn.Parameter(torch.ones(1, n_heads, 1, 1) * init_temp)  # keep as base
self.temp_proj = nn.Linear(n_hidden, n_heads, bias=False)
nn.init.zeros_(self.temp_proj.weight)  # zero-init: starts as global temp
```

**New forward computation (replace attn block):**
```python
# x_node: [B, N, n_hidden]
tau_delta = self.temp_proj(x_node)            # [B, N, n_heads]
tau_delta = tau_delta.permute(0, 2, 1)        # [B, n_heads, N]
tau = self.temperature + tau_delta.unsqueeze(2)  # [B, n_heads, 1, N] broadcast

# attn shape: [B, heads, num_slices, N]
attn = q_slice @ k_node.permute(0, 1, 3, 2)
attn = attn / tau                              # per-node temperature scaling
attn = F.softmax(attn, dim=-1)
```

Zero-initialization of `temp_proj.weight` ensures the model starts from the existing global temperature and only deviates as training proceeds — this is critical for stable initialization.

## Hypothesis 2 — Dedicated Aft-Foil Surface Refinement Branch

### One-line summary

Add a second `SurfaceRefinementHead` that operates exclusively on boundary ID=7 (aft foil surface nodes in tandem configurations), with FiLM conditioning on the gap and stagger features, directly targeting the model's hardest prediction task.

### Background and motivation

The current `SurfaceRefinementHead` (srf_head) was the single biggest win in the programme — PR #2093 (well upstream of this) reports it gave a 72.7% improvement on p_re. It works by applying a small learned MLP to surface-only nodes after the Transolver forward pass, predicting additive corrections.

However, it is a single shared head applied to ALL surface nodes equally: ID=5 (pressure side, single foil), ID=6 (suction side, single foil + fore foil tandem), and ID=7 (aft foil in tandem configs).

**The problem:** Boundary ID=7 is qualitatively different from IDs 5 and 6:
- Aft foil surface sits in the unsteady wake of the fore foil
- The pressure distribution depends heavily on the gap (x[:,0,21]) and stagger (x[:,0,22]) between foils
- In contrast, ID=5 and ID=6 surface nodes are exposed to undisturbed (or mildly disturbed) freestream

The srf_head's corrections for ID=7 nodes are constrained by the same weight tensor used to correct ID=5 nodes — this is a capacity bottleneck. A dedicated branch with direct access to gap/stagger context could learn a much richer correction function for the aft foil.

**Prior related experiments:**
- PR #1893: Upweighted ID=7 nodes in the loss — marginal improvement (still one head)
- PR #1941: Dual surface/volume head — split by surface vs. volume, not by boundary ID
- PR #1863: Domain ensemble heads — never ran
- No prior experiment has tried a separate refinement MLP exclusively for aft-foil surface nodes with conditioning on the tandem geometry parameters

### How the existing srf_head works (from train.py, lines ~900-950)

The `SurfaceRefinementHead` takes as input the concatenation of the transformer hidden state at surface nodes and the base predictions at those nodes. It outputs additive corrections (delta predictions) for Ux, Uy, p. The last linear layer is zero-initialized so the model starts as a pure Transolver output.

The forward applies it only when surface nodes are present:
```python
if self.srf_head is not None and surf_mask.any():
    surf_hidden = hidden[surf_mask]   # [S, n_hidden]
    surf_base = out[surf_mask]        # [S, 3]
    delta = self.srf_head(surf_hidden, surf_base)
    out[surf_mask] += delta
```

### Implementation

**Add to Config dataclass:**
```python
aft_foil_srf: bool = False         # dedicated head for boundary_id=7 nodes
aft_foil_srf_film: bool = False    # FiLM conditioning on gap/stagger features
aft_foil_srf_hidden: int = 192     # hidden size for aft foil head
aft_foil_srf_layers: int = 3       # layers for aft foil head
```

**Add to `Transolver.__init__`:**
```python
if cfg.aft_foil_srf:
    self.aft_srf_head = SurfaceRefinementHead(
        n_hidden=cfg.n_hidden,
        hidden_size=cfg.aft_foil_srf_hidden,
        n_layers=cfg.aft_foil_srf_layers,
        out_dim=3,
        # If FiLM conditioning: input_dim = n_hidden + 2 (gap + stagger)
        film_cond_dim=2 if cfg.aft_foil_srf_film else 0,
    )
else:
    self.aft_srf_head = None
```

**Add to `Transolver.forward` (after the existing srf_head block):**
```python
if self.aft_srf_head is not None:
    # is_tandem detected via x[:, 0, 21].abs() > 0.01 (existing logic)
    aft_mask = (boundary_id == 7) & is_tandem_expanded  # [B, N] bool
    if aft_mask.any():
        aft_hidden = hidden[aft_mask]        # [A, n_hidden]
        aft_base = out[aft_mask]             # [A, 3]
        
        if self.cfg.aft_foil_srf_film:
            # gap and stagger are per-sample scalars; expand to per-node
            gap = x[:, 0, 21]               # [B]
            stagger = x[:, 0, 22]           # [B]
            # expand to aft-masked nodes
            gap_expanded = gap.unsqueeze(1).expand(-1, N)[aft_mask]      # [A]
            stagger_expanded = stagger.unsqueeze(1).expand(-1, N)[aft_mask]  # [A]
            cond = torch.stack([gap_expanded, stagger_expanded], dim=1)  # [A, 2]
            delta = self.aft_srf_head(aft_hidden, aft_base, cond=cond)
        else:
            delta = self.aft_srf_head(aft_hidden, aft_base)
        
        out[aft_mask] += delta
```

**FiLM conditioning in SurfaceRefinementHead (if `film_cond_dim > 0`):**

If the existing `SurfaceRefinementHead` class doesn't support a `cond` input, add FiLM as follows. After the first linear layer, apply:
```python
# scale and shift from condition vector
gamma = self.film_scale(cond)    # [A, hidden_size]
beta = self.film_shift(cond)     # [A, hidden_size]
h = h * (1 + gamma) + beta       # FiLM modulation
```
with `self.film_scale = nn.Linear(2, hidden_size, bias=False)` and `self.film_shift = nn.Linear(2, hidden_size)`, both zero-initialized so they start as identity.
