## Hypothesis 1 — Coordinate Frame Normalization per-Foil

### One-line summary

Normalize aft foil (boundary ID=7) mesh node coordinates into the aft foil's own local coordinate frame before feeding into the Transolver, giving the model equivariant representations of tandem geometry.

### Background and motivation

The Transolver receives raw global (x, y) coordinates for all mesh nodes — both fore and aft foil surfaces. In tandem configurations, the aft foil is displaced by `gap` (x[:,0,21]) and `stagger` (x[:,0,22]) from the fore foil. The model must implicitly learn that the same aft foil geometry at different (gap, stagger) settings is fundamentally the same object just translated.

This is a known problem in geometric deep learning: networks that receive globally-referenced coordinates must learn translation equivariance from scratch using gradient descent. For a 3-layer network with limited capacity, this is a significant waste of representational budget.

**The fix:** Before passing features through the Transolver, subtract the aft foil centroid from the (x, y) coordinates of ID=7 nodes. This puts aft foil nodes in their own local frame — the network now sees the same aft foil shape regardless of gap/stagger placement. The gap and stagger scalars (already present in x[:,0,21:23]) are kept as explicit conditioning features to preserve global placement information.

**Why this matters for p_tan specifically:** The aft foil pressure distribution is determined primarily by:
1. The aft foil's own shape (captured in local frame)
2. The incoming wake, which depends on gap and stagger (captured as explicit scalars)

The current representation conflates these two contributions. Local frame normalization disentangles them — the same physics appears at the same coordinates regardless of global placement, making the attention mechanism's job dramatically easier.

**Physical motivation:** This is the standard approach in aerodynamics: pitching moment coefficients are always computed about the quarter-chord of each individual foil, not a global reference. The network should see foil-local coordinates.

### Implementation

The change is localized to preprocessing within `Transolver.forward`. The aft foil centroid can be estimated from ID=7 node positions within each batch sample.

**Add to `Transolver.forward` (before the Transolver blocks, after input preparation):**

```python
if cfg.aft_foil_local_frame:
    # boundary_id shape: [B, N] or detectable from x feature
    # is_tandem: [B] bool, existing logic: x[:, 0, 21].abs() > 0.01
    is_tandem = (x[:, 0, 21].abs() > 0.01)  # [B]
    
    # aft foil mask: boundary_id == 7 for tandem samples
    # boundary_id is typically stored in x[:, :, some_channel]
    # From program.md: boundary ID is in the input feature vector
    aft_mask = (boundary_id == 7)  # [B, N] bool
    
    # Compute centroid of aft foil per sample
    for b in range(x.size(0)):
        if is_tandem[b] and aft_mask[b].any():
            aft_x = x[b, aft_mask[b], 0]   # aft foil x-coords
            aft_y = x[b, aft_mask[b], 1]   # aft foil y-coords
            cx = aft_x.mean()
            cy = aft_y.mean()
            # Subtract centroid from aft foil node coordinates only
            x[b, aft_mask[b], 0] -= cx
            x[b, aft_mask[b], 1] -= cy
            # gap and stagger in x[:,0,21:23] are preserved as global context
```

A vectorized version (avoiding the for loop):

```python
if cfg.aft_foil_local_frame:
    is_tandem = (x[:, 0, 21].abs() > 0.01)       # [B]
    aft_mask = (boundary_id == 7)                  # [B, N]
    aft_mask_tandem = aft_mask & is_tandem.unsqueeze(1)  # [B, N]
    
    # Masked mean for centroid (handle empty mask with clamp)
    aft_count = aft_mask_tandem.float().sum(dim=1, keepdim=True).clamp(min=1)  # [B, 1]
    cx = (x[:, :, 0] * aft_mask_tandem.float()).sum(dim=1, keepdim=True) / aft_count  # [B, 1]
    cy = (x[:, :, 1] * aft_mask_tandem.float()).sum(dim=1, keepdim=True) / aft_count  # [B, 1]
    
    # Subtract centroid from aft foil nodes only
    x[:, :, 0] -= cx * aft_mask_tandem.float()
    x[:, :, 1] -= cy * aft_mask_tandem.float()
```

## Hypothesis 2 — Progressive Surface Focus Schedule

### One-line summary

Ramp the surface loss weight from a low initial value (e.g., 1.0) up to the target value (e.g., 20.0) over the first N training epochs, using curriculum learning to prevent the model from overfitting surface patterns before it has learned the bulk flow structure.

### Background and motivation

The current training loss applies a fixed `surf_weight=20.0` (dynamically adjusted by loss ratio, clamped [5, 50]) from the very first epoch. This means the network receives a 20× amplified surface gradient signal before it has learned any useful bulk-flow representation.

**The problem:** Early in training, the model's predictions are random. The high surface weight makes the network immediately optimize for surface nodes — but without a good bulk-flow backbone, the surface predictions are built on a poor foundation. This is analogous to training a segmentation model with full boundary weight from epoch 1, before the backbone has learned shape features.

**Curriculum learning solution:** Start with `surf_weight=1.0` (equal treatment of surface and volume nodes) and ramp it up to the target value over a warm-in period. This forces the model to first build a coherent global flow field, then refine the surface predictions on top of that foundation.

**Distinct from existing mechanisms:**
- `tandem_ramp`: ramps the tandem surface loss contribution from 0→1 over epochs 10-50. This is about tandem vs. single-foil weighting, not about surface vs. volume weighting.
- Dynamic `surf_weight`: already adapts based on loss ratio. The proposal is to additionally impose a curriculum schedule on top of — or instead of — the dynamic ratio adjustment.
- `vol_ramp_epochs`: ramps volume weight, not surface weight.

The proposed change is to the INITIAL value and ramp schedule of `surf_weight`, not its final target or dynamic adjustment logic.

### Implementation

**Add to Config dataclass:**
```python
surf_weight_ramp_epochs: int = 0    # 0 = disabled; >0 = epochs to ramp surf_weight 1→target
surf_weight_ramp_start: float = 1.0  # initial surf_weight during ramp phase
```

**In the training loop (around the surf_weight assignment, near line 1379):**

```python
# Progressive surface focus schedule

    # Linear ramp from surf_weight_ramp_start to dynamic surf_weight
    dynamic_surf_weight = max(5.0, min(50.0, prev_vol_loss / max(prev_surf_loss, 1e-8)))
    surf_weight = cfg.surf_weight_ramp_start + (dynamic_surf_weight - cfg.surf_weight_ramp_start) * ramp_frac
else:
    surf_weight = max(5.0, min(50.0, prev_vol_loss / max(prev_surf_loss, 1e-8)))
```

Or simpler — linear ramp ignoring the dynamic adjustment during warm-in:

## Hypothesis 3 — Contrastive Tandem-Single Regularization

### One-line summary

Add a contrastive regularization loss that pushes apart the Transolver's hidden representations of tandem configurations (aft foil wake) and single-foil configurations, forcing the model to develop distinct internal representations for these physically different flow regimes.

### Background and motivation

A core challenge in this dataset is that tandem and single-foil configurations are drawn from physically distinct flow regimes, but the model must handle both with the same weights. The model currently has no explicit incentive to maintain separate internal representations for the two regimes.

This leads to a representational entanglement: the attention mechanism's slice assignments are shared across tandem and single-foil samples within each batch. When the model learns to route single-foil nodes efficiently (which dominate the dataset), it may simultaneously damage its routing for tandem-specific patterns.

**Contrastive regularization solution:** Add a loss term that explicitly penalizes the model if the mean hidden representation of tandem samples is too similar to the mean hidden representation of single-foil samples within the same batch. This is analogous to the supervised contrastive loss (Khosla et al., NeurIPS 2020), but applied to the model's intermediate features rather than final predictions.

**Distinct from all prior experiments:**
- PR #1893 (loss upweighting): Added more loss weight to ID=7 nodes — still one head, no representation separation
- PCGrad (merged #1456): Gradient projection across OOD splits — different mechanism (gradient space vs. representation space)
- DomainLayerNorm (existing): Normalizes separately per domain — operates at normalization level, not representation level

This operates at the level of the hidden state $h \in \mathbb{R}^{B \times N \times d}$ before the final output projection.

### Implementation

**In the training loop, after the forward pass:**

```python
if cfg.contrastive_tandem_weight > 0.0:
    # hidden: [B, N, n_hidden] — Transolver output before final projection
    is_tandem = (x[:, 0, 21].abs() > 0.01)   # [B] bool
    
    if is_tandem.any() and (~is_tandem).any():
        # Mean-pool hidden state per sample → [B, n_hidden]
        h_mean = model_out["hidden"].mean(dim=1)  # [B, n_hidden]
        
        # Mean centroid for tandem and single batches
        h_tandem = h_mean[is_tandem].mean(dim=0)      # [n_hidden]
        h_single = h_mean[~is_tandem].mean(dim=0)     # [n_hidden]
        
        # L2 distance in representation space (want this to be large)
        rep_dist = F.normalize(h_tandem, dim=0) @ F.normalize(h_single, dim=0)
        # rep_dist = cosine similarity ∈ [-1, 1]; we want it to be small/negative
        # Loss: push representations apart (minimize cosine similarity)
        contrastive_loss = cfg.contrastive_tandem_weight * (1.0 + rep_dist)
        loss = loss + contrastive_loss
```

The `(1.0 + rep_dist)` term is zero when the representations are orthogonal (cosine sim = -1) and maximized at 2.0 when they are identical (cosine sim = +1). This is equivalent to the "push" term in the NT-Xent loss without the temperature.

**Require `model_out["hidden"]` to be exposed from forward** (check whether it already is; if not, add `"hidden": fx` to the model's output dict).

**Add to Config dataclass:**
```python

```

## Hypothesis 4 — Asymmetric Fixed Asinh Scales (Positive/Negative Pressure)

### One-line summary

Apply two separate fixed (non-learnable) asinh compression scales to the pressure channel — one scale for positive pressures and a different scale for negative pressures — to better handle the physical asymmetry of pressure distributions around airfoils.

### Background and motivation

The current asinh pressure transform uses a single fixed scale `s=0.75` applied symmetrically: `asinh(p * 0.75)`. This treats positive and negative pressure values identically in terms of compression.

However, pressure around an airfoil is physically asymmetric:
- **Positive pressure** (stagnation regions, pressure side): moderate magnitude, concentrated near leading edge
- **Negative pressure** (suction side, suction peaks): much larger magnitude, can reach -5 to -10 Cp, highly concentrated in a small region near leading edge

The single scale 0.75 was found to improve p_oodc by 4-6% (PR #2054). But this single scale must compromise between compressing the large negative suction peaks (needs large s) and preserving the moderate positive stagnation pressure (needs smaller s).

**Asymmetric fixed scales solution:** Apply `asinh(p * s_pos)` where p > 0 and `asinh(p * s_neg)` where p < 0, with `s_pos` and `s_neg` independently chosen. This allows the transform to optimally compress both the suction peak extremes and the stagnation region separately.

**CRITICAL DISTINCTION FROM DEAD END #2096:** PR #2096 tried a *learnable* asinh scale `nn.Parameter(0.75)` and found it collapsed to `s → 0.003` (trivial shortcut). This proposal uses **fixed, non-learnable** scales — the values are set as config hyperparameters and are not optimized by gradient descent. The failure mode of #2096 (joint optimization creating trivial shortcuts) does not apply here.

**Suggested initial values based on physical reasoning:**
- `s_neg = 1.5` — stronger compression for large-magnitude negative suction peaks
- `s_pos = 0.5` — lighter compression for moderate positive stagnation pressures

These can be tuned empirically.

### Implementation

**In the preprocessing code where asinh transform is applied (near the existing `asinh_pressure` block):**

```python
if cfg.asinh_pressure:
    if cfg.asinh_asymmetric:
        # Separate scales for positive and negative pressure
        p_channel = _yp[:, :, 2:3]
        p_neg = torch.where(p_channel < 0, torch.asinh(p_channel * cfg.asinh_scale_neg), torch.zeros_like(p_channel))
        p_pos = torch.where(p_channel >= 0, torch.asinh(p_channel * cfg.asinh_scale_pos), torch.zeros_like(p_channel))
        _yp[:, :, 2:3] = p_neg + p_pos
    else:
        # Existing symmetric transform
        _yp[:, :, 2:3] = torch.asinh(_yp[:, :, 2:3] * cfg.asinh_scale)
```

Apply the same asymmetric transform to the target `_y` (must be symmetric with the input transform):
```python

        p_channel = _y[:, :, 2:3]
        p_neg = torch.where(p_channel < 0, torch.asinh(p_channel * cfg.asinh_scale_neg), torch.zeros_like(p_channel))
        p_pos = torch.where(p_channel >= 0, torch.asinh(p_channel * cfg.asinh_scale_pos), torch.zeros_like(p_channel))
        _y[:, :, 2:3] = p_neg + p_pos
```

## Hypothesis 5 — Test-Time Augmentation via AoA Perturbation

### One-line summary

At inference time, evaluate the model at AoA ± δ (small angle-of-attack perturbations) and average the predictions, exploiting the physical smoothness of CFD solutions with respect to AoA to reduce prediction variance without any training cost.

### Background and motivation

The `aug="aoa_perturb"` training augmentation applies random ±1° AoA rotations during training. **This is training-time only** — at inference, the model receives a single input and produces a single prediction. Test-Time Augmentation (TTA) is a distinct and separately-tested mechanism.

**Physical motivation:** CFD solutions are smooth functions of angle of attack. For a given geometry, the pressure distribution at AoA=8° is similar to — and physically interpolated between — the solutions at AoA=7° and AoA=9°. A neural surrogate with any remaining overfitting to specific AoA values will produce slightly different errors at AoA ± δ. Averaging these three predictions cancels the error components that vary rapidly with AoA (i.e., model artifacts) while preserving the true physical signal (which varies slowly).

**Distinct from the training augmentation:** The training augmentation applies random rotation during training to improve generalization. TTA at inference is a separate technique that can be applied regardless of whether the training augmentation was used, and works through a different mechanism (variance reduction across a small neighborhood vs. data augmentation for better learning).

**This has never been tried at inference time in this programme.** The `aug="aoa_perturb"` flag exists only for training. Inference-time TTA requires separate implementation in the evaluation loop.

**AoA feature location:** From the analysis of `train.py` and `prepare_multi.py`, the AoA is encoded in the input features via the rotation applied to (x, y) node coordinates (the `aoa_perturb` code rotates `x[:,:,0:2]` by the angle). TTA must apply the same coordinate rotation to the input, not perturb a raw feature index.

### Implementation

**In the evaluation/inference loop:**

```python
if cfg.tta_aoa and delta_degrees > 0:
    aoa_deltas = [-cfg.tta_aoa_delta, 0.0, cfg.tta_aoa_delta]  # e.g., [-1, 0, +1] degrees
    preds_list = []
    
    for delta in aoa_deltas:
        x_aug = x.clone()
        if abs(delta) > 1e-6:
            angle_rad = delta * (torch.pi / 180.0)
            cos_a = torch.cos(torch.tensor(angle_rad, device=x.device))
            sin_a = torch.sin(torch.tensor(angle_rad, device=x.device))
            # Rotate spatial coordinates (x[:,:,0:2])
            x_orig = x_aug[:, :, 0:1].clone()
            y_orig = x_aug[:, :, 1:2].clone()
            x_aug[:, :, 0:1] = cos_a * x_orig - sin_a * y_orig
            x_aug[:, :, 1:2] = sin_a * x_orig + cos_a * y_orig
        
        with torch.no_grad():
            pred = model(x_aug, ...)["preds"]
            # Rotate predictions back (velocity channels need inverse rotation)
            if abs(delta) > 1e-6:
                ux_pred = pred[:, :, 0:1].clone()
                uy_pred = pred[:, :, 1:2].clone()
                pred[:, :, 0:1] = cos_a * ux_pred + sin_a * uy_pred   # inverse rotation
                pred[:, :, 1:2] = -sin_a * ux_pred + cos_a * uy_pred
        preds_list.append(pred)
    
    final_pred = torch.stack(preds_list).mean(0)
else:
    final_pred = model(x, ...)["preds"]
```

**Key implementation note:** The velocity predictions (Ux, Uy) must be rotated BACK from the perturbed frame to the original frame before averaging. Pressure predictions (scalar) do not need rotation. This is critical for correctness — failing to unrotate velocities will corrupt the Ux/Uy predictions even if p improves.

**Add to Config dataclass:**


**Assignment priority:**

1. **Asymmetric Asinh** — assign first: zero parameter cost, no joint optimization risk, directly targets the physical asymmetry of the pressure distribution that the current single-scale transform cannot handle. Distinct from dead-end #2096.

2. **Coordinate Frame Normalization** — assign in parallel: targets p_tan directly, zero parameters, theoretically motivated by equivariance principles, surgical change.

3. **Progressive Surface Focus Schedule** — assign in parallel: ultra-low complexity, no parameters, may compound with both ideas above; targets all tracks.

4. **Contrastive Tandem-Single Regularization** — assign once a student becomes available: more complex, needs careful weight tuning, but targets the fundamental representation entanglement issue.

5. **TTA via AoA Perturbation** — assign last or as a fast add-on to any completed run: inference-only change, can be applied to any checkpoint including the current baseline, zero training cost.
